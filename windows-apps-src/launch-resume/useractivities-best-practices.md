# <a name="user-activities-best-practices"></a>사용자 작업에 대 한 유용한 정보

이 가이드에서는 만들기 (영문) 및 사용자 작업을 업데이트 하기 위한 권장 되는 방법에 설명 합니다. Windows에서 사용자 활동 기능 개요, [장치 에서도 계속 사용자 활동](https://docs.microsoft.com/windows/uwp/launch-resume/useractivities)을 참조 하십시오. 또는 다른 개발 플랫폼에서 작업의 구현에 대 한 프로젝트 로마의 [사용자 작업 섹션](https://docs.microsoft.com/windows/project-rome/user-activities/) 을 참조 하십시오.

## <a name="when-to-create-or-update-user-activities"></a>만들거나 사용자 활동을 업데이트 하는 경우

모든 응용 프로그램은 다른, 이므로 사용자 작업으로 응용 프로그램 내에서 작업을 매핑하는 가장 좋은 방법은 확인 하려면 각 개발자에 게 있습니다. 사용자 작업에 초점을 맞춘 증가 함에 따라 사용자의 생산성과 효율성을 이전에가 방문 하는 콘텐츠를 다시 볼 줌으로써 Cortana 및 시간 표시 막대를 보여 줍니다 될 됩니다.

**일반 지침**

* **관련된 사용자 작업의 그룹에 대 한 단일 활동을 기록 합니다.** 이 특히 음악 재생 목록 또는 TV 프로그램에 대 한 관련 된: 사용자의 진행 상황을 반영 하기 위해 정기적으로 단일 활동을 업데이트할 수 있습니다. 이 경우 단일 사용자 활동을 여러 며칠 또는 몇 주가 별 계약의 기간을 나타내는 여러 기록 항목 사용 해야 합니다. 같은 문서 기반 작업에는 사용자가 응용 프로그램 내에서 점진적으로 진행 상황에 적용 됩니다.
* **클라우드에서 사용자 데이터를 저장 합니다.** 장치 간 활동을 지원 하려는 경우 다시이 작업을 연결 하는 데 필요한 콘텐츠 클라우드 위치에 저장 되어 있는지 확인 하려면 필요 합니다. 장치별 활동 활동 만들어진 위치 장치에서 시간 표시 막대에 표시 됩니다 하지만 다른 장치에 나타나지 않을 수도 있습니다.
* **사용자가 다시 시작할 필요가 없습니다 있는 작업에 대 한 활동을 만들지 마십시오.** 응용 프로그램 상태를 유지 하지 않는 간단 하 고 일회성 작업을 완료 하는데 사용 하는 경우에 잠재적인 사용자 활동을 만들 하지 필요가 있습니다.
* **다른 사용자가 완료 된 작업에 대 한 활동을 만들지 마십시오.** 외부 계정 사용자는 메시지를 보내면 또는 @-mentions 해당 응용 프로그램 내에서 만들지 마십시오 활동이 합니다. 이 유형의 작업은 더 잘 작업 센터 알림에 의해 반환 됩니다.
  * 공동 작업 시나리오는 예외가:는 다른 사용자가 변경한 후에 사용자의 경우 될 여러 사용자가 작동 하는 경우 동일한 활동에서 함께 바인딩 (예: Word 문서) 합니다. 이 경우 다음 문서에 대 한 변경 내용을 반영 하도록 기존 활동을 업데이트 하는 것이 좋습니다. 도 여기에 새 기록 항목 만들기 (영문) 하지 않고 기존 사용자 활동 콘텐츠 데이터를 업데이트 합니다.

**특정 유형의 응용 프로그램에 대 한 지침**

모든 응용 프로그램은 다른, 하는 동안 대부분의 앱 다음 상호작용 패턴 중 하나에 속합니다 됩니다.
* **문서 기반 앱** -사용 기간을 반영 하는 하나 이상의 기록 항목을 사용 하 여 문서, 당 하나의 활동을 만들기. 것은 문서에 변경 된 작업을 업데이트 하는 것이 중요 합니다.
* **게임** -각 게임 저장 또는 해외에 대 한 하나의 활동을 만듭니다. 게임에만 수준 단일 시퀀스를 지 원하는 경우 다시 게시할 수 동일한 작업 시간이 지남에 따라 최신 진행률 또는 성과 표시 하려면 콘텐츠 데이터를 업데이트 하는 것이 좋을 있지만 합니다.
* **유틸리티 앱** -사용자 작업을 사용 하 여 필요 없을 경우 사용자는 그대로 유지 하 고 다시 시작 하는 응용 프로그램 내에서 아무것도, 없습니다. 좋은 예 계산기와 같은 간단한 응용 프로그램입니다.
* **비즈니스 라인 앱** -하 여 간단한 작업 또는 워크플로 관리 하기 위한 많은 앱 존재 합니다. 앱을 통해 액세스 하는 별도 워크플로별로 하나의 활동을 만듭니다 (등 경비 보고 별도 활동을 각각 될 사용자는 특정 보고서를 승인 하는 경우를 참조 하는 작업을 클릭 수 있도록).
* **미디어 재생 앱** -콘텐츠 (예: 재생 목록, 프로그램 또는 독립 실행형 콘텐츠)의 논리적 그룹 당 하나의 활동을 만듭니다. 앱 개발자를 위한 원본으로 사용 되는 질문은 독립 실행형 콘텐츠 또는 컬렉션의 일부로 (TV 에피소드, 노래나) 콘텐츠의 각 부분을 계산 하는 여부입니다. 일반적으로 사용자 참여할 컬렉션이 나 순차 콘텐츠를 재생 하는 경우 전체 컬렉션에는 활동입니다. 단일 콘텐츠의 재생을 거부 하는 경우 해당 하나의 콘텐츠의 활동입니다. 아래의 보다 구체적인 지침을 참조 하십시오.
  * **음악: 앨범/아티스트/장르** -앨범, 음악가, 또는 장르를 선택 하 고 **재생**를 방문한 하는 사용자, 해당 컬렉션은 작업입니다. 각 노래나에 대 한 별도 활동을 기록 하지 않습니다. 단일 앨범 같은 짧은 컬렉션 또는 임의의 순서로 다시 재생 되는 모음에 대 한 사용자의 현재 위치를 반영 하도록 활동을 업데이트 하려면 필요 하지 있습니다. 앨범 또는 재생 목록 같은 긴 순차 재생에 대 한 앨범 내에서 위치를 녹음/녹화이 좋을 수도 있습니다.
  * **음악: 재생 목록 스마트** -임의의 순서로 음악을 재생 하는 응용 프로그램 해당 재생 목록에 대 한 단일 활동을 기록해 야 합니다. 사용자는 재생 시간이 내, 재생 하는 경우 동일한 활동에 대 한 추가 기록 레코드를 만듭니다. 재생 목록에서 사용자의 현재 위치를 녹음/녹화의 순서를 지정 하는 것은 임의의 하기 때문에 필요 하지 않습니다.
  * **TV 시리즈** -앱을 현재 완료 되 면 다음 에피소드를 재생 하도록 구성 하는 경우 TV 계열에 대 한 단일 활동을 작성 해야 합니다. 여러 보기 세션에서 다양 한 에피소드를 재생할 때는 일련의 현재 위치를 반영 하기 위해 작업을 업데이트 해 봅니다 및 여러 기록 레코드를 만들 합니다.
  * **동영상** -동영상 단일 콘텐츠의 이며 자체 기록 레코드가 있어야 합니다. 사용자를 통해 동영상 파트 방향 시청 중지 되 면 해당 위치를 기록 하는 것이 좋습니다. 나중에 다시 시작 하려고 할 때 활동에 마지막으로, 동영상을 다시 시작 하거나도 다시 시작 하거나 처음부터 시작 하려는 경우 사용자에 게 수 없습니다.

## <a name="user-activity-design"></a>사용자 작업 디자인

사용자 활동 세 구성 요소로 구성:는 정품 인증 URI, 시각적 데이터 및 콘텐츠 메타 데이터입니다.
* 정품 인증 URI는 하는 URI와 특정 컨텍스트의 응용 프로그램을 다시 시작 하기 위해 응용 프로그램 또는 경험에 전달할 수입니다. 일반적으로 이러한 링크는 구성표 (예를들어, "내-app://page2?action=edit")에 대 한 프로토콜 처리기의 형식을 따릅니다. 응용 프로그램을 통해 하 여 URI 매개 변수를 처리 하는 방법을 결정 하는 개발자 달려있습니다. 자세한 내용은 [URI를 처리 정품 인증](https://docs.microsoft.com/windows/uwp/launch-resume/handle-uri-activation) 을 참조 하십시오.
* 필수 및 선택적 속성 집합으로 이루어진 시각적 데이터는 (예: 제목, 설명 또는 환경에 적합 한 카드 요소)를 시각적으로 활동을 식별 하는 사용자를 허용 합니다. 작업에 대 한 환경에 적합 한 카드 시각 효과 만들기 (영문)에 대 한 지침은 아래를 참조 합니다.
* 콘텐츠 메타 데이터는 JSON 데이터를 그룹화 하 고 특정 컨텍스트 내에서 작업을 검색 하는데 사용할 수입니다. 일반적으로이 형식은 다음과 http://schema.org 데이터입니다. 이 데이터를 채울에 대 한 지침은 아래를 참조 합니다.

### <a name="adaptive-card-design-guidelines"></a>환경에 적합 한 카드 디자인 지침

작업 시간 표시 막대에 표시, [환경에 적합 한 카드 프레임 워크](https://docs.microsoft.com/adaptive-cards/)를 사용 하 여 표시 됩니다. 개발자는 각 작업에 대 한 환경에 적합 한 카드는 제공 하지 않으면, 시간 표시 막대의 앱 이름/아이콘, 필요한 제목 필드 및 선택적 설명 필드에 따라 간단한 카드를 자동으로 생성 됩니다. 

응용 프로그램 개발자는 단순한 환경에 적합 한 카드 JSON 스키마를 사용 하 여 사용자 지정 카드를 제공 하는 것이 좋습니다. 환경에 적합 한 카드 개체를 생성 하는 방법에 대 한 자세한 내용은 기술 [환경에 적합 한 카드 설명서](https://docs.microsoft.com/adaptive-cards/authoring-cards/getting-started) 를 참조 합니다. 사용자 활동에 환경에 적합 한 카드 디자인 (영문)에 대 한 아래 지침을 참조 하십시오.
* 이미지를 사용 하 여
  * 가능한 경우 각 작업에 대 한 고유 이미지를 사용 합니다. 응용 프로그램 이름 및 아이콘 자동으로 표시할 활동의 카드; 옆에 있습니다 추가 이미지에는 사용자가 원하는 활동을 찾는 데 도움이 됩니다.
  * 이미지 텍스트를 포함 하지 않아야 사용자를 읽는 예상 되는 합니다. 이 텍스트 내게 필요한 옵션 요구를 가진 사용자에 게 사용할 수 없으며 검색할 수 없습니다.
  * 이미지의 텍스트를 포함 하지 하 2:1 비율로 하는 방법에 대 한에 잘릴 수 하는 경우 배경 이미지로 사용 해야 합니다. 이 시간 표시 막대 눈에 띄는 굵게 활동 카드 만들어집니다. 이미지는 텍스트 계속의 카드에 표시 하 고만 활동이 이름을 사용이 경우 더 작은 텍스트는 읽기가 될 수 있는 것을 권장 하는 되도록 약간 어 두 됩니다.
  * 2: 1로 이미지를 자를 수 없습니다.을 하는 경우 활동 카드 내에서 배치 해야 합니다.  
    * 가로 세로 비율이 정사각형 또는 세로 있으면 여백 없이 카드의 오른쪽에 이미지를 고정 합니다.
    * 가로 세로 비율을 사용 하는 경우에 카드의 오른쪽 위 모서리에 이미지를 고정 합니다.
* 각 작업은 항상 표시 되어야 하는 활동 이름을 제공 해야 합니다.
  * 이 이름은 큰 텍스트를 굵게 표시 옵션을 사용 하 여 카드의 왼쪽 위 모서리에 표시할 합니다. 상태를 발이 있는 경우에 그대로 이름 인지 쉽게 알아볼 수 있는 일부 사용자 활동 Cortana 시나리오에 표시 되는 경우에 표시 됩니다. 시간 표시 막대에 동일한 대화명을 사용 하면 많은 활동을 이동 하는 사용자에 대 한 보다 쉽게 합니다.
* 사용자가 시간 표시 막대에서 앱의 활동을 쉽게 찾을 수 있도록 모든 응용 프로그램에서 작업에 대 한 동일한 시각적 스타일을 사용 합니다.
  * 예, 활동 모두 사용 해야 동일한 배경색 합니다.
* 추가 텍스트 정보를 신중 하 게 사용 합니다. 
  * 카드를 텍스트를 채운 하지 마십시오 하 고만 오른쪽 활동 찾기 (영문)에서 사용자를 지원 하거나 (예: 특정 작업의 현재 진행) 상태 정보를 반영 하는 추가 정보를 사용 합니다.

### <a name="content-metadata-guidelines"></a>콘텐츠 메타 데이터 지침

사용자 작업 활동을 분류 하 고 유추가 생성를 사용 하 여 Windows 및 Cortana 콘텐츠 메타 데이터를 포함할 수도 있습니다. 활동 (사용자가 다른 응용 프로그램 및 웹 사이트 전체에서 특정 제품에 대 한 쇼핑) 하는 경우 위치 (사용자가 휴가 연구) 하는 경우, 개체 (사용자가 항목을 리서치) 하는 경우 또는 작업 등의 특정 항목 축을 그룹화 한 다음 수 있습니다. 명사와 활동에 관련된 된 동사를 모두 표시 하는 것이 좋습니다. 

다음 예제에서는 콘텐츠 메타 데이터 [Schema.org](https://schema.org/)표준에 따라 JSON 나타냅니다 시나리오: "John 재생 화난 새 Steve와."

```json
// John played angry birds with Steve.
{
  "@context": "http://schema.org",
```
