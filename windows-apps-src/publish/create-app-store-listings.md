# <a name="create-app-store-listings"></a>앱 스토어 목록 만들기

[앱 제출 프로세스](app-submissions.md) 의 **스토어 목록** 섹션은 고객이 Microsoft Store에서 앱 목록을 볼 때 표시 되는 텍스트와 [이미지](app-screenshots-and-images.md) 를 제공 하는 위치입니다.

**매장 목록** 에 있는 대부분의 필드는 선택 사항 이지만 여러 이미지를 제공 하 고 가능한 한 많은 정보를 제공 하 여 목록이 표시 되도록 하는 것이 좋습니다. **저장소 목록** 단계를 완료로 간주 하는 데 필요한 최소는 텍스트 설명 및 하나 이상의 [스크린샷](app-screenshots-and-images.md#screenshots)입니다.

> [!TIP]
> 파트너 센터에서 직접 정보를 제공 하 고 파일을 업로드 하는 대신 .csv 파일에 나열 정보를 오프 라인으로 입력 하려는 경우에는 필요에 따라 [저장소 목록을 가져오고 내보낼](import-and-export-store-listings.md) 수 있습니다. 가져오기 및 내보내기 옵션을 사용 하는 것은 여러 언어에서 한 번에 여러 업데이트를 수행할 수 있으므로 특히 편리 합니다.

이전에 게시 된 앱이 Windows 8.x 및/또는 Windows Phone .x 이전 버전을 지 원하는 경우 해당 고객에 게 표시할 [플랫폼별 저장소 목록을 만들](create-platform-specific-store-listings.md) 수 있습니다.

## <a name="store-listing-languages"></a>스토어 목록 언어

하나 이상의 언어에 대해 **스토어 목록** 페이지를 완료 해야 합니다. 패키지에서 지 원하는 각 언어의 상점 목록을 제공 하는 것이 좋지만 스토어 목록을 제공 하지 않으려는 언어를 유연 하 게 제거할 수 있습니다. 패키지에서 지원 하지 않는 추가 언어로 스토어 목록을 만들 수도 있습니다.

> [!NOTE]
> 제출에 이미 패키지가 포함 된 경우 해당 패키지를 제거 하지 않는 한 제출 개요 페이지에서 패키지에서 지원 되는 [언어](supported-languages.md) 를 표시 합니다.

스토어 목록의 언어를 추가 하거나 제거 하려면 제출 개요 페이지에서 **언어 추가/제거** 를 클릭 합니다. 패키지를 이미 업로드 한 경우 **패키지에서 지 원하는 언어** 섹션에 해당 언어가 표시 됩니다. 이러한 언어를 하나 이상 제거 하려면 **제거**를 클릭 합니다. 나중에이 섹션에서 제거한 언어를 포함 하도록 결정 한 경우 **추가**를 클릭 하면 됩니다.

**추가 스토어 목록 언어** 섹션에서 **추가 언어 관리** 를 클릭 하 여 패키지에 포함 *되지* 않은 언어를 추가 하거나 제거할 수 있습니다. 추가 하려는 언어의 확인란을 선택 하 고 **업데이트**를 클릭 합니다. 선택한 언어가 **추가 매장 목록 언어** 섹션에 표시 됩니다. 이러한 언어를 하나 이상 제거 하려면 **제거** 를 클릭 하거나, **추가 언어 관리** 를 클릭 하 고 제거 하려는 언어의 확인란을 선택 취소 합니다.

선택을 완료하면 **저장**을 클릭하여 제출 개요 페이지로 돌아갑니다.

## <a name="add-and-edit-store-listing-info"></a>저장소 목록 정보 추가 및 편집

매장 목록을 편집 하려면 제출 개요 페이지에서 언어 이름을 선택 합니다. 저장소 목록을 내보내고 오프 라인으로 작업을 선택한 다음, 모든 목록 데이터를 한 번에 가져오는 경우가 아니면 각 언어를 별도로 편집 해야 합니다. 작동 방식에 대 한 자세한 내용은 [저장소 목록 가져오기 및 내보내기](import-and-export-store-listings.md)를 참조 하세요.

사용 가능한 필드는 아래에 설명 되어 있습니다.

## <a name="product-name"></a>제품 이름

이 드롭다운 상자를 사용 하 여 스토어 목록에서 사용할 이름을 지정할 수 있습니다 (앱에 대해 둘 이상의 이름을 예약한 경우).

작업 중인 스토어 목록과 동일한 언어로 패키지를 업로드 한 경우 해당 패키지에 사용 된 이름이 선택 됩니다. 앱을 이미 게시 한 후 [이름을 변경](manage-app-names.md#rename-an-app-that-has-already-been-published) 해야 하는 경우 새 이름을 사용 하는 패키지를 업로드 한 후 새 제출을 만들 때 다른 예약 된 이름을 선택할 수 있습니다.

작업 중인 언어에 대 한 패키지를 업로드 하지 않은 경우 두 개 이상의 이름을 예약한 경우 이름을 가져올 해당 언어에 연결 된 패키지가 없으므로 예약 된 앱 이름 중 하나를 선택 해야 합니다.

> [!NOTE]
> 선택한 **제품 이름은** 작업 중인 언어의 스토어 목록에만 적용 됩니다. 고객이 앱을 설치할 때 표시 되는 이름에는 영향을 주지 않습니다. 이 이름은 설치 된 패키지의 매니페스트에서 가져옵니다. 혼동을 피하려면 각 언어의 패키지 및 매장 목록에서 같은 이름을 사용 하는 것이 좋습니다.

## <a name="description"></a>설명

설명 필드를 통해 고객에 게 앱의 용도를 알려줄 수 있습니다. 이 필드는 필수 이며 일반 텍스트의 최대 1만 자까지 허용 됩니다.

설명의 작성에 대 한 몇 가지 팁 [은 유용한 앱 설명 작성](write-a-great-app-description.md)을 참조 하세요.

<a name="release-notes"></a>

## <a name="whats-new-in-this-version"></a>이 버전의 새로운 기능

앱을 처음으로 제출 하는 경우이 필드를 비워 둡니다. 기존 앱에 대 한 업데이트의 경우 최신 버전에서 변경 된 내용을 고객에 게 알릴 수 있습니다. 이 필드의 문자 제한은 1500입니다. (이전에는이 필드를 **릴리스 정보**라고 했습니다.)

## <a name="product-features"></a>제품 기능

앱의 주요 기능에 대 한 간단한 요약입니다. 이러한 항목은 **설명**뿐만 아니라 앱의 상점 목록에 있는 **기능** 섹션에 있는 글머리 기호 목록으로 고객에 게 표시 됩니다. 기능 당 몇 개의 단어 (및 200 자이 하)만 포함 하 여 이러한 내용을 간단 하 게 유지 합니다. 최대 20 개의 기능을 포함할 수 있습니다.

> [!NOTE]
> 이러한 기능은 스토어 목록에 글머리 기호를 표시 하므로 고유한 글머리 기호를 추가 하지 않습니다.

## <a name="screenshots"></a>스크린샷

앱을 제출 하려면 하나의 스크린샷에 필요 합니다. 앱이 지 원하는 각 장치 유형에 대해 네 개 이상의 스크린샷을 제공 하 여 앱이 장치 유형에 서 어떻게 보이는지 확인할 수 있도록 하는 것이 좋습니다.

자세한 내용은 [앱 스크린샷 및 이미지](app-screenshots-and-images.md#screenshots)를 참조 하세요.

## <a name="store-logos"></a>스토어 로고

매장 로고는 앱이 고객에 게 표시 되는 방식을 개선 하기 위해 업로드할 수 있는 선택적 이미지입니다. 필요에 따라 스토어에서 앱 패키지의 로고 이미지를 사용 하도록 허용 하는 대신, 여기에서 업로드 하는 이미지만 앱의 스토어에서 사용 하도록 지정할 수도 있습니다 (Xbox 포함).

> [!IMPORTANT]
> 앱에서 Xbox를 지 원하는 경우 또는 Windows Phone 8.1 이전 버전을 지 원하는 경우에는 특정 이미지를 제공 해야 목록이 스토어에 제대로 표시 됩니다.

자세한 내용은 [매장 로고](app-screenshots-and-images.md#store-logos)를 참조 하세요.

## <a name="trailers-and-additional-assets"></a>트레일러 및 추가 자산

비디오 트레일러 및 판촉 이미지를 포함 하 여 제품에 대 한 추가 자산을 제출할 수 있습니다. 이러한 옵션은 모두 선택 사항 이지만 최대한 많은 항목을 업로드 하는 것이 좋습니다. 이러한 이미지를 통해 고객이 제품을 더 잘 이해할 수 있도록 하 고 더 enticing 목록을 만들 수 있습니다.

자세한 내용은 [트레일러 및 추가 자산](app-screenshots-and-images.md#trailers-and-additional-assets)을 참조 하세요.

<a name="supplemental-information"></a>

## <a name="supplemental-fields"></a>추가 필드

이 섹션의 필드는 모두 선택 사항입니다. 아래 정보를 검토 하 여이 정보를 제공 하면 제출에 적합 한지 확인 하세요. 특히 대부분의 서브 미션에는 **간단한 설명을 입력** 하는 것이 좋습니다. 다른 필드는 다양 한 시나리오에서 제품에 대 한 최적의 환경을 제공 하는 데 도움이 될 수 있습니다.

### <a name="short-title"></a>짧은 제목

제품 이름의 약식 버전입니다. 제공 되는 경우 제품의 전체 제목 대신 Xbox One (설치 중, 업적 등)의 다양 한 위치에이 짧은 이름이 표시 될 수 있습니다.

이 필드의 문자 제한은 50입니다.

### <a name="sort-title"></a>제목 정렬

제품의 사전순 또는 철자가 다른 방식으로 표시 될 수 있는 경우 여기에 다른 버전을 입력할 수 있습니다. 이렇게 하면 고객이 검색 하는 동안에서 해당 버전을 입력 하는 경우 더 빠르게 제품을 찾을 수 있습니다.

이 필드의 문자 제한은 255입니다.

### <a name="voice-title"></a>음성 제목

제품에 대 한 대체 이름 (제공 된 경우)은 Kinect 또는 헤드셋을 사용할 때 Xbox One의 오디오 환경에서 사용할 수 있습니다.

이 필드의 문자 제한은 255입니다.

### <a name="short-description"></a>간단한 설명

제품의 상점 목록 위쪽에서 사용할 수 있는 짧은 catchy 설명입니다. 제공 하지 않으면 긴 [설명](#description) 의 첫 번째 단락 (최대 500 자)이 대신 사용 됩니다. 이 텍스트 아래에 설명도 표시 되므로 상점 목록이 반복 되지 않도록 다른 텍스트를 사용 하 여 간단한 설명을 제공 하는 것이 좋습니다.

게임의 경우 간단한 설명이 Xbox One의 게임 허브 정보 섹션에도 표시 될 수 있습니다.

최상의 결과를 위해 270 문자 아래에서 간단한 설명을 유지 하세요. 이 필드에는 1000 자 제한이 있지만 일부 보기에서는 첫 번째 270 문자만 표시 됩니다 (간단한 설명의 나머지 부분을 볼 수 있는 링크를 사용 하 여).

### <a name="additional-system-requirements"></a>추가 시스템 요구 사항

필요한 경우 앱이 제대로 작동 하는 데 필요한 하드웨어 구성을 설명할 수 있습니다 ( [앱 속성](enter-app-properties.md#system-requirements)의 **시스템 요구 사항** 섹션에 제공 된 정보 외). 이는 앱에 모든 컴퓨터에서 사용 하지 못할 수 있는 하드웨어가 필요한 경우에 특히 중요 합니다. 예를 들어 앱이 3D 프린터나 마이크로 컨트롤러와 같은 외부 USB 하드웨어 에서만 제대로 작동 하는 경우 여기에 입력 하는 것이 좋습니다. 입력 한 정보는 제품의 속성 페이지에 표시 되는 요구 사항과 함께 Windows 10, 버전 1607 이상 (Xbox 포함)에 앱 스토어를 표시 하는 고객에 게 표시 됩니다.

**최소 하드웨어** 및 **권장 하드웨어**에 대해 최대 11 개의 항목을 입력할 수 있습니다. 이러한 항목은 고객에 게 매장 목록에서 글머리 기호 목록으로 표시 됩니다. 항목 마다 몇 개의 단어 (및 200 자이 하)만 포함 하 여이 내용을 짧게 유지 합니다.

> [!NOTE]
> 추가 시스템 요구 사항은 스토어 목록에 글머리 기호를 표시 하므로 사용자 고유의 글머리 기호를 추가 하지 않습니다.

<a name="shared-fields"></a>

## <a name="additional-information"></a>추가 정보

아래에 설명 된 항목은 고객이 제품을 검색 하 고 이해 하는 데 도움이 됩니다. (이 섹션은 이전에 **공유 필드**라고 했습니다.)

### <a name="search-terms"></a>검색어

검색 용어 (이전에는 키워드)는 고객에 게 표시 되지 않는 단일 단어 또는 짧은 구 이지만 고객이 해당 용어를 사용 하 여 검색할 때 스토어에서 앱을 검색할 수 있도록 하는 데 도움이 될 수 있습니다. 최대 30 자까지 최대 7 개의 검색 용어를 포함할 수 있으며 모든 검색어에서 21 개 이하의 개별 단어를 사용할 수 있습니다.

검색 용어를 추가할 때 고객이 앱의 이름에 포함 되지 않은 경우와 같은 앱을 검색할 때 사용할 수 있는 단어를 생각해 보십시오. 실제로 앱과 관련이 없는 검색 용어를 사용 하지 않도록 해야 합니다.

### <a name="copyright-and-trademark-info"></a>저작권 및 상표 정보

추가 저작권 및/또는 상표 정보를 제공 하려면 여기에 입력 합니다. 이 필드의 문자 제한은 200입니다.

### <a name="additional-license-terms"></a>추가 사용 조건

앱 [개발자 규약](https://docs.microsoft.com/legal/windows/agreements/app-developer-agreement)에서 연결 된 **표준 응용 프로그램 사용 조건** 에 따라 고객에 게 앱을 사용 하려면이 필드를 비워 둡니다.

사용 조건이 **표준 응용 프로그램 사용 조건과**다른 경우 여기에 입력 합니다.

이 필드에 단일 URL을 입력 하면 추가 사용 조건에 대 한 읽기를 위해 클릭할 수 있는 링크로 고객에 게 표시 됩니다. 이는 추가 사용 조건이 매우 길거나 추가 사용 조건에 클릭 가능한 링크나 서식 지정을 포함 하려는 경우에 유용 합니다.

이 필드에는 최대 1만 자의 텍스트를 입력할 수도 있습니다. 이렇게 하면 고객에 게 일반 텍스트로 표시 되는 이러한 추가 사용 조건이 표시 됩니다.

### <a name="developed-by"></a>개발한 사람

앱의 스토어 목록에서 **개발한 사람** 필드를 포함 하려면 여기에 텍스트를 입력 하세요. ( **게시** 한 사람 필드에는 사용자가 **개발한 사람** 필드에 대 한 값을 제공 하는지 여부에 관계 없이 계정에 연결 된 게시자 표시 이름이 나열 됩니다.)

이 필드의 문자 제한은 255입니다.
