# <a name="acquisitions-report"></a>구입 보고서


**획득** 보고서 [파트너 센터](https://partner.microsoft.com/dashboard) 획득 한 사람 및 인구 통계와 함께 앱 설치를 확인할 수 있습니다 및 플랫폼 세부 정보 및 방법에 대 한 정보를 보여 줍니다 (포함 하는 Windows 10에는 고객 Xbox) 앱의 목록에 도착 한 것입니다. 마지막 시간 또는/72 두 시간 기간에 대 한 거의 실시간 취득 데이터도 볼 수 있습니다. 

파트너 센터에서이 데이터를 볼 수 있습니다 또는 [보고서 다운로드](download-analytic-reports.md) 오프 라인으로 볼 수 있습니다. 또는 [분석 REST API](../monetize/access-analytics-data-using-windows-store-services.md)를 사용하여 프로그래밍 방식으로 이 데이터를 검색할 수 있습니다.

이 보고서에서 **구입**은 새 고객이 앱에 대한 라이선스를 획득했음을 의미합니다(금액을 청구했거나 무료로 제공했는지와 관계없음). **설치**는 Windows 10 장치에 앱이 설치 중임을 의미합니다.

> [!IMPORTANT]
> 합니다 **합병** 보고서 환불, 반전, 거절 하는 방법에 대 한 데이터를 포함 하지 않습니다. 사용자 앱 수익을 계산 하려면 방문 [지급액 요약](payout-summary.md)합니다. **예약됨** 섹션에서 **예약된 거래 다운로드** 링크를 클릭합니다.
>
> 페이지 보기 데이터(아래 설명된)를 제외하고 이 보고서에는 Microsoft 계정에 로그인 하지 않고 앱을 구입하는 고객과 관련된 데이터는 포함되어 있지 않습니다.


## <a name="apply-filters"></a>필터 적용

페이지 위쪽에서 데이터가 표시되는 기간을 선택할 수 있습니다. 기본 설정은 **30D**(30일)이지만, 3개월, 6개월 또는 12개월 동안 데이터를 표시하거나 사용자가 지정한 데이터 범위에서 데이터를 표시하도록 선택할 수 있습니다. 거의 실시간으로 데이터 옵션을 모두 표시 됩니다 (제외 **누적 앱** 데이터). **1 시간** 및 **72 시간** 기간에만 적용 합니다 **매일 앱** 탭을 **획득** 차트 및는  **획득** 탭의 **시장** 차트입니다. 

또한 **필터**를 확장하여 시장 및/또는 장치 유형을 기준으로 이 페이지의 모든 데이터를 필터링할 수 있습니다.

-   **시장**: 기본 필터는 **모든 시장**, 있지만 하나 이상의 시장에서 구매 데이터를 제한할 수 있습니다.
-   **장치 유형**: 기본 설정은 **모든 장치**합니다. 특정 디바이스 유형(PC, 콘솔, 태블릿 등)에서만 구입에 대한 데이터를 표시하려는 경우 여기에서 특정한 디바이스를 선택할 수 있습니다.

아래에 나열된 모든 차트의 정보는 선택한 날짜 범위와 모든 필터를 반영합니다. 또한 일부 섹션에서는 추가 필터를 적용할 수 있습니다.


## <a name="acquisitions"></a>구입

**구입** 차트는 선택한 기간 동안 일일 또는 주간 구입(앱에 대한 라이선스를 획득하고 있는 신규 사용자) 수를 보여줍니다. (**필터 적용**을 사용하여 더 오랜 시간 동안 데이터를 표시하면 구입 데이터가 주 단위로 그룹화됩니다.) 이 차트에는 유효한 Microsoft 계정으로 로그인 하는 고객을 수행한 획득만 포함 됩니다. 

기본적으로 표시 합니다 **앱 매일** 거의 실시간으로 데이터를 포함 하는 보기입니다. 또한 **앱 누적 수**를 선택하여 앱에 대한 총 구입 수를 볼 수도 있습니다. 여기에는 앱이 처음 게시될 때부터 시작하여 총 누적 구입 수가 표시됩니다.

**총 판매액** (2016 년 10 월-존재)에서 앱도 사용할 수 있습니다이 차트,에 대 한 (USD)에서 앱 판매에서 획득 한 총 금액을 표시 합니다. 참고가이 크기는 모든 환불, 반전, 비용 정산 등에 대해 설명 하지 않습니다.

필요에 따라 구입이 클라이언트 또는 웹 기반 Microsoft Store에서 시작되었는지 여부와 OS 버전에 따라 결과를 필터링할 수 있습니다.

> [!NOTE]
> 또한 [분석 REST API](../monetize/access-analytics-data-using-windows-store-services.md)에서 [앱 구입 데이터 가져오기](../monetize/get-app-acquisitions.md) 메서드를 사용하여 프로그래밍 방식으로 이 데이터를 검색할 수 있습니다.

에 **앱 매일** 보기는 **30 D** 기간을 선택 하면 원형 표식 표시 될 수 있습니다. 이러한 크게 증가 나타내거나 파악 해야 하는 것이 생각 하는 지정 된 값은 감소 합니다. 원이 표시 되는 날짜는 상당한 증가 또는 감소 하는 이전 주에 비해 감지는 주 끝을 나타냅니다. 변경 된 내용에 대 한 자세한 내용은 원 위로 가져갑니다.  

> [!TIP]
> 지난 30 일 동안 중요 한 변경 내용과 관련 된 자세한 정보를 볼 수 있습니다 합니다 [Insights 보고서](insights-report.md)합니다.

## <a name="installs"></a>설치

**설치** 차트에서 선택한 기간 동안 고객이 Windows 10 장치(Xbox One 본체 포함)에 앱을 몇 번 설치했는지 확인할 수 있습니다. 일별 또는 주별(선택한 기간에 따라 다름)로 설치 수를 표시하는 차트와 함께 총계가 표시됩니다. 선택에 따라 특정 패키지 버전별로 결과를 필터링할 수 있습니다.

설치 총계에는
-   **여러 Windows 10 장치에 설치합니다.** 예를 들어 같은 고객이 Windows 10 PC 두 대와 Xbox One 본체 한 대에 앱을 설치한 경우 총 3회 설치로 계산됩니다.
-   **다시 설치합니다.** 예를 들어 고객이 오늘 앱을 설치하고 내일 앱을 제거한 후 다음 달에 이 앱을 다시 설치하면 2회 설치로 계산됩니다.

설치 총계에 다음은 포함되지 않습니다.
-   **비 Windows 10 장치에 설치합니다.** 앱에서 Windows 8.x 또는 Windows Phone 8.x 같은 이전 OS 버전을 지원하는 경우 해당 장치에서의 설치 수는 계산하지 않습니다.
-   **제거합니다.** 고객이 장치에 앱을 제거해도 설치 총계에서 해당 값이 빠지지 않습니다.
-   **업데이트합니다.** 예를 들어 고객이 오늘 앱을 설치하고 일주일 후에 앱 업데이트를 설치해도 2회가 아닌 1회 설치로 계산됩니다.
-   **이 사전 설치 합니다.** 앱이 사전 설치된 장치를 고객이 구입하는 경우에는 1회 설치로 계산되지 않습니다.
-   **시스템에서 시작한를 설치합니다.** 어떤 이유에서든 Windows가 앱을 자동으로 설치해도 1회 설치로 계산되지 않습니다.

> [!NOTE]
> 또한 [분석 REST API](../monetize/access-analytics-data-using-windows-store-services.md)에서 [앱 설치 데이터 가져오기](../monetize/get-app-installs.md) 메서드를 사용하여 프로그래밍 방식으로 이 데이터를 검색할 수 있습니다.

## <a name="acquisition-funnel"></a>취득 깔때기

**취득 깔때기**는 변환율과 함께 Microsoft Store 페이지 조회부터 앱 사용에 이르기까지 취득 깔때기의 각 단계를 완료한 고객의 수를 보여줍니다. 이 데이터는 구입, 설치 또는 사용 수를 늘리기 위해 더 많은 투자가 필요한 영역을 식별하는 데 도움이 됩니다.

> [!IMPORTANT]
> **취득 깔때기**는 Windows 10(Xbox 포함)의 고객들에게만 지난 90일 동안의 데이터를 보여줍니다.

깔때기의 단계는 아래와 같습니다.

- **페이지 보기**: 이 숫자를 나타냅니다 앱의 스토어 목록에 없는 Microsoft 계정으로 로그인 하는 사람들을 비롯 한 총 보기. 이 데이터에는 Microsoft에 이 정보를 제공하는 것을 거부한 고객의 데이터는 포함되지 않습니다.
- **획득**: 해당 저장소 목록을 표시 하는 48 시간 이내 (Microsoft 계정으로 로그인) 하는 경우 앱에 라이선스를 받은 새로운 고객의 수입니다.
- **설치**: 이 가져오면 앱을 설치 하는 고객의 수입니다.
- **사용 현황**: 설치한 후 앱을 사용 하는 고객의 수입니다.

선택에 따라 성별 및/또는 연령 그룹을 비롯해 사용자 지정 캠페인 ID에 따라 결과를 필터링할 수 있습니다.

> [!NOTE]
> 또한 [분석 REST API](../monetize/access-analytics-data-using-windows-store-services.md)에서  [앱 취득 깔때기 데이터 가져오기](../monetize/get-acquisition-funnel-data.md) 메서드를 사용하여 프로그래밍 방식으로 이 데이터를 검색할 수 있습니다.

## <a name="markets"></a>지역/국가

**시장** 차트는 앱이 출시된 각 시장에서 선택한 기간 동안의 구입 또는 설치 총계를 보여줍니다. **구입** 또는 **설치** 데이터의 표시 여부를 선택할 수 있습니다.

시각적인 **지도** 형식으로 이 데이터를 보거나 설정을 전환하여 **표** 형식으로 볼 수 있습니다. 표 형식은 한 번에 5개의 시장을 알파벳 순서나 구입 또는 설치 최고/최저 수에 따라 정렬해서 표시합니다. 또한 이 데이터를 다운로드하여 모든 시장에 대한 정보를 한꺼번에 볼 수 있습니다.


## <a name="customer-demographic"></a>고객 인구 통계

**고객 인구 통계** 차트는 앱을 구입한 사람에 대한 인구 통계 정보를 보여 줍니다. 선택한 기간에 구매한 항목 수를 특정 연령 그룹의 사람별 및 성별별로 확인할 수 있습니다.

> [!NOTE]
> 일부 고객은 이 정보를 공유하지 않도록 선택했습니다. 연령 그룹 또는 성별을 판별할 수 없으면 구입은 **알 수 없음**으로 분류됩니다.

 

## <a name="app-page-views-and-conversions-by-channel"></a>채널별 앱 페이지 보기 및 변환

합니다 **앱 페이지 보기 및 채널에 의해 변환** 차트를 사용 하면 Windows 10에서 고객에 게 선택한 기간 동안 앱의 목록에 도착 하는 방법을 참조 하세요.

이 차트에서 *채널*은 고객이 앱의 목록 페이지에 방문한 방법(예: Microsoft Store 탐색 및 검색, 외부 웹 사이트의 링크, 사용자 지정 캠페인 중 하나의 링크 등)을 의미합니다. 다음과 같은 채널 유형이 포함됩니다.

-   **저장소 트래픽:** 고객 검색 되었거나 앱의 목록이 표시 된 저장소 내에서 검색 됩니다.
-   **사용자 지정 캠페인:** 고객이 사용 하는 링크 뒤에 [사용자 지정 캠페인 ID](create-a-custom-app-promotion-campaign.md)합니다.
-   **기타:** 고객 (모든 사용자 지정 캠페인 ID)가 없는 외부 링크를 웹 사이트에서 앱의 목록에 따르거나 고객 앱의 목록에 검색 엔진에서 링크를 따라 합니다.

A *페이지 보기* 웹 기반 상점을 통해 또는 내에서 고객이 앱의 스토어 목록 페이지를 볼는 Windows 10에서 스토어 앱입니다. 여기에는 Microsoft 계정으로 로그인 하지 않은 사람에 의한 보기도 포함이 됩니다. 일부 고객들은 Microsoft에 이 정보를 제공하기를 거부했습니다.

*변환*은 (Microsoft 계정으로 로그인한) 고객이 앱에 대한 라이선스를 새로 획득했음을 의미합니다(금액 청구 또는 무료 제공 여부와는 관계 없음).

페이지 보기 및 변환 수는 고유한 고객의 수가 아닙니다. 변환율 정보는 [취득 깔때기](#acquisition-funnel) 차트를 참조하십시오.

> [!NOTE]
> 고객이 사용자가 만들지 않은 사용자 지정 캠페인을 클릭하여 앱 목록에 도달할 수 있습니다. 고객이 Microsoft Store에 처음 들어올 때 사용한 캠페인 ID로 세션 내의 모든 페이지 보기가 스탬핑됩니다. 그런 다음 24 시간 이내에 모든 구입에서 해당 캠페인 ID로 특성 변환이 됩니다. 따라서 캠페인 ID에 대한 변환 총계보다 변환 총계가 더 높게 나타나고 제로 페이지 보기가 있는 변환 또는 추가 기능 변환이 있을 수 있습니다.

> [!NOTE]
> 또한 [분석 REST API](../monetize/access-analytics-data-using-windows-store-services.md)에서 [채널별 앱 변환 데이터 가져오기](../monetize/get-app-conversions-by-channel.md)  메서드를 사용하여 프로그래밍 방식으로 이 데이터를 검색할 수 있습니다.

## <a name="app-page-views-and-conversions-by-campaign-id"></a>캠페인 ID별 앱 페이지 보기 및 변환

**캠페인 ID별 앱 페이지 보기 및 변환** 차트에서 위에서 설명한 대로 [사용자 지정 홍보 캠페인](create-a-custom-app-promotion-campaign.md) 각각에 대해 변환 및 페이지 보기를 추적할 수 있습니다. 인기 캠페인 ID가 표시되고 이 필터를 사용하여 특정 캠페인 ID를 제외 또는 포함시킬 수 있습니다.

## <a name="total-campaign-conversions"></a>총 캠페인 변환

**총 캠페인 변환** 차트는 선택한 기간 동안 사용자 지정 캠페인의 총 앱 및 추가 기능 변환 수를 보여 줍니다.
