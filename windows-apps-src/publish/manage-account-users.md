# 계정 사용자 관리


Azure Active Directory를 사용하여 개발자 센터 계정에 사용자를 추가할 수 있습니다. 각 사용자는 계정에 대해 특정 사용 권한 집합을 제공하는 역할이 할당됩니다. 또한 사용자 그룹 또는 Azure AD 응용 프로그램에 역할을 할당할 수 있습니다.

> **중요** 계정 사용자를 추가하고 관리하려면 먼저 개발자 센터 계정을 조직의 Azure Active Directory와 연결해야 합니다. 이렇게 하려면 [전역 관리자](http://go.microsoft.com/fwlink/?LinkId=746654) 계정으로 Azure AD에 로그인해야 합니다. 이 연결을 설정하면 제거할 때 고객 지원에 반드시 문의해야 합니다.

 

## 개발자 센터 계정을 조직의 Azure Active Directory와 연결합니다.

Windows 개발자 센터는 다중 사용자 관리 및 역할 할당에 대해 Azure Active Directory를 활용합니다. 조직에서 이미 Office 365 또는 Microsoft의 다른 비즈니스 서비스를 사용하는 경우 이미 Azure AD가 있습니다. 아니면 추가 요금 없이 개발자 센터 내에서 Azure 새 광고를 만들 수 있습니다.

Azure AD에는 단 하나의 개발자 센터 계정만 연결할 수 있습니다. 마찬가지로 개발자 센터 계정은 단 하나의 Azure AD만 연결할 수 있습니다.

> **참고** 조직의 Azure AD의 일부인 사용자만 개발자 센터 계정에 추가할 수 있습니다(또는 새 Azure AD 계정을 만든 경우). 사용자의 개인 Microsoft 계정으로 사용자를 개발자 센터 계정에 추가할 수는 없습니다.

### 개발자 센터 계정을 조직의 기존 Azure AD와 연결

조직에서 이미 Azure AD를 사용하는 경우 개발자 센터 계정에 연결하려면 다음 단계를 따릅니다.

1.  **계정 설정**으로 이동하여 **사용자 관리**를 클릭합니다.
2.  **Azure AD와 개발자 센터 계정 연결** 단추를 클릭합니다.
3.  Azure AD 계정에 로그인합니다. 연결을 설정하려면 이 계정에 [전역 관리자](http://go.microsoft.com/fwlink/?LinkId=746654) 권한이 있어야 합니다.
4.  Azure AD 계정에 대한 조직 및 도메인 이름을 검토합니다. 연결을 완료하려면 **확인**을 클릭합니다.
5.  연결되면 아래 섹션에서 설명한 대로 계정의 **사용자 관리** 페이지에서 계정 사용자를 추가하고 관리할 준비가 됩니다.

### 개발자 센터 계정과 연결하는 새로운 Azure AD 만들기

개발자 센터 계정과 연결할 새 Azure AD를 설정해야 하는 경우 다음 단계를 따릅니다.

1.  **계정 설정**으로 이동하여 **사용자 관리**를 클릭합니다.
2.  **Create new Azure AD**(새 Azure AD 만들기) 단추를 클릭합니다.
3.  새 Azure AD에 대한 디렉터리 정보를 입력합니다.
 - **도메인 이름**: Azure AD 도메인에 사용할 고유한 이름으로 ".onmicrosoft.com"과 함께 사용합니다. 예를 들어 "example"을 입력한 경우 Azure AD 도메인은 "example.onmicrosoft.com"이 됩니다.
 - **연락처 메일**: 필요한 경우 계정에 대해 문의할 수 있는 메일 주소입니다.
 - **전역 관리자 사용자 계정 정보**: 새 관리자 계정에 사용할 성, 이름, 사용자 이름 및 암호입니다.
4.  **만들기**를 클릭하여 새 도메인 및 계정 정보를 확인합니다.
5.  새 Azure AD 전역 관리자 사용자 이름과 암호를 사용하여 로그인하고 아래 섹션에 설명된 대로 계정의 **사용자 관리** 페이지에서 계정 사용자를 추가하고 관리하기 시작합니다.


> **중요** Azure AD와 개발자 센터 계정을 연결하고 나면 계정 사용자를 추가하고 관리하기 위해 Azure AD 전역 관리자 계정을 사용하여(개인 Microsoft 계정은 안 됨) 개발자 센터에 항상 로그인해야 합니다.

## 계정 사용자, 그룹 및 Azure AD 응용 프로그램 추가 및 관리


연결을 설정했으면 계정에 사용자, 그룹 및 Azure AD 응용 프로그램을 추가할 수 있습니다. 또한 역할 변경, 계정 세부 사항 편집 또는 사용자 제거를 수행할 수 있습니다.

> **참고** 조직에서 [디렉터리 통합](http://go.microsoft.com/fwlink/p/?LinkID=724033)을 사용하여 온-프레미스 디렉터리 서비스와 Azure AD를 동기화하면 개발자 센터에서 새 사용자, 그룹 또는 Azure AD 응용 프로그램을 만들 수 없습니다. 개발자 센터에서 새 사용자, 그룹 또는 Azure AD 응용 프로그램을 보고 추가할 수 있으려면 귀하 또는 온-프레미스 디렉터리의 다른 관리자가 온-프레미스 디렉터리에서 직접 만들어야 합니다.

사용자를 관리할 때 다음을 기억해야 합니다.

-   모든 개발자 센터 사용자는 조직의 Azure AD에 활성 계정이 있어야 합니다.
-   개발자 센터에서 **새** 사용자 또는 그룹을 만들면 조직의 Azure AD에도 추가됩니다.
-   개발자 센터의 사용자 또는 그룹의 이름을 변경하면 조직의 Azure AD에서도 변경됩니다.
-   사용자(그룹 및 Azure AD 응용 프로그램 포함)는 할당된 역할과 연결된 사용 권한으로 전체 개발자 센터 계정에 액세스할 수 있습니다. 특정 앱 및/또는 추가 기능만 사용할 수 있도록 사용자의 액세스를 제한할 수 없습니다.
-   여러 역할을 선택하여 사용자, 그룹 또는 Azure AD 응용 프로그램이 둘 이상의 역할 기능에 액세스하도록 할 수 있습니다.
-   특정 역할이 있는 사용자는 다른 역할이 있는 그룹의 일부가 될 수도 있습니다. 이런 경우 사용자는 두 역할 모두와 연결된 기능에 액세스할 수 있습니다.

### 역할 및 권한

계정에 추가하는 각 사용자, 그룹 또는 Azure AD 응용 프로그램에는 다음 역할 중 하나 이상이 할당되어야 합니다. 각 역할에는 계정 내에서 특정 기능을 수행하기 위해 특정 사용 권한 집합이 있습니다.

> **참고** 계정 소유자는 Microsoft 계정에 처음으로 해당 계정을 만든 사용자입니다(Azure AD를 통해 추가된 사용자가 아님). 앱 삭제, 모든 계정 사용자 생성 및 편집, 모든 재무 및 계정 설정 변경 기능을 비롯하여 계정에 대한 전체 액세스 권한은 이 계정 소유자에게만 있습니다. 

| 역할                 | 설명              |
|----------------------|--------------------------|
| 관리자              | 세금 및 지급액 설정 변경을 제외하고 계정에 대한 모든 권한이 있습니다. 여기에는 개발자 센터의 사용자를 관리하는 권한이 포함되지만 사용자를 생성하고 삭제하는 기능은 Azure AD에서 계정의 사용 권한에 따라 다릅니다. 즉, 사용자에게 관리자 역할이 할당되었지만 조직의 Azure AD에서 관리자 권한은 없는 경우 새 사용자를 만들거나 디렉터리에서 사용자를 삭제할 수 없습니다(사용자의 개발자 센터 역할은 변경할 수 있음). |
| 개발자            | 패키지를 업로드하고 앱 및 추가 기능을 제출할 수 있으며 원격 분석 세부 사항에 대한 [사용 보고서](usage-report.md)를 볼 수 있습니다. 재무 정보 또는 계정 설정을 볼 수 없습니다.                                                                                                                                                                                                                                                                                                                     |
| 비즈니스 기여자 | 재무 정보에 액세스할 수 있고 가격 책정 세부 사항을 설정할 수 있습니다. 새 앱 및 추가 기능을 만들거나 제출할 수 없고 계정 설정을 변경할 수 없습니다.                                                                                                                                                                                                                                                                                                                                                              |
| 재무 기여자  | [지급 보고서](payout-summary.md)를 볼 수 있습니다. 앱, 추가 기능 또는 계정 설정을 변경할 수 없습니다.                                                                                                                                                                                                                                                                                                                                                                                 |
| 마케팅 담당자             | [고객 리뷰에 응답](respond-to-customer-reviews.md)할 수 있고 비재무 [분석 보고서](analytics.md)를 볼 수 있습니다. 앱, 추가 기능 또는 계정 설정을 변경할 수 없습니다.                                                                                                                                                                                                                                                                                                            |

### 계정 사용자 추가 및 관리

개발자 센터 계정에 추가하고 역할을 할당하려는 사용자를 식별하려면 **사용자 추가**를 클릭합니다.

조직 디렉터리의 하나 이상의 사용자를 개발자 센터 계정에 추가할 수 있습니다. 둘 이상의 사용자를 동시에 추가하는 경우 동일한 역할을 할당해야 합니다. 사용자를 추가하지만 서로 다른 역할을 할당하려는 경우 각 역할에 대해 아래 단계를 반복합니다.

**조직 디렉터리의 사용자 추가**

1.  **사용자 관리** 페이지에서 **사용자 추가**를 클릭합니다.
2.  표시되는 목록에서 하나 이상의 사용자를 선택합니다. 검색 상자를 사용하여 특정 사용자를 검색할 수 있습니다.
3.  사용자 선택이 완료되면 **선택된 항목 추가**를 클릭합니다.
4.  **역할** 섹션에서 하나 이상의 역할을 선택하여 이 사용자 집합에 할당합니다.
5.  **저장**을 클릭합니다.

새 사용자 계정에 개발자 센터 액세스 권한을 부여하려는 경우 **사용자 관리** 섹션에서 새 사용자 계정을 만들 수 있습니다. 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 새 계정이 만들어집니다.

**새 사용자 계정 만들기**

1.  **사용자 관리** 페이지에서 **사용자 추가**를 클릭합니다.
2.  다음 페이지에서 **새 사용자**를 클릭합니다.
3.  새 사용자의 이름, 성 및 사용자 이름을 입력합니다.
4.  **역할** 섹션에서 하나 이상의 역할을 선택하여 새 사용자에게 할당합니다.
5.  **그룹 구성원** 섹션에서 새 사용자가 속하게 될 그룹을 선택합니다.
6.  **저장**을 클릭합니다.
7.  확인 페이지에 임시 암호를 비롯하여 새 사용자의 로그인 정보가 표시됩니다. 이 페이지를 나간 후에는 임시 암호에 액세스할 수 없으므로 이 정보를 메모하고 새 사용자에게 제공해야 합니다.

**사용자 관리** 섹션에서 개발자 센터 계정에 추가한 사용자 계정을 변경할 수 있습니다. 사용자 이름 또는 그룹 구성원에 대한 변경 사항은 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 반영됩니다. 사용자 역할에 대한 변경 사항은 개발자 센터 액세스에만 영향을 줍니다.

**사용자 계정 편집**

1.  **사용자 관리** 페이지에서 편집하려는 사용자 계정의 이름을 클릭합니다.
2.  다음과 같이 변경할 수 있습니다.
    -   사용자의 이름, 성 또는 사용자 이름을 편집합니다. 이러한 변경 사항은 조직의 디렉터리에 반영된다는 점을 기억하세요.
    -   **역할** 섹션에서 이 사용자에 대해 추가 또는 제거하려는 역할을 선택 또는 선택 취소합니다.
    -   **그룹 구성원** 섹션에서 사용자가 가입하거나 제거되어야 할 그룹을 선택 또는 선택 취소합니다. 이러한 변경 사항은 조직의 디렉터리에 반영된다는 점을 기억하세요.

3.  **저장**을 클릭합니다.

개발자 센터 계정에 추가한 사용자 계정용 암호를 변경해야 하는 경우 **사용자 관리** 섹션에서 수행할 수 있습니다. 개발자 센터 액세스용 암호뿐만 아니라 사용자의 디렉터리 암호도 변경됩니다.

**사용자의 디렉터리 암호 변경**

1.  **사용자 관리** 페이지에서 편집하려는 사용자 계정의 이름을 클릭합니다.
2.  페이지 맨 아래에서 **암호 재설정** 단추를 클릭합니다.
3.  임시 암호를 비롯하여 사용자에 대한 로그인 정보를 표시하는 확인 페이지가 나타납니다.
  > **중요** 이 페이지를 나간 후에는 임시 암호에 액세스할 수 없으므로 이 정보를 인쇄하거나 복사하고 사용자에게 제공해야 합니다.

### 그룹 추가 및 관리

조직 디렉터리의 그룹을 개발자 센터 계정에 추가하는 경우 해당 그룹의 멤버인 모든 사용자는 그룹의 할당된 역할과 연결된 사용 권한을 사용하여 액세스할 수 있습니다. 그룹의 모든 변경 사항(이름 및 구성원 포함)이 조직의 디렉터리에 반영된다는 점을 기억하세요.

둘 이상의 그룹을 동시에 추가하는 경우 동일한 역할을 할당해야 합니다. 그룹을 추가하지만 서로 다른 역할을 할당하려는 경우 각 역할에 대해 아래 단계를 반복합니다.

**조직 디렉터리의 그룹 추가**

1.  **사용자 관리** 페이지에서 **그룹 추가**를 클릭합니다.
2.  표시되는 목록에서 하나 이상의 그룹을 선택합니다. 검색 상자를 사용하여 특정 그룹을 검색할 수 있습니다.
3.  그룹 선택이 완료되면 **선택된 항목 추가**를 클릭합니다.
4.  **역할** 섹션에서 하나 이상의 역할을 선택하여 이 그룹 집합에 할당합니다.
5.  **저장**을 클릭합니다.

새 그룹에 개발자 센터 액세스 권한을 부여하려는 경우 **사용자 관리** 섹션에서 새 그룹을 만들 수 있습니다. 새 그룹이 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 만들어집니다.

**새 그룹 계정 만들기**

1.  **사용자 관리** 페이지에서 **그룹 추가**를 클릭합니다.
2.  다음 페이지에서 **새 그룹**을 클릭합니다.
3.  새 그룹에 대한 표시 이름을 입력합니다.
4.  새 그룹에 할당할 하나 이상의 역할을 선택합니다. 그룹의 모든 멤버가 해당 역할과 연결된 사용 권한을 사용하여 개발자 센터 계정에 액세스할 수 있습니다.
5.  표시되는 목록에서 하나 이상의 사용자를 선택합니다. 검색 상자를 사용하여 특정 사용자를 검색할 수 있습니다.
6.  사용자 선택이 완료되면 **선택된 항목 추가**를 클릭합니다.
7.  **저장**을 클릭합니다.

**사용자 관리** 섹션에서 개발자 센터 계정에 추가한 그룹 계정을 변경할 수 있습니다. 그룹의 이름 및 구성원에 대한 변경 사항은 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 반영됩니다. 그룹 역할에 대한 변경 사항은 해당 그룹의 개발자 센터 액세스에만 영향을 줍니다.

**그룹 계정 편집**

1.  **사용자 관리** 페이지에서 편집하려는 그룹 계정의 이름을 클릭합니다.
2.  그룹 정보를 변경하려면 그룹 이름에 대해 원하는 대로 변경합니다. 이러한 변경 사항은 조직의 디렉터리에 반영된다는 점을 기억하세요.
3.  그룹 역할을 변경하려면 그룹에 적용하려는 역할을 선택 또는 선택 취소합니다.
4.  **저장**을 클릭합니다.

### Azure AD 응용 프로그램 추가 및 관리

조직의 Azure AD에 포함된 응용 프로그램이나 서비스에서 개발자 센터 계정에 액세스하도록 허용할 수 있습니다.

둘 이상의 Azure AD 응용 프로그램을 동시에 추가하는 경우 동일한 역할을 할당해야 합니다. 그룹을 추가하지만 서로 다른 역할을 할당하려는 경우 각 역할에 대해 아래 단계를 반복합니다.

**조직 디렉터리의 Azure AD 응용 프로그램 추가**

1.  **사용자 관리** 페이지에서 **Azure AD 응용 프로그램 추가**를 클릭합니다.
2.  표시되는 목록에서 하나 이상의 Azure AD 응용 프로그램을 선택합니다. 검색 상자를 사용하여 특정 Azure AD 응용 프로그램을 검색할 수 있습니다.
3.  Azure AD 응용 프로그램 선택이 완료되면 **선택된 항목 추가**를 클릭합니다.
4.  **역할** 섹션에서 하나 이상의 역할을 선택하여 이 Azure AD 응용 프로그램의 집합에 할당합니다.
5.  **저장**을 클릭합니다.

새 Azure AD 응용 프로그램 계정에 개발자 센터 액세스 권한을 부여하려는 경우 **사용자 관리** 섹션에서 새 사용자 계정을 만들 수 있습니다. 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 새 계정이 만들어집니다.

> **팁** 개발자 센터 인증에 이 Azure AD 응용 프로그램을 주로 사용하고 사용자가 직접 액세스할 필요가 없는 경우, 해당 값을 디렉터리의 다른 Azure AD 응용 프로그램에서 사용하지 않으면 **회신 URL** 및 **앱 ID URI**에 유효한 주소를 입력할 수 있습니다.

**새 Azure AD 응용 프로그램 만들기**

1.  **사용자 관리** 페이지에서 **Azure AD 응용 프로그램 추가**를 클릭합니다.
2.  다음 페이지에서 **새 Azure AD 응용 프로그램**을 클릭합니다.
3.  새 Azure AD 응용 프로그램에 대한 **회신 URL**을 입력합니다. 이 URL를 통해 사용자가 Azure AD 응용 프로그램에 로그인하고 이를 사용할 수 있습니다(앱 URL 또는 로그온 URL이라고도 함). **회신 URL**은 256자 이내여야 합니다.
4.  새 Azure AD 응용 프로그램에 대한 **앱 ID URI**를 입력합니다. 이 URI는 Azure AD에 Single Sign-On 요청을 보낼 때 제공되는 Azure AD 응용 프로그램의 논리적 식별자입니다. 디렉터리에서 각 Azure AD 응용 프로그램에 대한 **앱 ID URI**는 고유해야 하고 256자 이내여야 합니다.
5.  **역할** 섹션에서 하나 이상의 역할을 선택하여 새 Azure AD 응용 프로그램에 할당합니다.
6.  **저장**을 클릭합니다.

Azure AD 응용 프로그램을 추가하거나 만든 후 **사용자 관리** 섹션으로 돌아가 응용 프로그램 이름을 클릭하면 테넌트 ID, 클라이언트 ID, 회신 URL 및 앱 ID URI를 비롯한 응용 프로그램의 설정을 검토할 수 있습니다.

>**참고** [Windows 스토어 서비스](../monetize/using-windows-store-services.md)에서 제공하는 REST API를 사용하려는 경우 서비스 호출을 인증하는 데 사용할 수 있는 Azure AD 액세스 토큰을 얻으려면 이 페이지에 표시된 테넌트 ID 및 클라이언트 ID 값이 필요합니다.   

**사용자 관리** 섹션에서 개발자 센터 계정에 추가한 Azure AD 응용 프로그램을 변경할 수 있습니다. 회신 URL 및 앱 ID URI에 대한 변경 사항은 개발자 센터 계정뿐만 아니라 조직의 디렉터리에도 반영됩니다. 역할 변경 사항은 개발자 센터 내에서 Azure AD 응용 프로그램의 권한에만 영향을 줍니다.

**Azure AD 응용 프로그램 편집**

1.  **사용자 관리** 페이지에서 편집하려는 Azure AD 응용 프로그램 계정의 이름을 클릭합니다.
2.  **회신 URL** 또는 **앱 ID URI**를 변경하려면 여기에 새 값을 입력합니다. 이러한 변경 사항은 조직의 디렉터리에 반영된다는 점을 기억하세요.
3.  Azure AD 응용 프로그램 역할을 변경하려면 적용하려는 역할을 선택 또는 선택 취소합니다.
4.  **저장**을 클릭합니다.

Azure AD 응용 프로그램이 Microsoft Azure AD에서 데이터를 읽고 쓸 경우 키가 필요합니다. 개발자 센터에서 정보를 편집하여 Azure AD 응용 프로그램에 대한 키를 만들 수 있습니다. 더 이상 필요하지 않은 키를 제거할 수도 있습니다.

**Azure AD 응용 프로그램에 대한 키 관리**

1.  **사용자 관리** 페이지에서 Azure AD 응용 프로그램의 이름을 클릭합니다.

    > **팁** Azure AD 응용 프로그램의 이름을 클릭하면 Azure AD 응용 프로그램에 대한 모든 활성 키가 키를 만든 날짜 및 키 만료 시기와 함께 표시됩니다. 더 이상 필요하지 않은 키를 제거하려면 **제거**를 클릭합니다.

2.  새 키를 추가하려면 **새 키 추가**를 클릭합니다.

3.  **클라이언트 ID** 및 **키** 값을 보여 주는 화면이 표시됩니다.

    > **중요** 이 페이지를 나간 후에는 정보에 액세스할 수 없으므로 이 정보를 인쇄하거나 복사해야 합니다.

4.  키를 추가로 만들려면 **다른 키 추가**를 클릭합니다.

### 사용자, 그룹 및 Azure AD 응용 프로그램 제거

개발자 센터 계정에서 사용자, 그룹 또는 Azure AD 응용 프로그램을 제거하려면 **사용자 관리** 페이지에 해당 이름으로 표시되는 **제거** 링크를 클릭합니다. 제거하기를 원한다고 확인한 후에는 나중에 다시 추가하지 않는 한 그 사용자, 그룹 또는 Azure AD 응용 프로그램이 더 이상 개발자 센터 계정에 액세스할 수 없게 됩니다.

> **참고** 사용자, 그룹 또는 Azure AD 응용 프로그램 제거는 개발자 센터 계정에 더 이상 액세스 권한이 없음을 의미합니다. 사용자, 그룹 또는 Azure AD 응용 프로그램을 조직의 디렉터리에서는 삭제하지 않습니다.
