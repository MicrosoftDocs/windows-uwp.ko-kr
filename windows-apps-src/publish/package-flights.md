# <a name="package-flights"></a>패키지 플라이트

제한 된 그룹 테스터에 게 특정 패키지를 배포 하려면 패키지 플라이트를 사용할 수 있습니다. 스토어에 이미 게시 한 패키지 경험을 중단 되지 하므로 하면 다른 고객에 대 한 사용 됩니다.

패키지 플라이트를 사용 하 여 패키지만 다릅니다. 스토어 목록 세부 정보는 모든 고객에 대해 동일 하 게 됩니다. 플라이트 그룹의 모든 사용자가 플라이트 그룹에 속하지 않는 고객이 계속 일반 (플라이트 외) 패키지를 받게 패키지 플라이트에 포함 된 패키지를 받게 됩니다.  나중에 고객에 게 패키지 플라이트에서 패키지를 사용할 수 있도록 하려면, 플라이트 외 제출에서 이러한 동일한 패키지를 쉽게 사용할 수 있습니다. Note 패키지 플라이트는 [인증 프로세스](the-app-certification-process.md)를 모든 제출 것과 마찬가지로 통과 해야 합니다.

패키지 플라이트를 설정할 때에는 **알려진 사용자 그룹** (플라이트 그룹 라고도 함)를 추가 하 여 특정 패키지를 다운로드 해야 하는 사람을 지정할 수 있습니다. 플라이트 그룹에서 패키지 플라이트(Windows.Desktop 빌드 10586 이상, Windows.Mobile 빌드 10586.63 이상 또는 Xbox One)를 지원하는 Windows10 버전을 실행 중인 장치의 모든 사용자는 해당 특정 그룹에 대해 지정된 패키지 플라이트에서 패키지를 다운로드하게 됩니다. (패키지 플라이트에 Windows 8.1 / Windows Phone 8.1 이전에 게시 된 앱에서 이미 지 원하는 경우이 비롯 한 모든 OS 버전을 대상으로 하는 패키지를 포함할 수 있습니다.) 플라이트 그룹 중 하나에 추가 되지 않은 또는 패키지 플라이트를 지원 하지 않는 장치를 사용 하는 모든 사용자가 플라이트 외 제출에서 패키지를 가져오지 합니다.

> [!IMPORTANT] 
> 데스크톱 및 모바일 장치에서 플라이트 그룹의 사용자는 업데이트를 제공할 때마다 자동으로 플라이트의 패키지를 얻게 됩니다. 그러나 최신 패키지를 가져오려면 **Xbox 장치를 사용 중인 플라이트 그룹의 사용자가 수동으로 업데이트를 확인**하고 알려진 사용자 그룹에 포함된 관련 메일 주소를 사용하여 Microsoft 계정을 사용하는 장치에 로그인해야 합니다.

패키지 플라이트는 [비즈니스용 Microsoft 스토어](https://businessstore.microsoft.com/store) 및 [교육용 Microsoft Store](https://educationstore.microsoft.com/store)를 통해 배포되지 않습니다. 이는 알려진 사용자 그룹의 사용자가 패키지 플라이트를 받으려면 해당 Microsoft 계정을 사용하여 로그인해야 하기 때문입니다. 비즈니스용 Microsoft 스토어 및 교육용 Microsoft 스토어를 통해 구입한 모든 항목은 플라이트 외 패키지를 받게 됩니다.

> [!TIP]
> 패키지 플라이트는 지정된 고객에게만 패키지를 제공합니다. 임의의 고객을 선택하여 특정 비율만큼 패키지를 배포하려면 [점진적 패키지 출시](gradual-package-rollout.md)를 사용합니다. 또한 업데이트를 플라이트 그룹의 구성원 중 한 명에게 점진적으로 배포하려는 경우 패키지 플라이트를 출시와 결합할 수 있습니다.
>
> 패키지 플라이트와 달리 점진적 패키지 출시는 비즈니스용 Microsoft 스토어 및 교육용 Microsoft 스토어를 통해 앱을 구입하는 고객에게 적용됩니다. 

> [!TIP]
> 패키지 플라이트의 사용자가 앱에 대한 입력을 제공할 수 있는 방법도 고려해야 할 수 있습니다. 고객이 직접 입력을 제공할 수 있도록 [피드백 허브를 시작하는 컨트롤을 앱에 추가](../monetize/launch-feedback-hub-from-your-app.md)하는 것이 좋습니다. 그런 다음 앱의 [피드백 보고서](feedback-report.md)에서 피드백을 검토할 수 있습니다.


## <a name="create-a-new-package-flight"></a>새 패키지 플라이트 만들기

앱에 대한 제출을 게시하고 나면 앱 개요 페이지에 **패키지 플라이트** 섹션이 나타납니다. **새 패키지 플라이트**를 클릭하여 시작합니다.

아직 알려진 사용자 그룹을 만들지 않은 경우 계속하려면 먼저 알려진 사용자 그룹을 만들라는 메시지가 표시됩니다. 자세한 내용은 [알려진 사용자 그룹 만들기](create-known-user-groups.md)를 참조하세요. **플라이트 그룹 만들기**를 선택하여 이 페이지에서 직접 알려진 사용자 그룹을 만들 수 있습니다.

패키지 플라이트 생성 페이지에서 플라이트의 이름을 입력하고 하나 이상의 플라이트 그룹을 지정해야 합니다. 그런 다음 **플라이트 만들기**를 선택합니다. 나중에 이러한 세부 정보를 변경할 수 없습니다(그러나 입력한 내용이 마음에 들지 않으면 언제든지 삭제하고 대신 사용할 새 패키지 플라이트를 만들 수는 있음).

> [!NOTE]
> 패키지 플라이트가 두 개 이상인 경우, 각 패키지 플라이트에 순위를 할당해야 합니다. 자세한 내용은 [순위 패키지 플라이트 추가 및](#add-and-rank-additional-package-flights) 아래를 참조 하세요.


## <a name="specify-packages-to-include-in-your-package-flight"></a>패키지 플라이트에 포함할 패키지 지정

패키지 플라이트 세부 정보를 저장하고 나면 해당 개요 페이지가 나타납니다. **패키지**를 클릭하여 플라이트에 포함할 패키지를 지정합니다. 앱에서 지원 되는 모든 OS 버전을 대상으로 하는 패키지를 포함할 수 있습니다.

이전에 게시된 제출과 연결된 패키지를 선택할 수 있습니다. 두 개 이상인 경우 플라이트 외 제출을 선택하거나 다른 패키지 플라이트 중 하나를 선택합니다. 이 패키지 플라이트에 사용할 새 패키지를 업로드 해야 할 경우 수 여기서 업로드할 있습니다 ( [정기적인 플라이트 외 제출에 앱 패키지를 업로드할 때와 동일한 프로세스](upload-app-packages.md)를 사용 하 여). 이 패키지 플라이트에 포함할 패키지 지정을 완료했으면 **저장**을 클릭합니다.

앱에서 여러 장치 패밀리를 지원하는 경우 동일한 장치 패밀리 집합을 지원하도록 플라이트에 패키지를 포함해야 합니다. 플라이트 그룹의 사용자는 해당 플라이트에서**만** 패키지를 가져올 수 있습니다. 다른 플라이트나 플라이트 외 제출에 포함된 패키지에 액세스할 수는 없습니다. 

장치 패밀리 가용성은 플라이트 외 제출 기준 및도 스토어 목록 정보를 해야 합니다. 플라이트 그룹에 속한 고객은 플라이트 외 제출에서 지원하는 장치 패밀리에서만 앱을 다운로드할 수 있습니다. 자세한 내용은 [장치 패밀리 지원](#device-family-support)을 참조하세요. 


## <a name="gradual-package-rollout"></a>점진적 패키지 배포

기본적으로 제출의 패키지는 플라이트 그룹의 모든 사용자가 동시에 사용할 수 있습니다. 이를 변경하려면 **이 제출을 게시(Windows10 고객에게만)한 후 업데이트를 단계적으로 롤아웃하세요.** 상자를 선택합니다. 플라이트 그룹의 사용자 중 새 제출에서 패키지를 받을 비율을 선택하고 피드백 및 분석 데이터를 모니터링하여 플라이트 그룹의 나머지 사용자에게 출시하기 전에 업데이트의 품질을 확인할 수 있습니다. 패키지 플라이트에 대한 새 제출을 만들 필요 없이 언제든지 비율을 늘리거나 업데이트를 중지할 수 있습니다. 

> [!IMPORTANT]
> 패키지 플라이트에서 패키지를 점진적으로 출시, (없으면 높은 평점의 플라이트를 사용할 수) 새 패키지를 받는 비율에 포함 되지 않은 사용자는 이전 패키지 플라이트 제출에서의 패키지를 얻게 됩니다.

자세한 내용은 [점진적 패키지 출시](gradual-package-rollout.md)를 참조하세요.


## <a name="configure-additional-package-flight-options"></a>추가 패키지 플라이트 옵션 구성

기본적으로 패키지 플라이트는 인증 프로세스가 완료되는 즉시 게시되고 플라이트 그룹에 제공됩니다. **플라이트 옵션** 섹션에서 [게시 날짜](set-app-pricing-and-availability.md#publish-date)를 변경하거나 [인증에 대한 참고 사항](notes-for-certification.md)을 추가할 수 있습니다. **저장**을 클릭하여 패키지 플라이트 개요 페이지로 돌아갑니다. 


## <a name="submit-your-package-flight-to-the-store"></a>스토어에 패키지 플라이트 제출

패키지를 지정하고 필요한 옵션을 구성했으면 **스토어에 제출**을 클릭합니다. 그러면 패키지 플라이트는 [앱 인증 프로세스](the-app-certification-process.md)를 거치게 됩니다. 참고 패키지 플라이트에 포함 된 패키지는 모든 제출과 마찬가지로 [Microsoft Store 정책](https://docs.microsoft.com/legal/windows/agreements/store-policies)따라야 합니다.

이미 앱이 있고 해당 패키지 플라이트와 연결된 플라이트 그룹에 속한 사용자는 이제 패키지 플라이트에 포함된 패키지를 사용하여 업데이트를 다운로드합니다. 앱이 없는 경우 앱을 설치할 때 패키지 플라이트에서 패키지가 다운로드됩니다. 

> [!NOTE]
> 패키지 플라이트에서만 사용할 수 있는 패키지를 가진 사용자는 앱에 별 평점을 주고 리뷰를 남길 수 있지만 해당 평점 및 리뷰는 다른 고객에게 표시되지 않습니다. (레거시 7.x 또는 8.0 XAP 패키지는 제외됩니다. 이러한 패키지를 사용하는 플라이트 그룹의 구성원이 남긴 평점 및 리뷰는 다른 고객에게 표시됩니다.) 앱의 **리뷰** 및 **피드백** 보고서에서 플라이트 그룹의 고객을 비롯한 모든 고객의 평점과 피드백을 볼 수 있습니다.


## <a name="device-family-support"></a>장치 패밀리 지원

대부분의 경우 플라이트 외 제출에서 지원하는 장치 패밀리와 동일한 집합을 지원하는 패키지를 포함하려고 합니다. 앱에 대한 장치 패밀리의 가용성은 고객이 플라이트 그룹에 속해 있는지 여부에 상관없이 플라이트 외 제출을 기반으로 결정됩니다.

**플라이트 외 제출에서 해당 패키지 플라이트가 지원하지 않는 장치 패밀리를 지원하는 경우** 플라이트 그룹의 사용자는 해당 장치 패밀리에서 앱을 다운로드할 수 없게 됩니다. 예를 들어, 플라이트 외 제출에 모바일 및 데스크톱 패키지가 포함되어 있고 모바일 패키지만 포함하는 패키지 플라이트를 만든 경우 플라이트에 속하지 않는 고객이 데스크톱 패키지를 사용할 수 있더라도 해당 플라이트 그룹의 사용자는 모바일 장치에서만 앱을 다운로드할 수 있게 됩니다. 모바일 패키지에서 변경 내용을 테스트하기 위한 용도로 패키지 플라이트를 사용하더라도 패키지 플라이트의 플라이트 외 제출에 데스크톱 패키지를 포함시켜야 해당 플라이트 그룹에 있는 고객이 데스크톱 장치에서 앱을 다운로드할 수 있습니다.

**패키지 플라이트가 플라이트 외 제출에서 지원하지 않는 장치 패밀리를 지원하는 경우에는** 플라이트 그룹에 속해 있는지 여부에 상관없이 아무도 해당 장치 패밀리에서 앱을 다운로드할 수 없습니다. 예를 들어, 플라이트 외 제출에 모바일 패키지가 포함되어 있고 모바일 및 데스크톱 패키지가 포함된 패키지 플라이트를 만든 경우 해당 플라이트 그룹에 있는 사용자는 여전히 모바일 장치에서만 앱을 다운로드할 수 있습니다. 데스크톱 패키지는 어떤 사용자에게도 제공되지 않습니다. 해당 플라이트 그룹에 속한 사용자도 마찬가지입니다. 플라이트 그룹의 사용자가 데스크톱 패키지를 사용할 수 있게 하려면 먼저 데스크톱 패키지를 포함하도록 플라이트 외 제출을 업데이트해야 합니다. 앱 고객 모두에게 최상의 환경을 제공하려면 플라이트 외 제출에서 패키지 플라이트와 동일한 장치 패밀리를 지원해야 합니다. 

> [!NOTE]
> 패키지 플라이트에 추가된 패키지는 모든 OS 버전(또는 모든 Windows10 빌드 버전)을 지원할 수 있지만 위에서 언급한 대로, 플라이트 그룹의 사용자는 패키지 플라이트에서 패키지를 가져올 수 있도록 패키지 플라이트를 지원하는 Windows10 버전을 실행하는 장치를 사용해야 합니다(Windows.Desktop 빌드 10586 이상, Windows.Mobile 빌드 10586.63 이상).


## <a name="update-or-modify-your-package-flight"></a>패키지 플라이트 업데이트 또는 수정

이미 게시 한 패키지 플라이트에 대 한 새 제출을 만들려면 앱 개요 페이지에서 플라이트 이름 옆에 있는 **업데이트** 를 클릭 합니다. 그런 다음 플라이트 외 제출을 사용할 때처럼 새 패키지를 업로드(및 불필요한 패키지 제거)할 수 있습니다. 원하는 대로 다른 설정을 변경한 다음 **스토어에 제출**을 클릭하여 [앱 인증 프로세스](the-app-certification-process.md)를 통해 업데이트된 패키지 플라이트를 전송합니다.

새 업데이트를 만들어 제출하지 않고 기존 플라이트를 수정하려면 플라이트 이름 옆에 있는 **수정**을 클릭합니다. 이렇게 하면 패키지 플라이트가 인증 프로세스를 다시 통과하지 않고도 플라이트 그룹, 이름, 순위 등의 세부 정보를 변경할 수 있습니다. note 진행 중인 업데이트를가지고 있거나 패키지 플라이트를 아직 게시 되지 않은, **수정** 옵션 표시 되지 않습니다. 


## <a name="add-and-rank-additional-package-flights"></a>추가 패키지 플라이트 추가 및 순위 지정

고객 집합마다 다른 패키지를 배포하기 위해 동일한 앱에 대해 여러 패키지 플라이트를 만들 수 있습니다. 

첫 번째 패키지 플라이트를 만든 후 위에 설명된 프로세스에 따라 다른 패키지 플라이트를 만듭니다. 단, 이미 패키지 플라이트를 만든 경우 **순위** 섹션에서 모든 패키지 플라이트의 우선 순위를 지정해야 합니다. 이렇게 하면 스토어 둘 이상의 플라이트 그룹에 있는 경우 각 고객에 게 패키지를 결정할 수 있습니다. 더 낮은 순위의 패키지 플라이트에 더 높은 버전 번호의 패키지가 포함되어 있어도 플라이트 그룹의 사용자는 항상 사용 가능한 최상위 순위 패키지 플라이트를 다운로드합니다.

기본적으로 새 패키지 플라이트에 가장 높은 순위가 지정됩니다. 이 순위를 변경하려면 오른쪽 위치에서 다른 패키지 플라이트의 위쪽 또는 아래쪽으로 이동합니다.

플라이트 외 제출 항상는 순위가 가장 낮은 (#1)입니다. 즉, 플라이트 그룹에 속하지 않은 사용자는 스토어를 통해 플라이트 외 제출의 패키지만 다운로드할 수 있습니다. 플라이트 그룹의 사용 가능한 최상위 패키지 플라이트에서 패키지 (최상위 플라이트 외 제출에 가장 낮은 순위 있기 때문) 얻게 항상 됩니다. 이렇게 하면 둘 이상의 플라이트 그룹의 구성원인 사용자에게 패키지를 배포하는 방법을 유연하게 결정할 수 있습니다.

예를 들어 정기적인 플라이트 외 제출과 함께 두 개의 패키지 플라이트를 더 만들려고 하는데, 하나는 비교적 안정적이어서 광범위한 대상에게 테스트할 수 있고 다른 하나는 확신할 수 없어 일부 테스터로만 제한하려고 합니다. Testers라는 플라이트 그룹을 만들어 Tester Flight라는 패키지 플라이트에 포함한 다음 더 많은 구성원이 있는 Enthusiasts라는 플라이트 그룹을 만들어 Enthusiast Flight라는 다른 패키지 플라이트에 포함할 수 있습니다. Tester Flight의 순위를 Enthusiast Flight보다 높게 설정한 경우 Enthusiast Flight에서 꽤 확실한 패키지를 사용할 수 있지만 Testers의 경우 위험할 수 있지만 Tester Flight의 패키지만 사용하게 됩니다. Testers 그룹의 구성원이 Enthusiasts 그룹에 속한 경우에도 항상 Tester Flight에 제공된 패키지를 다운로드합니다. 나중에 Tester Flight의 패키지가 올바르게 수행된다고 밝혀지면 원래 Tester Flight에 배포된 패키지를 사용하도록 Enthusiast Flight를 업데이트할 수 있으며 최종적으로는 플라이트 외 제출에서 이러한 패키지를 사용할 수 있습니다.


## <a name="make-packages-from-a-package-flight-available-to-all-your-customers"></a>모든 고객이 사용할 수 있는 패키지 플라이트에서 패키지를 만듭니다.

게시된 패키지 플라이트에 포함된 패키지 중 하나 이상을 플라이트 그룹에 속하지 않은 고객이 사용할 수 있도록 설정해야 하는 경우 동일한 패키지를 또다시 업로드하지 않고 해당 패키지를 사용하도록 플라이트 외 제출을 업데이트할 수 있습니다. 

새 제출을 만들면 패키지 플라이트 중 하나에서 패키지를 복사하는 옵션이 포함된 드롭다운이 [**패키지**](upload-app-packages.md) 페이지에 표시됩니다. 끌어오려는 패키지가 있는 패키지 플라이트를 선택합니다. 그런 다음 해당 패키지를 일부 또는 전부 선택하여 플라이트 외 제출에 포함할 수 있습니다.

이전에 게시된 제출의 패키지를 사용하는 경우에도 동일한 패키지 유효성 검사 규칙이 모두 적용됩니다. 


## <a name="delete-a-package-flight"></a>패키지 플라이트 삭제

더 이상 지원하지 않으려는 패키지 플라이트를 삭제하려면 앱 개요 페이지에서 해당 이름을 클릭합니다. 플라이트 개요 페이지에서 **수정**을 클릭한 다음 **삭제** 삭제 링크를 클릭하여 패키지 플라이트를 삭제하세요. 게시되지 않은 패키지 플라이트의 제출이 진행 중인 경우 해당 제출을 먼저 삭제해야 합니다. 이 작업을 완료하는 데 최대 30분이 걸릴 수 있습니다.

패키지 플라이트를 삭제할 경우 더 높은 버전 번호의 패키지가 있으면(또는 곧 이러한 패키지가 제공될 경우) 해당 패키지 플라이트에서 배포한 패키지를 소유한 모든 고객은 앱 업데이트를 다운로드하게 됩니다. 앱을 제거한 후 나중에 다시 설치하면 새 구입으로 처리되어 현재 사용 가능한 버전 중 가장 높은 버전이 다운로드됩니다.
