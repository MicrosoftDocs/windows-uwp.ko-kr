# <a name="discovering-xbox-tournaments"></a>Xbox 토너먼트를 검색합니다.

게이머는 여러 가지가 Xbox 대시보드 또는 Xbox 앱에서 pc에 참여 하는 토너먼트를 시청 하거나 하는 방법에 대 한 자세한 내용을 보려면. 제목 내에서 토너먼트 검색을 향상 시킬 기회가 있습니다.  

게이머는 아레나 가능한 토너먼트에서 찾을 수 있습니다.

* Xbox 홈에서 최근에 재생된 한 게임 토너먼트 허브를 엽니다.
* 토너먼트 재생, 지금 재생 또는 등록에 대 한 플레이어 프로필을 방문 합니다.
* Looking 그룹 (LFG) 클럽에 게시 하는 팀에 대 한 응답에 속해 있습니다.
* 믹서 응용 프로그램, 웹 사이트 또는 조사식 피벗 된 게임 허브/아레나 토너먼트 스트림을 보고 있습니다.
* Xbox 대시보드에서 게임 허브를 방문 합니다.

## <a name="promoting-tournaments-in-game"></a>토너먼트 게임 홍보

제목에 관심이 있기 때문에 토너먼트를 시청 하는 사람. Xbox 게임 특별 한 기회를 게이머 자신의 게임 플레이 경험을 확장 하는 새로운 방법에 대 한 보고 경우 그 중요 한 시점에 있으며

> [!TIP]
> **UX 권장 사항:** 의사 결정의 순간에 토너먼트를 수준을 올립니다.
>
> 토너먼트에 대 한 인지도를 때때로 게이머 재생을 유지 하거나 (예를 들어, 게임 실행, 주 메뉴, 멀티 플레이어 메뉴에서 시작 또는 끝에 일치) 종료 여부를 결정 하는 데 필요한 때 발생 시킵니다.

### <a name="1--use-a-dynamic-announcement-feature-example-message-of-the-day"></a>1. 사용 하 여 동적 알림 기능 (예: 얼 리 전스 메시지)

게임의 UI에 알림 기능에는 일반적으로 작성 되어 있는 타이틀 빠른 업데이트를 허용 하는 콘텐츠 관리 시스템을 사용 합니다. 콘텐츠 업데이트 또는 다운로드 가능한 콘텐츠 출시에 의존 하지 않고 새 콘텐츠 게이머 푸시할 때 유용 합니다. 예를 들어, Halo 표면 커뮤니티 공지 사항 및 팁에 "메시지가 오늘의" 기능을 사용 합니다. 토너먼트 프로 모션은이 시나리오를 완벽 하 게 적합 합니다.

**사용자 영향**

* 게이머는 새로운 경쟁력 있는 게임 기회 게임 세션을 시작 하기 전에 도입 되었습니다.
* 노출은 (다른 공지 사항을 함께 출시)에서 간헐적으로 발생 합니다.
* 자세한 내용은 게이머 아레나 허브로 이동 하 고 게임을 이동 합니다.
* 기능을 채우는 콘텐츠 업데이트 하지 않아도 됩니다.
* 관련성 및 프로 모션의 유연한 유지 됩니다.

관통 연결에 게임 허브/토너먼트-게임에 관련 된 모든 토너먼트 홍보 것이 좋습니다.

#### <a name="c-game-exit-confirmation"></a>C. 게임 종료 확인

## <a name="browsing-tournaments-in-game"></a>토너먼트 게임 검색

아레나 Api는 게임 내에서 자유로이 검색 제목에 대 한 충분 한 데이터를 제공 합니다. 탐색 기능을 작성 하려는 경우 멀티 섹션에서 **토너먼트** 진입점을 추가 합니다.

> [!TIP]
> **UX 권장 사항**  
> 토너먼트는 멀티 플레이어 게임 모드를 제공 합니다.  
> Xbox 아레나는 새로운 기능 이므로 1 계층 또는 2 계층 수준에서 보이는 상태로 유지 해야 합니다.

###### <a name="ui-example-tournaments-listed-as-an-additional-mode-in-the-multiplayer-section"></a>추가 모드로 멀티 섹션에서 나열 된 예제 UI: 토너먼트

![토너먼트 모드](../../images/arena/arena-ux-tournament-mode.png)

### <a name="provide-a-comprehensive-list-of-tournaments"></a>토너먼트의 포괄적인 목록을 제공 합니다.

토너먼트에 등록 된 현재 것의 상태를 빠르게 확인 하 고 토너먼트 게임 플레이 세션 간에 탐색 하는 게이머의 역량을 강화.

#### <a name="user-impact"></a>사용자 영향

* 대시보드에 토너먼트 상태를 확인 하려면 게임 게이머의 필요성을 최소화 합니다.
* 게이머는 Xbox 아레나 알림 알림 놓칠 경우 좋은 백업 솔루션으로 사용 됩니다.
* 관리 게임 개발자를 위한 추가 비용을 포함지 않습니다.
* 게이머 참가 등록, 체크 인, 시드 및 팀 대형 아레나 UI에 연결 됩니다.

> [!NOTE]  
> 아레나 Api는 사용자가 생성 한 토너먼트 (클럽 생성)에 대 한 쿼리를 제공 하지 않습니다.


> [!TIP]
> **UX 권장 사항**  
> 비율을 게이머 토너먼트 큰 목록을 관리 하기 위한 방법을 제공할 수 있는 UI를 만듭니다.

### <a name="filters"></a>필터

**모두**, **현재** (토너먼트는 종료 될 때까지 모든 것을 포함)를, **취소 됨**, **완료 됨** 상태에 대 한 토너먼트와 토너먼트는 게이머에 있거나 (예를 들어, 내 토너먼트)에 참여 하는 것을 필터링 합니다.

###### <a name="ui-example"></a>UI의 예:

![토너먼트 필터 화면](../../images/arena/arena-ux-filters.png)

> [!NOTE]  
> 외부 API에서 지 원하는 사용자 지정 필터를 추가 하는 것은 가능 *하지만 권장 되지 않습니다*.

제목 요청을 하면 다음 쿼리 매개 변수를 지정할 수 없습니다 다른 속성으로 필터링 할 수 있습니다. 따라서,이 이와 같은 UI에 대 한 신뢰할 수 없는 다른 속성에 의해 필터링 됩니다. 아레나 Api는 제목이 지정 된 수의 결과 한 번에 검색-제목 10 토너먼트 **maxItems** 값을 지정할 수 있습니다 예를 들어. 성능 관리 제목 주며 큰 목록 사용 하 여 사용자 폭주의 위험을 최소화 합니다. 그러나 게임에서 제공 되는 사용자 지정 필터 요청 당시에 10 개의 항목에만 적용 됩니다. 각 API 호출 후에 필터링 된 결과 일치 하지 않는 수가 발생할 수 있습니다. 예를 들어 첫 번째 집합이 반환 10 토너먼트 2 '게임.' 처럼 표시 되는 사용자 지정 필터와 관련 된 포함 하지만 앞으로 10 수 9 항목을 표시 하 고 있습니다. 제목의 각 사용자 지정 필터를 완전히 채울 수 있는 믿을 수 있는 유일한 방법은 모든 활성 토너먼트를 요청 하 고 사용 하는 게 모두를 필터링 하는 것.

#### <a name="filter-recommendations"></a>권장 되는 필터:

##### <a name="all"></a>모든

모든 **활성**, **취소 됨**, **완료 됨** 토너먼트 자주 정렬 보기 최근.

결과:

* 토너먼트의 상태
* 토너먼트 시작 날짜
* 토너먼트 상태-예를 들어, 등록 열기
* 대시보드에 토너먼트 세부 정보 페이지에 딥 링크

##### <a name="my-tournaments"></a>내 토너먼트

현재 참여자에 등록 되어 있는 보기 토너먼트.

결과:

* 현재 로그인 한 사용자에 참여 하는 토너먼트
* 토너먼트의 상태
* 토너먼트와 일치 하는 ('수 일치' 상태)를 입력 하는 방법
* 대시보드에 토너먼트 세부 정보 페이지에 딥 링크

##### <a name="active"></a>활성

만들어져 있는 모든 토너먼트 보기: 앞, 등록, 체크 인, 및 재생을 위해 열려 있습니다.

결과:

* 현재 사용자는 아직 참여 하지 않은, 등록으로 토너먼트
* 등록을 닫을 때까지 남은 시간

##### <a name="completedcanceled"></a>완료/취소

최근 결과 봅니다. 방금 종료 된 토너먼트 완료 후 간단 하 게 사라지는 대신 시간이 지나면서 에이징 아웃 수 있습니다.

결과:

* 대시보드에 토너먼트 세부 정보 페이지에 딥 링크
* 토너먼트의 상태
* 끝 날짜/시간

##### <a name="canceled"></a>취소

취소 된 보기 토너먼트입니다.

결과:

* 대시보드에 토너먼트 세부 정보 페이지에 딥 링크
* 취소의 날짜/시간
