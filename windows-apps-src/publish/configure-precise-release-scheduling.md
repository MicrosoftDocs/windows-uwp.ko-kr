# <a name="configure-precise-release-scheduling"></a>정확한 릴리스 예약 구성

[가격 책정 및 가용성](set-app-pricing-and-availability.md) 페이지의 **일정** 섹션에서 앱을 스토어에서 사용할 수 있는 정확한 날짜와 시간을 설정 하 여 더 많은 유연성을 제공 하 고 다른 시장의 날짜를 사용자 지정할 수 있습니다.

> [!NOTE]
> 이 항목은 앱을 참조 하지만 추가 기능 제출에 대 한 릴리스 일정은 동일한 프로세스를 사용 합니다.

제품을 스토어에서 더 이상 사용할 수 없는 날짜를 설정 하도록 선택할 수도 있습니다. 이는 검색 또는 검색을 통해 제품을 스토어에서 더 이상 찾을 수 없다는 것을 의미 하지만 직접 링크를 가진 고객은 제품의 상점 목록을 볼 수 있습니다. 제품을 이미 소유 하 고 있거나 [프로 모션 코드가](generate-promotional-codes.md) 있고 Windows 10 장치를 사용 하는 경우에만 다운로드할 수 있습니다.

기본적으로 ( **이 앱을 사용할 수 있도록 설정** 하 고, [표시 유형](choose-visibility-options.md#discoverability) 섹션의 스토어 옵션에서 검색할 수 없습니다.) 중 하나를 선택 하지 않은 경우 앱은 인증을 전달 하 고 게시 프로세스를 완료 하는 즉시 고객에 게 제공 됩니다. 다른 날짜를 선택 하려면 **옵션 표시** 를 선택 하 여이 섹션을 확장 합니다.

앱이 고객에 게 릴리스되지 않기 때문에 [표시 유형](choose-visibility-options.md#discoverability) 섹션의 **스토어 옵션에서이 앱을 사용할 수 있지만 검색할 수 없음** 옵션 중 하나를 선택한 경우 **일정** 섹션에서 날짜를 구성할 수 없습니다. 따라서 구성할 릴리스 날짜가 없습니다.

> [!IMPORTANT]
> 일정 섹션에서 지정한 날짜는 Windows 10의 고객 에게만 적용 됩니다.
>
>이전에 게시 된 앱에서 이전 OS 버전을 지 원하는 경우 선택한 **중지 취득** 날짜는 해당 고객에 게 적용 되지 않습니다. [표시 유형](choose-visibility-options.md#discoverability) 섹션에서 새 선택 항목을 사용 하 여 업데이트를 제출 하지 않거나 앱 **개요** 페이지에서 **앱을 사용할 수 없음** 을 선택 하는 경우에도 앱을 가져올 수 있습니다.

## <a name="base-schedule"></a>기본 일정

[특정 시장에 대해 사용자 지정](#customize-the-schedule-for-specific-markets)을 선택 하 여 나중에 특정 시장 (또는 시장 그룹)에 대 한 날짜를 추가 하지 않는 한 기본 일정을 선택 하는 것은 앱을 사용할 수 있는 모든 시장에 적용 됩니다.

**릴리스** 및 **중지 획득** 에서 두 가지 옵션을 볼 수 있습니다.

## <a name="release"></a>Release

**릴리스** 드롭다운에서 앱을 스토어에서 사용할 수 있도록 하려는 시기를 설정할 수 있습니다. 즉, 검색 또는 검색을 통해 앱이 스토어에서 검색 가능 하 고, 고객이 스토어 목록을 보고 앱을 획득할 수 있습니다.

>[!NOTE]
> 앱을 게시 하 고 스토어에서 사용할 수 있게 되 면 앱이 이미 출시 되었으므로 **릴리스** 날짜를 더 이상 선택할 수 없습니다.

제품의 **릴리스** 일정에 대해 구성할 수 있는 옵션은 다음과 같습니다.
- **가능 하면 빨리** : 제품이 인증 되 고 게시 되는 즉시 출시 됩니다. 기본 옵션입니다.
- **at** : 선택한 날짜와 시간에 제품이 출시 됩니다. 또한 다음과 같은 두 가지 옵션을 사용할 수 있습니다.
   - **Utc** : 선택 하는 시간은 응용 프로그램이 어디에서 나 동시에 릴리스 되도록 Utc (Universal 협정 time) 시간이 됩니다.
   - **로컬** : 선택한 시간은 시장에 연결 된 각 표준 시간대에 사용 됩니다. (둘 이상의 표준 시간대를 포함 하는 시장의 경우 해당 시장의 표준 시간대 하나만 사용 됩니다. 미국의 경우 동부 표준 시간대가 사용 됩니다. 표준 시간대의 포괄적인 목록은이 페이지 아래쪽에 표시 됩니다.
- **예약 되지 않음** : 스토어에서 앱을 사용할 수 없습니다. 이 옵션을 선택 하는 경우 나중에 새 제출을 만들고 다른 옵션 중 하나를 선택 하 여 스토어에서 앱을 사용할 수 있도록 설정할 수 있습니다.

## <a name="stop-acquisition"></a>획득 중지

**획득 중지** 드롭다운에서 새 고객이 스토어에서이를 획득 하거나 목록을 검색할 수 있도록 하는 데 필요한 날짜 및 시간을 설정할 수 있습니다. 이는 앱 중 둘 이상의 가용성을 조정 하는 경우와 같이 앱이 더 이상 새 고객에 게 제공 되지 않을 시기를 정확 하 게 제어 하려는 경우에 유용할 수 있습니다.

기본적으로 **Stop 획득** 은 never로 설정 됩니다. 이를 변경 하려면 드롭다운에서 **를 선택 하** 고 위에서 설명한 대로 날짜 및 시간을 지정 합니다. 선택한 날짜와 시간에 고객이 더 이상 앱을 획득할 수 없게 됩니다.

이 옵션은 **이 앱을 검색 가능 하지만 표시 안 함 섹션에서 사용할 수** 없도록 선택 하 고 취득 중지를 [Visibility](choose-visibility-options.md#discoverability) 선택 하는 것과 동일한 영향을 줍니다 **. 직접 링크를 사용 하는 모든 고객은 제품의 상점 목록을 볼 수 있지만 이전에 제품을 소유 하 고 있거나 프로 모션 코드가 있고 Windows 10 장치를 사용 하는 경우에만 다운로드할 수 있습니다** . 새 고객에 게 앱 제공을 완전히 중지 하려면 앱 개요 페이지에서 **앱을 사용할 수 없음** 을 클릭 합니다. 자세한 내용은 [스토어에서 앱 제거](guidance-for-app-package-management.md#removing-an-app-from-the-store)를 참조 하세요.

> [!TIP]
> **획득을 중지할** 날짜를 선택 하 고 나중에 앱을 다시 사용할 수 있도록 설정 하려는 경우 새 제출을 만들고 **중지 획득** 을 다시 사용 **안 함** 으로 변경할 수 있습니다. 업데이트 된 제출이 게시 된 후 앱을 다시 사용할 수 있게 됩니다.

## <a name="customize-the-schedule-for-specific-markets"></a>특정 시장의 일정 사용자 지정

기본적으로 위에서 선택 하는 옵션은 앱이 제공 된 모든 시장에 적용 됩니다. 특정 시장의 가격을 사용자 지정 하려면 **특정 시장에 대해 사용자 지정** 을 클릭 합니다. 앱을 사용할 수 있도록 선택한 모든 시장을 나열 하는 **시장 선택** 팝업 창이 표시 됩니다. [시장 섹션에서](./define-market-selection.md) 시장을 제외한 경우 해당 시장이 표시 되지 않습니다.

한 시장에 대 한 일정을 추가 하려면 선택 하 고 **저장** 을 클릭 합니다. 그러면 위에서 설명한 것과 동일한 **릴리스** 및 **중지 획득** 옵션이 표시 되지만 선택 항목은 해당 시장에만 적용 됩니다.

여러 시장에 적용 되는 일정을 추가 하려면 *시장 그룹* 을 만듭니다. 이렇게 하려면 포함 하려는 시장을 선택 하 고 그룹의 이름을 입력 합니다. 이 이름은 참조용 이며 고객에 게 표시 되지 않습니다. 예를 들어 북아메리카에 대 한 시장 그룹을 만들려는 경우 **캐나다** , **멕시코** 및 **미국** 을 선택 하 고 이름을 **북아메리카** 또는 선택한 다른 이름으로 지정할 수 있습니다. 시장 그룹 만들기를 완료 한 후 **저장** 을 클릭 합니다. 그러면 위에서 설명한 것과 동일한 **릴리스** 및 **중지 획득** 옵션이 표시 되지만 선택 항목은 해당 시장 그룹에만 적용 됩니다.

추가 시장 또는 추가 시장 그룹에 대 한 사용자 지정 일정을 추가 하려면 **특정 시장의 사용자** 지정을 다시 클릭 하 고이 단계를 반복 합니다. 시장 그룹에 포함 된 시장을 변경 하려면 해당 이름을 선택 합니다. 시장 그룹 (또는 개별 시장)의 사용자 지정 일정을 제거 하려면 **제거** 를 클릭 합니다.

> [!NOTE]
> 시장는 **일정** 섹션에서 사용 하는 두 개 이상의 시장 그룹에 속할 수 없습니다.
