# <a name="app-screenshots-images-and-trailers"></a>앱 스크린샷, 이미지 및 예고편

잘 디자인된 이미지는 Microsoft Store의 잠재 고객에게 앱을 보여줄 수 있는 주요 방법 중 하나입니다.

앱의 Microsoft Store 목록에 포함시킬 [스크린샷](#screenshots), [로고](#store-logos), 기타 아트 자산([예고편](#trailers) 및 [홍보 이미지](##additional-art-assets))을 제공할 수 있습니다. 이들 중 일부는 필수이고 일부는 선택 사항입니다(단, Microsoft Store에서 최적의 표시를 위해 일부 이미지 옵션을 포함시킬 것을 권장). 

[앱 제출 프로세스](app-submissions.md) 중에 [Microsoft Store 목록](create-app-store-listings.md) 단계에서 이러한 아트 자산을 제공해 주세요. Microsoft Store에서 사용되는 이미지와 이들의 표시 방법은 고객의 운영 체제와 기타 요소에 따라 다를 수 있습니다.

또한 Microsoft Store에서는 앱의 타일 및 그 밖에 앱 패키지에 포함한 기타 이미지를 사용합니다. [앱을 제출하기 전에 Windows 앱 인증 키트](../debug-test-perf/windows-app-certification-kit.md)를 실행하여 필수 이미지가 누락되었는지 여부를 확인하세요. 이러한 이미지에 대한 지침과 권장 사항은 [타일 및 아이콘 자산](../design/shell/tiles-and-notifications/app-assets.md)을 참조하세요.

## <a name="screenshots"></a>스크린샷

스크린샷은 앱의 Store 목록에서 고객에게 표시되는 앱 이미지입니다.

앱에서 지원하는 다른 장치 패밀리에 스크린샷을 제공하여 고객이 해당 유형의 장치에서 앱의 Store 목록을 확인할 때 적절한 스크린샷이 표시되도록 지원하는 옵션이 있습니다 

모든 장치 패밀리에서 반드시 제출해야 하는 스크린샷은 하나이지만, 여러 개의 스크린샷을 제공할 수 있습니다(데스크톱 스크린샷은 최대 9개, 다른 장치 제품군은 최대 8개). 고객이 자신의 장치 유형에서 어떻게 앱이 표시되는지 알 수 있도록 앱에서 지원하는 각 장치 패밀리에 최소 4개의 스크린샷을 제공할 것을 권장합니다. (앱에서 지원하지 않는 장치 패밀리에 대한 스크린샷은 포함되지 않음). **데스크톱** 스크린샷은 Surface Hub 장치에서 고객에게도 표시된다는 점에 유의하세요.

> [!NOTE]
> Microsoft Visual Studio는 [스크린샷을 캡처하는 데 유용한 도구](https://docs.microsoft.com/visualstudio/debugger/run-windows-store-apps-in-the-simulator#BKMK_Capture_a_screenshot_of_your_app_for_submission_to_the_Microsoft_Store)를 제공합니다.

각 스크린샷은 가로 및 세로 방향의 .png 파일이어야 하고 파일 크기는 50MB 이하여야 합니다.

크기 요구 사항은 장치 패밀리에 따라 달라집니다.
- 데스크톱: 1366 x 768 픽셀 이상. 4K 이미지(3840 x 2160)를 지원합니다 (Surface Hub 디바이스에서 고객에게도 표시).
- 모바일: 이미지가 1080 x 1920, 1920 x 1080, 768 x 1280, 1280 x 768, 720 x 1280, 1280 x 720, 800 x 480 또는 480 x 800 픽셀 중 하나여야 합니다.
- Xbox: 3480 x 2160 픽셀 이하. 4K 이미지(3840 x 2160)를 지원합니다.
- 홀로그램: 1268 x 720 픽셀 이상. 4K 이미지(3840 x 2160)를 지원합니다.

최적의 표시를 위해 스크린샷을 생성할 때 다음 지침을 기억해야 합니다.
- 중요한 화면 효과와 문자는 이미지 위 3/4에 유지합니다. 문자 오버레이는 아래 1/4에 표시합니다. 
- 스크린샷에 추가 로고, 아이콘, 마케팅 메시지를 추가하지 마세요.
- 문자 오버레이의 가독성을 방해할 수 있는 너무 밝거나 어두운 색, 대비되는 줄무늬를 사용하지 마세요.

각 스크린샷을 설명하는 200자 이내의 짧은 캡션을 제공할 수 있습니다.

> [!TIP]
> 스크린샷은 목록에 순서대로 표시됩니다. 스크린샷을 업로드 한 후, 끌어서 순서를 바꿀 수 있습니다. 

Microsoft Store 목록을 [여러 언어](supported-languages.md)로 생성할 경우, 각 언어 별로 **Microsoft Store 목록** 페이지를 만들어야 합니다. 각 언어에 대한 이미지를 개별적으로 업로드하고(동일한 이미지를 사용하는 경우에도 해당) 각 언어에 사용할 자막을 제공해야 합니다.


## <a name="store-logos"></a>Microsoft Store 로고

Store에서 더 많은 사용자 정의 디스플레이 만들기 위해 Store 로고를 업로드할 수 있습니다. 앱에서 지원되는 모든 장치 및 OS 버전에서 최적의 Store 목록 표시를 위해 이러한 이미지를 제공하는 것이 좋습니다. Xbox의 고객이 앱을 사용할 수 있는 경우에는 이러한 이미지의 일부가 필요할 수 있다는 점에 유의하세요.

다음 지침에 따라 .png 파일(크기가 50MB 이하)로 이미지를 제공할 수 있습니다.

### <a name="916-poster-art-720-x-1080-or-1440-x-2160-pixels"></a>9:16 포스터 아트(720 x 1080 또는 1440 x 2160 픽셀)

이는 Windows 10 및 Xbox 장치의 고객을 위한 주 로고 이미지로도 사용되기 때문에 적절한 표시를 위해 이 이미지를 제공할 것을 **적극 권장**합니다. 이를 포함시키지 않으면 목록이 적절하게 표시되지 않을 수 있으며, Store를 검색하는 동안 고객에게 표시되는 다른 목록과 일치하지 않게 됩니다. 이 이미지는 검색 결과나 편집본 컬렉션에서 사용될 수도 있습니다.

이 이미지에는 앱의 이름이 포함되어 있어야 하고, 이미지의 텍스트는 사용 가능한 가독성 요구 사항(4.51 대비 비율)을 충족해야 합니다. 텍스트 오버레이가 이 이미지의 하단 1/4 부분에 표시될 수 있으므로 텍스트나 키 이미지가 거기에 포함되어 있어서는 안 됩니다.

> [!NOTE]
> Xbox에서 고객이 앱을 사용할 수 있는 경우에는 이 이미지가 **필수**이며 제품의 제목을 포함해야 합니다. 텍스트 오버레이가 이미지의 하단 1/4 부분에 나타날 수 있으므로 제목은 이미지의 상담 3/4 부분에 나타납니다.

### <a name="11-box-art-1080-x-1080-or-2160-x-2160-pixels"></a>1:1 박스 아트(1080 x 1080 또는 2160 x 2160 픽셀)

이 이미지는 Windows 10(Xbox 포함)의 다양한 Store 페이지에 나타날 수 있으며, **16:9 포스터 아트** 이미지를 제공하지 않는 경우에는 주 로고로 사용됩니다. 이 이미지에는 앱의 이름도 포함되어야 합니다. 텍스트 오버레이가 이 이미지의 하단 1/4 부분에 표시될 수 있으므로 텍스트나 키 이미지가 거기에 포함되어 있어서는 안 됩니다. 이 이미지에는 앱의 이름도 포함되어야 합니다. 

> [!NOTE]
> Xbox에서 고객이 앱을 사용할 수 있는 경우에는 이 이미지가 **필요**하며 제품의 제목을 포함해야 합니다. 텍스트 오버레이가 이미지의 하단 1/4 부분에 나타날 수 있으므로 제목은 이미지의 상단 3/4 부분에 나타납니다.

### <a name="11-app-tile-icon-300-x-300-pixels"></a>1:1 앱 타일 아이콘(300 x 300 픽셀)

Windows Phone 8.1 이전 버전에서 적절한 표시를 위해서는 이 이미지가 필요합니다. 앱에서 Windows Phone 8.1 이전 버전이 지원되지만 이 이미지가 제공되지 않는 경우에는 고객에게 앱 목록과 함께 공백 아이콘이 표시됩니다 (앱이 Windows Phone 8.1 이전 버전을 대상으로 하는 패키지만 갖고 있는 경우에는 Windows 10 고객에게도 적용됩니다.) 제출에 *오직* UWP 패키지만 포함되어 있다면, 이 이미지를 제공할 필요가 없습니다. (제출에 Windows Phone 8.x 패키지와 UWP 패키지가 모두 포함되어 있는 상태에서 이 이미지가 제공되는 경우에는 특정 Store 레이아웃으로 Windows 10에서 이를 사용할 수 있다는 점에 유의하세요. 이러한 문제를 막으려면 앱에서 지원하는 Windows Phone 버전용 [플랫폼별 목록](create-platform-specific-store-listings.md)을 생성하여 여기에 앱 타일 아이콘만 포함시킬 수 있습니다.).

### <a name="display-only-uploaded-logo-images-in-the-store"></a>디스플레이 스토어 로고 이미지를 업로드

저장소 (Xbox 포함) Windows 10의 고객에 게 목록을 표시할 때 앱 패키지의 로고 이미지를 사용 하는 것을 방지 하며 대신 업로드 한 이미지만 사용 하 여 스토어에는 옵션이 있습니다. 이렇게 하면 Windows 10(Xbox 포함) 고객을 위한 Store에서 다양한 디스플레이에 앱이 표시되는 모양을 훨씬 세밀하게 제어할 수 있습니다. (제품 이전 OS 버전을 지 원하는 경우 해당 고객에 게 표시 될 수 이미지, 패키지에서이 옵션을 사용 하는 경우에.)

Windows 10 및 Xbox에서 고객에 대 한 저장소 (고객을 위한 Xbox를 포함 하 여 Windows 10)을 업로드 한 이미지만 사용 하지 패키지의 모든 이미지를 사용, **이라는 확인란에 표시를 로고 이미지 패키지 이미지 대신 업로드 **.

이 확인란을 선택 하면 **이미지를 표시 하는 저장소** 라는 섹션이 새로 나타납니다. 여기에서는 **1:1 앱 타일 아이콘 (300 x 300 픽셀)** 크기 (이 섹션으로 이동 해당 이미지를 제공 하는 필드가 상자를 확인) 등 이미지 3 개를 업로드할 수 있습니다. 이 옵션을 사용하는 경우 300 x 300 픽셀, 150 x 150 픽셀, 71 x 71 픽셀 등 3가지 이미지 크기를 제공하는 것이 좋습니다. 그러나 반드시 제공해야 하는 이미지 크기는 300x300입니다.

## <a name="additional-art-assets"></a>추가 아트 자산

이 섹션에서는 Store에서 제품을 더욱 효과적으로 표시하는 데 도움이 되는 아트를 제공할 수 있습니다. 고객의 눈길을 끄는 Store 목록을 만들려면 이러한 이미지를 제공하는 것이 좋습니다.

> [!TIP]
> **16:9 슈퍼 히어로 아트**는 Store 목록에 [비디오 예고편](#trailers)을 포함시킬 계획인 경우에 특히 좋습니다. 이것을 포함시키지 않으면 목록 위쪽에 예고편이 표시되지 않습니다.

이미지를 추가하려면 **추가 아트 자산** 섹션에서 **세부 정보 표시**를 선택하세요.



### <a name="windows-10-and-xbox-image-169-super-hero-art"></a>Windows 10 및 Xbox 이미지(16:9 수퍼 히어로 아트)

**16:9 수퍼 히어로 아트(1920 x 1080 또는 3840 x 2160 픽셀)** 이미지는 모든 유형의 Windows 10 장치에서 Microsoft Store의 다양한 레이아웃에 사용됩니다. 앱이 지원하는 OS 버전이나 장치 유형에 관계 없이 이 이미지를 제공하는 것이 좋습니다.

목록에 [비디오 예고편](#trailers)이 포함되어 있는 경우에는 최적의 표시를 위해 이 이미지가 *필수*입니다. Windows 10 버전이 1607 이상인 경우(Xbox 포함), 이 이미지가 Microsoft Store 목록 맨 위의 기본 이미지로 사용됩니다(또는 예고편 재생이 완료된 후 표시). Microsoft Store 전반에서 홍보용 레이아웃으로 앱을 특징짓는 데 사용될 수도 있습니다. 이 이미지에는 제품의 제목이나 다른 텍스트가 포함되어서는 안 됩니다.

다음은 이러한 이미지를 디자인할 때 주의해야 할 몇 가지 팁입니다.

- 이미지는 1920 x 1080 픽셀 또는 3840 x 2160 픽셀인 .png 파일이어야 합니다.
- 앱과 관련한 동적 이미지를 선택하여 인식을 남기고 차별화할 수 있는 방향으로 진행합니다. 상투적인 사진 또는 일반적인 화면 효과를 사용하지 않습니다.
- 이미지에 텍스트를 포함하지 마십시오.
- 이미지의 하단 1/3 부분에 중요한 시각 요소를 배치하지 않도록 합니다(일부 레이아웃의 경우 해당 부분에 그라디언트를 적용할 수 있기 때문).
- 가장 중요한 세부 정부를 이미지 중앙에 배치합니다(일부 레이아웃의 경우 이미지가 잘릴 수 있기 때문).
- 빈 공간을 최소화합니다.
- 앱의 UI를 표시하지 않고, 장치별 이미지를 사용하지 않습니다.
- 정치적이거나 특정 국가의 테마, 국기 또는 종교적 기호를 사용하지 않습니다.
- 민감하지 않은 제스처, 노출, 도박, 통화, 마약, 흡연 또는 음주 관련 이미지를 포함하지 않습니다.
- 무기를 사용자를 향하도록 하거나 과도한 폭력과 유혈을 사용하지 않도록 합니다.

이 이미지를 제공한다는 것은 앱을 추천 홍보 대상으로 고려할 수 있다는 뜻이지만, 앱이 반드시 추천된다고 보증하는 것은 아닙니다. 자세한 내용은 [앱을 판촉하기 쉽게 만들기](make-your-app-easier-to-promote.md)를 참조하세요.


### <a name="xbox-images"></a>Xbox 이미지

앱을 Xbox에 게시하는 경우 적절한 표시를 위해서는 이러한 이미지가 필수입니다. 

업로드 할 수 있는 이미지 크기는 세 가지입니다.
- **브랜드 키 아트, 584 x 800 픽셀**: 제품 타이틀을 포함해야 합니다. 이 이미지에서 브랜딩 표시줄이 필요합니다. 오버레이가 하단 1/4 부분에 나타날 수 있기 때문에 제목과 모든 키 이미지는 이미지의 상단 4/3 부분에 배치합니다.
- **타이틀이 지정된 히어로 하트, 1920 x 1080 픽셀**: 제품 타이틀을 포함해야 합니다. 오버레이가 하단 1/4 부분에 나타날 수 있기 때문에 제목과 모든 키 이미지는 이미지의 상단 4/3 부분에 배치합니다.
- **추천 홍보 스퀘어 아트, 1080 x 1080 픽셀**: 제품 타이틀을 포함해서는 *안 됩니다*.

> [!NOTE]
> Xbox에서 최적의 표시를 위해서는 [Microsoft Store 로고](#store-logos) 섹션에 **9:16 (720 x 1080 or 1440 x 2160 픽셀)** 이미지도 제공해야 합니다.


### <a name="holographic-image"></a>홀로그램 이미지

**2:1(2400 x 1200)** 이미지는 앱이 홀로그램 장치 패밀리를 지원하는 경우에만 사용됩니다. 가능한 경우에는 홀로그램 이미지를 제공하는 것이 좋습니다.

### <a name="images-only-for-windows-8x-andor-windows-phone-8x"></a>Windows 8.x 및/또는 Windows Phone 8.x 전용 이미지 

앱이 이전 OS 버전(Windows 8.x 및/또는 Windows Phone 8.x)을 지원하는 경우, 홍보 레이아웃에서 앱이 추천될 수 있도록 이 이미지를 제공해야 합니다(반드시 앱이 추천되는 것은 아니지만). 앱이 이전 OS 버전을 지원하지 않는 경우에는 이 섹션을 건너뛸 수 있습니다 (이전에는 이 섹션이 **홍보 이미지 옵션**으로 불림).

**Windows Phone 8.1 이전 버전**에서는 홍보 레이아웃에 **1000 x 800 픽셀(5:4)** 및 **358 x 358 픽셀(1:1)** 등 두 가지 이미지 크기를 사용할 수 있습니다. 따라서 앱이 Windows Phone 8.1 이하에서 실행되는 경우 홍보용 이미지로 이러한 두 가지 크기의 이미지를 제공하는 것이 좋습니다.  

> [!TIP]
> Windows Phone 8.1 이전 버전을 지원하는 제출의 경우, [Microsoft Store 로고](#store-logos) 섹션에 300 x 300 앱 타일 아이콘을 제공해야 합니다. 이렇게 해야 Microsoft Store에 앱이 비어 있는 아이콘으로 표시되지 않습니다.  

**Windows 8.1 이전 버전**, 일부 홍보 레이아웃은 **414 x 180** 픽셀 크기의 이미지를 사용할 수 있습니다. 따라서 앱이 Windows 8.1 이하에서 실행되는 경우 홍보용 이미지로 이 크기의 이미지를 제공하는 것이 좋습니다.


## <a name="trailers"></a>예고편

예고편은 고객이 홍보 중인 제품을 확인하고 제품에 대한 이해를 높일 수 있도록 돕는 짧은 동영상입니다. 예고편은 앱의 Microsoft Store 목록 맨 위에 표시됩니다([홍보 이미지](#promotional-images) 섹션에 **1920 x 1080 픽셀 이미지(16:9)** 가 포함되어 있는 경우에 한함). 

예고편은 [부드러운 스트리밍](http://www.iis.net/downloads/microsoft/smooth-streaming)으로 인코딩되기 때문에 사용할 수 있는 대역폭과 CPU 리소스를 기준으로 고객에게 전달되는 비디오 스트림의 품질이 실시간으로 조정됩니다.

> [!NOTE]
> 예고편은 Windows 10, 버전 1607 이상(Xbox 포함)을 사용하는 고객에게만 표시됩니다.

### <a name="upload-trailers"></a>예고편 업로드

Store 목록에 최대 15개의 예고편을 추가할 수 있습니다. 아래 제시된 모든 [요구 사항](#trailer-requirements)을 충족해야 합니다.

각 예고편 별로 비디오 파일(.mp4 또는 .mov), 미리 보기 이미지, 타이틀을 업로드해야 합니다.

> [!IMPORTANT]
> 또한 예고편이 Microsoft Store 목록 맨 위에 나타나도록 하려면 [홍보 이미지](#promotional-images) 섹션에 **1920 x 1080 픽셀 이미지(16:9)** 를 제공해야 합니다. 이 이미지는 예고편 재생이 완료된 후에 나타납니다.

예고편을 효과적으로 만들려면 다음 권장 사항을 적용하세요.
- 예고편은 품질이 좋아야 하고, 길이가 아주 짧아야 합니다(60초 이하, 2GB 미만 권장). 
- 고객이 차이점을 알 수 있도록 각 예고편에 다른 미리보기를 사용하세요.
- 일부 Store 레이아웃은 예고편 맨 위와 맨 아래가 조금 잘려 표시될 수 있기 때문에, 중요 정보는 화면 중앙에 표시되도록 만드세요.
- 프레임 속도와 해상도가 소스 자료와 일치해야 합니다. 예를 들어 720p60으로 촬영한 콘텐츠는 720p60으로 인코딩해 업로드하세요. 

또 아래의 요구 사항을 적용해야 합니다.

**예고편을 사용자 목록에 추가합니다.**
1. 표시된 상자에 예고편 **비디오 파일**을 업로드합니다. 이미 업로드 한 예고편을 다시 사용하기 원할 경우 드롭다운 상자가 표시됩니다(다른 언어의 Microsoft Store 목록 등).
2. 예고편을 업로드 한 후, 여기에 해당되는 **미리 보기 이미지** 를 업로드해야 합니다. 1920x1080 픽셀의 .png 파일이어야 합니다. 일반적으로 예고편의 정지 이미지를 사용합니다.
3. 연필 모양 아이콘을 클릭해 예고편 **제목**을 추가하세요(255자 이하).
4. 목록에 더 많은 예고편을 추가하려면 **예고편 추가** 를 클릭한 후 위 단계를 반복합니다.

> [!TIP]
> 여러 언어로 Store 목록을 만든 경우 **기존 예고편에서 선택**을 선택해 이미 업로드한 예고편을 다시 사용할 수 있습니다. 각 언어마다 따로 업로드할 필요가 없습니다.

목록에서 예고편을 제거하려면 파일 이름 옆의 **X**를 클릭합니다. 현재 작업 중인 Store 목록에서만 제거할지, 아니면 제품의 모든 Store 목록(모든 언어)에서 제거할지 선택할 수 있습니다.


### <a name="trailer-requirements"></a>예고편 요구 사항

예고편을 제공할 때, 다음 요구 사항을 준수해야 합니다.

- 비디오 형식은 MOV 또는 MP4입니다. 
- 비디오 길이는 60초를 초과할 수 없습니다.
- 예고편 파일 크기는 2GB 이하입니다. 
- 비디오 해상도는 1920 x 1080 픽셀 또는 3840 x 2160 픽셀이어야 합니다.
- 미리보기는 1920 x 1080 픽셀 또는 3840 x 2160 픽셀 해상도의 PNG 파일이어야 합니다.
- 제목은 255자 이하입니다. 
- 예고편에 연령별 등급을 포함하지 마세요.

Microsoft Store 목록 페이지의 다른 필드와 마찬가지로, 승인을 받은 예고편만 Microsoft Store에 게시할 수 있습니다. 예고편이 [Microsoft Store 정책](https://docs.microsoft.com/legal/windows/agreements/store-policies)을 준수하는지 확인하세요.
