# <a name="developer-home-on-the-console-dev-home"></a>콘솔 (개발자 홈)에 대 한 개발자 홈
   
  
개발자 홈 개발자의 생산성을 지원 하도록 설계 된 Xbox 하나 개발 키트에서 도구 환경입니다. 개발자 홈 캡처하고 추적 관리 및 개발 키트를 구성, 사용자 관리, 실행에 설치 된 제목 및 수행 하는 기능을 제공 합니다. 나중에 보도 자료 여러분의 의견에 따라 추가 기능을 활성화 하 고 확장성 및 사용자 고유의 도구 추가 하도록 설정 하는 기능을 확장 하는 동안 계속 될 예정입니다.   
   
  
개발자 홈 및 지원 하기에 가장 관심 하는 시나리오에 대 한 사용자 의견에 매우 관심이 있습니다. 응용 프로그램의 주 메뉴에 **사용자 의견 보내기** 아래에서 설명 하는 방법 또는 통해 대화 개발자 계정 관리자 (댐을)를 통해 의견을 입력 하십시오.   
   
  
년 11 월 2015에 개발 집 또는 나중에 복구를 시작 합니다.  
 
   1. 집에서 왼쪽 또는 원활한 연결 단추를 클릭 하면 이중 이동 하 여이 가이드를 엽니다.  
   1. **설정** (기어 아이콘)을 아래로 이동 합니다.   
   1. **모든 설정** 을 선택 합니다.  
   1. 기본 **개발자** 페이지에서 **개발자 Home** (홈 아이콘)를 선택 합니다.   

 ![](images/dev_home_icons.png)   
  
이전 복구에 **콘텐츠를 갖춘** 에서 홈 화면 오른쪽에 개발 홈 타일을 선택 또는 Xbox 한 명의 관리자에서 응용 프로그램 목록 보기 및 **개발 홈**을 실행 합니다.   
 ![](images/dev_home_1.png)

## <a name="user-interface"></a>사용자 인터페이스  
   
  
개발 홈 사용자 인터페이스의 헤더 "한눈" 중요 한 다음이 포함 개발 콘솔에 대 한 정보:   
 
   *  **IP 콘솔:** 콘솔의 현재 IP 주소를 지정 합니다.   
   *  **콘솔 이름:** 콘솔의 현재 호스트 이름입니다.  
   *  **샌드박스:** 콘솔에 있는 샌드박스의 이름입니다.  
   *  **운영 체제 버전:** 콘솔에서 실행 되는 현재 복구 버전입니다.
   *  현재 시스템 시간입니다.   

   
  
홈 개발 UI의 나머지 부분에는 다음 페이지 나뉩니다. 이러한 페이지에 있는 도구에 대 한 자세한 내용은 자신의 개별 항목을 참조 하십시오.

## <a name="main-menu"></a>주 메뉴

## <a name="snap-mode-ux"></a>모드 UX 맞추기  
   
  
개발자 홈 네트워킹 및 멀티 플레이어, 등의 다양 한 기존 및 예정 된 도구에 맞춰 지 옆면에 제목을 실행 하는 동안 테스트 하는 동안에 도구에 쉽게 액세스할 수를 볼 수 있도록 사용 하도록 설계 됩니다.   
   
  
맞춤 모드에 액세스 하려면 적합 한 도구의 제목을 강조 표시 하 고 컨트롤러에서 **보기** 단추를 눌러 상황에 맞는 메뉴에서 **맞춤** 을 선택:

개발자 홈이 오른쪽에 맞춰집니다. 평소대로 Nexus 단추를 두 번 탭하여 컨텍스트를 전환할 수 있습니다.

## <a name="customizing-dev-home"></a>개발자 홈 사용자 지정  
   
  
개발자 홈은 사용자 지정 가능하고 개인적으로 설계되었습니다. 워크플로에서 맞게 응용 프로그램을 구성할 수 있으며 다음 작업 영역으로 저장 하는 수 있습니다. 이 작업 영역을 내보낼 수 있습니다 하 고 가져온, 수 있도록 다음을 필요한 경우 레이아웃으로 다른 콘솔을 복사할 수 있습니다. 이러한 옵션은 **작업 영역**에서 주 메뉴에 있습니다. 내보낸된 파일에서 스크래치 시스템 드라이브에 배치할 수는 `Dev Home\Workspaces` 디렉터리입니다.   
 
<a id="ID4EVF"></a>

   

### <a name="resizing-and-reordering-tools"></a>크기를 조정 하 고 도구를 다시 정렬  
   
  
크기 또는 도구의 위치를 변경 하려면 제목 하는 동안 상황에 맞는 메뉴 단추 (컨트롤러에서 보기 단추)를 사용 하 여 포커스가 키를 누릅니다. 상황에 맞는 메뉴에서 **이동** 하거나 **크기를 조정**합니다.를 선택 합니다.   
 ![](images/dev_home_6.png)  
<a id="ID4EEG"></a>

   

### <a name="changing-theme-color-and-background-image"></a>테마 색 및 배경 이미지 변경  
   
  
주 메뉴에서 **작업 영역** 및 **변경 테마 색**을 선택할 수 있습니다. 새 색을 선택 하 고 포커스를 강조 표시에 사용 되는 테마 색을 업데이트 하려면 **저장** 을 선택 합니다.   
 ![](images/dev_home_7.png)  
<a id="ID4EVG"></a>

   

### <a name="setting-the-default-application-for-a-package"></a>패키지에 대 한 기본 응용 프로그램 설정  
   
  
패키지를 여러 응용 프로그램을 포함 하는 경우 개발 홈은 설정할 수 있도록 기본 응용 프로그램을 시작할 수 있습니다. 시작 관리자에서 패키지를 강조 표시 하 고 사용 가능한 응용 프로그램의 목록을 열려면 **A** 단추를 누릅니다. 기본으로 설정 하 고 **보기** 단추를 눌러 후 상황에 맞는 메뉴에서 **기본 설정으로** 선택 하려면 1을 강조 표시 합니다.   
 ![](images/dev_home_setdefault.png)  
<a id="ID4EGH"></a>

   

### <a name="using-dev-home-to-register-and-launch-titles-from-a-network-share"></a>개발자 홈을 사용 하 여 등록 하 고, 네트워크 공유에서 제목을 실행 하려면  
   
  
설치 된 앱 및 게임 목록 맨 아래에 시작 관리자에서 제목을 느슨한 파일 버전을 원격으로 실행 하도록 **네트워크 공유에서 게임을 등록** 하는 옵션을 선택할 수 있습니다.   
 ![](images/dev_home_8.png)   
  
그런 다음 appxmanifest.xml 파일을 등록 하려는 제목에 대 한 네트워크 경로 입력할 수 있습니다. 개발자 홈은 해당 공유에 대 한 모든 기존 자격 증명을 사용 하 여 제목 등록 하려고 한 경우에는 새 네트워크 자격 증명을 확인 필요 합니다. 추가 공유 (예:를 별도 서버에서 리소스에 연결 된 기호로 액세스)에 액세스 해야하는 경우 아래에서 옵션을 통해 이러한 추가 해야 합니다.   
   
  
이러한 저장 된 자격 증명 관리 (추가 하는 추가 선언할) 주 메뉴의 **관리 네트워크 자격 증명** 옵션을 통해 콘솔에 있습니다.   
 ![](images/dev_home_9.png)   
  
콘솔에 현재 자격 증명을 보고 하 고, 자격 증명의 경로 선택 하 고 **A** 단추를 클릭 하 여 자격 증명을 편집 하 고, 제거 링크를 선택 하 고 **A** 단추를 클릭 하 여 자격 증명을 제거할 수 있습니다.
