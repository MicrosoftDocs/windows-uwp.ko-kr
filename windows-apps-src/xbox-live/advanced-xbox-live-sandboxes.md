# <a name="advanced-xbox-live-sandboxes"></a>고급 Xbox Live 샌드박스

> **참고** 이 문서 샌드박스의 고급 사용법을 설명 하며이 주로 큰 게임 스튜디오 여러 팀 및 복잡 한 사용 권한 요구 사항에 적용 됩니다.  Xbox Live 크리에이터 스 프로그램의 일부인 경우 또는 ID@Xbox [Xbox Live 샌드박스 소개](xbox-live-sandboxes.md) 를 살펴보는 것이 좋습니다 개발자

Xbox Live *샌드박스* 를 개발에 대 한 전체 개인 환경을 제공합니다. 이 문서는 샌드박스 이란, 존재 이유, 게시자에 적용 되는 방식 및 Xbox 가속화할 어떻게 영향을 설명 합니다. 이 문서의 대상 사용자는 게시자가 Xbox One 콘텐츠를 빌드하고 샌드박스를 사용 합니다.

## <a name="summary"></a>요약

Xbox Live에서 모든 시험판 (개발 중인 및 베타), 인증 및 콘텐츠 소매 있는 단일 프로덕션 환경만 있습니다.

콘텐츠 격리는 프로덕션 환경에서 게시자 콘텐츠 누출 되지 않았는지 확인 하는 방법. 기본적으로 콘텐츠 격리 모든 계정 사용자, 장치 또는 제목 (제목 또는 서비스) 리소스 액세스 권한을 요청 하는 리소스에 액세스 권한이 보장 합니다. 콘텐츠 격리를 사용 하 여 파티션은 제목이 나 서비스 데이터를 저장 하는 샌드박스 나뉩니다. 권한 부여 정책 즉, 샌드박스 범위 내에서 정의 됩니다.

샌드박스는 프로덕션 환경에 있는 데이터를 분할 하는 방법. Xbox 360 연대 서비스 PartnerNet 및 ProductionNet 두 가지 환경 됩니다. Xbox One 연대 서비스를 사용 하 여 단일 프로덕션 환경에 *n* 고유한 가상 환경을 있는 각 가상 환경 샌드박스 라고 포함 되어 있습니다. 모든 콘텐츠는 단일 프로덕션 환경 이기 때문에 샌드박스 환경에서 생성 된 데이터를 통해 교차 수 없는 위치 실제로 고유한 가상 환경 됩니다.

다음 그림은 게시자는 전용 개발 샌드박스를 만들 수 있는 단일 프로덕션 환경입니다. 유일한 권한이 있는 개발자 계정 또는 개발자 키트 이러한 샌드박스에 대 한 액세스를 허용 됩니다.

## <a name="sandboxes"></a>샌드박스

<a name="what-is-a-sandbox"></a>샌드박스 란 무엇 인가요?
------------------

언급 했 듯이, *샌드박스는 프로덕션 환경에서 데이터를 분할 하는 방법*.

<a name="why-do-we-need-sandboxes"></a>샌드박스 필요한 이유는 무엇입니까?
-------------------------

사용자 및 디바이스 타이틀에 액세스 하는 것 처럼 제목 서비스에 액세스 합니다. 타이틀 집합이 서비스 리소스에 대 한 액세스 권한이 부여 된 "제목 그룹"의 개념을 소개 합니다.

모든 콘텐츠 (시험판 및 일반 정품)에 대 한 Xbox One 용 단일 프로덕션 환경 이기 때문에 타이틀의 여러 인스턴스 (사전 시험판/소매)에서 리소스의 동일한 인스턴스에서 작동 하지 해야 합니다.

<a name="what-is-in-a-sandbox"></a>샌드박스에서 란?
---------------------

샌드박스는 샌드박스 추가 된 모든 제목에 대 한 제품 인스턴스를 포함 합니다.

<a name="what-is-a-sandbox-id"></a>샌드박스 ID 란?
---------------------

샌드박스 ID는 분할 단위 제목, 제품 또는 서비스 구성에 대 한 데이터입니다. 모든 서비스 구성 데이터를 공유할 필수 구성 요소는 동일한 샌드박스에서 제목을 여러 개 있을 수 있습니다.

샌드박스 ID (대/소문자 구분)가는 다음 형식의 문자열: &lt;PublisherMoniker&gt;. *n*. 예: 샌드박스 ID, XLDP.5, 아래에서 설명 합니다.

-   *게시자 모니커* 은 모든 게시자 간에 고유 합니다. 따라서 "XLPD"이 특정 게시자에 대 한 게시자 모니커입니다. 게시자 모니커 게시자 활성화 될 때"" XDP에서 개발자 계정 관리자에 의해 만들어집니다.

-   숫자 *"n"* 샌드박스의 번호를 식별 합니다. 이 경우이 게시자에 대해 만든 여섯 번째 샌드박스 "5"입니다.

제목 채운 서비스를 통해 Xbox 서비스 생성 되는 데이터에 대 한 "환경"을 고유 하 게 식별 하 샌드박스 ID를 사용 합니다.

<a name="global-override-sandbox"></a>글로벌 재정의 샌드박스
-----------------------

개발자가 개발 키트의 샌드박스 ID를 설정 하 고 따라서 개발 키트;에서 실행 되는 샌드박스를 설정 이 전역 재정의 샌드박스 라고도 합니다. Xbox Live 서비스 (예를 들어, 도전 과제, 매치 메이 킹 라이선싱 EDS, 등)에 대 한 따라서 모든 요청 제목 (셸 응용 프로그램 및 일반 앱) 개발자에서 키트 해당 샌드박스에서 이루어집니다.

전역 재정의 샌드박스 검색 되 면 전역 재정의 샌드박스에서 수집 내용만 표시 되는지 의미 합니다.

<a name="types-of-sandboxes"></a>샌드박스 유형
-------------------------------------------------------------------------------------------------------------------------------------------------------

두 개의 서로 다른 범주의 샌드박스 있습니다. 이러한 범주는 다음과 같이 정의 됩니다.

-   *게시자 샌드박스*합니다. 게시자의 내부 개발 샌드박스를 액세스할 수 있습니다. 이러한 XLDP.0, XLDP.1, XLDP.2, XLDP.3 등과 같이 보일 수 있습니다. 이 여기서 게시자에 게 제목 제품 인스턴스를 넣었습니다. 이러한 샌드박스에 액세스에 대 한 액세스를 게시자 부여 사용자/디바이스 여부가

-   *Microsoft 샌드박스*합니다. 다음은 기본 제공 샌드박스: 소매 및 인증서만 Microsoft에 이러한 보호 된 샌드박스를 게시할 수 있습니다.

<a name="cert-sandbox"></a>인증서 샌드박스
------------

제목을 일반 가용성에 대 한 준비 되 면 먼저 인증을 통과 해야 합니다. 인증서 샌드박스 개인 인증에만 액세스할 수 있는 Microsoft 제어 샌드박스가입니다. 게시자 인증 통과 사용자가 콘텐츠를 볼 수 있습니다.

인증에 실패 하는 모든 제품 인스턴스를 디버그 하 고 XDP 또는 Windows 개발자 센터를 사용 하 여 게시자로 개발 샌드박스를 다시 가져올 수 있습니다.

<a name="retail-sandbox"></a>소매 샌드박스
--------------

소매 샌드박스 Xbox One에 대해 만들어진 모든 콘텐츠에 대 한 최종 대상입니다.

타이틀 인증을 통과 하면 소매 샌드박스에 추가 됩니다. 녹색 서명 된 콘텐츠만 소매 샌드박스에서 실행을 허용 합니다. 이 게시자 구동 베타 소매 샌드박스에서 수행한도 중요 한 구현 됩니다. 소매 샌드박스 내에서 생성 된 데이터를 실제 고객 프로덕션 데이터를 나타냅니다.

콘텐츠에 대 한 액세스는 소매 샌드박스 여전히 콘텐츠 격리를 통해 제어할 수입니다.

예를 들어 게시자 구동 베타 게시자 베타 게시자 정의 된 리소스 제목 설정에 액세스할 수 있는 사용자 그룹 가져오기 하도록 선택 소매 샌드박스에서 실행 됩니다. 베타 타이틀에서 생성 된 서비스 데이터 실제 데이터를 프로덕션 이며 계속 제목 일반 공급으로 이동 되 면 존재 합니다.

<a name="cross-sandbox-data-interaction"></a>샌드박스 간 데이터 상호 작용
------------------------------

기본적으로 샌드박스는 데이터 공유를 제한 하는 컨테이너를 사용 합니다. 따라서 간 샌드박스 데이터 상호 작용이 불가능 합니다.

## <a name="organizing-your-sandboxes"></a>에 샌드박스를 구성합니다.

이 섹션에서는 게시자 수 샌드박스를 구성 하는 방법의 예를 제공 합니다. 게시자는 샌드박스를 사용 하 여 데이터를 구성 하는 방법을 이해 해야 합니다.

**참고** 콘텐츠 격리를 사용 하 여 런타임 액세스 관리만 표시 아래 예제입니다.

### <a name="scenario-1-two-titles-one-sandbox"></a>시나리오 1: 두 제목, 하나의 샌드박스

게시자의 기본 구조는 다음과 같습니다.

-   모든 사용자와 디자인 타임과 런타임 모두에 대 한 게시자가 소유 하는 장치에 액세스할 수 있는 두 개의 제목을 합니다.

-   제목 당 하나의 제품 인스턴스입니다.

이 인스턴스에 있는 게시자 모든 시험판 콘텐츠에 대 한 단일 샌드박스만 필요합니다.

아래 다이어그램에는 사용자 그룹을 보여 줍니다. 게시자는 쉽게 것으로 간주 하는 경우 사용자 그룹을 대신 장치 그룹을 사용 하도록 선택할 수 있습니다. 또한이 사용자 그룹이이 샌드박스에 샌드박스 XLDP.1 및 제목에 대 한 디자인 타임 및 런타임 액세스 합니다.

![](images/sandboxes/sandboxes_image6.png)

### <a name="scenario-2-one-title-different-teams"></a>시나리오 2: 하나 제목, 다른 팀

이 모델에서는 요구 사항은.입니다.

-   하나의 제목입니다.

-   개발자 팀이 매일 빌드 작업 합니다.

-   QA 팀 주간 LKGs에서 작동합니다.

-   개발자 팀 버그 발생 하는 경우 주간 LKGs 디버그 해야 합니다.

-   재무 팀 가격 카드 및 제목 카탈로그 릴리스 관련 된 기타 메타 데이터에 액세스를 해야 합니다.

아래 그림과 TitleX에는 두 제품 인스턴스: PI 1과 PI 2. 제품 인스턴스 샌드박스에서 수 있으며 동일한 샌드박스 내에서 두 제품 인스턴스 동일할 수 없습니다. 샌드박스에서 XLDP.1 TitleX-PI-1은 이므로 샌드박스 XLDP.2 TitleX PI 2 요소 이며 합니다.

개발자 사용자 그룹 QA 사용자 그룹에 샌드박스 XLDP.2에 대 한 액세스 하는 반면 두 샌드박스를 액세스할 수 있습니다.

또한 finance 사용자 (그룹 C)에 디자인 타임 TitleX에 액세스할 수 있습니다. 재무 사용자 그룹은 일반적으로 제목 런타임 디버깅, 수행 하지 않으므로 구분 됩니다.

**참고** 조직에 관계 없이 XDP 사용자는 둘 이상의 사용자 그룹에 속할 수 있습니다.

![](images/sandboxes/sandboxes_image7.png)

### <a name="scenario-3-two-titles-completely-separate"></a>완전히 별개의 시나리오 3: 두 제목

이 예제에서는 요구 사항을 약간 변경합니다.

-   두 개의 제목을 합니다.

-   각 제목에 대 한 액세스는 개인의 특정 집합으로 제한 해야 합니다.

-   제목 당 하나의 제품 인스턴스입니다.

-   제목에 대 한 디자인 타임 XDP 구성 데이터에 액세스 해야 하는 관리자 사용자 그룹. 이 그룹의 개인 게시자에 대 한 모든 관리자 되며 (카탈로그 메타 데이터, 마케팅 등, 인증 제출 재무) 카탈로그에 게시 된 모든 데이터를 제어할 수 있습니다.

이 모델에서는 게시자 모두 제목 완전히 분리 하 고 따라서 이러한 두 타이틀 두 개의 다른 샌드박스에 할당 된 상태로 선택 했습니다. 또한 게시자 별도 관리자 사용자 그룹을 만들려면 선택 하 고 두 제품에 할당 된 액세스 했습니다.

### <a name="scenario-4-anyway-you-like-it"></a>시나리오 4: 그래도 원합니다

이 예제에서는 요구 사항은.입니다.

-   특정 사용자만에 해당 게시자 내에서 특정 제목에는 액세스할을 수 있습니다.

-   게시자 다른 회사의 공급 업체와 함께 작동 하 고 이러한 공급 업체 단기 있을 수 있습니다.

-   게시자는 타이틀을 제거 하 고이 작업을 수행 하 여 공급 업체 또는 Fte 적에 액세스할 수 있는 모든 데이터에 대 한 액세스를 방지 하려면 수 있어야 합니다.

이 요구 사항은 모델을 하려면 아래와 같은 구조를 사용할 수 있습니다.

아래 뒤에 모델은입니다.

-   TitleX 및 TitleY 인스턴스가 하나만 있는 제품 각 샌드박스에 XLDP.1 합니다.

-   TitleZ 두 제품 인스턴스 하나 샌드박스에서 XLDP.2이 고 다른 샌드박스에서 XLDP.3에 있습니다.

-   모든 샌드박스 제품 인스턴스의 FTE 사용자 그룹 B 액세스를 부여 됩니다.

-   공급 업체 사용자 그룹 A XLDP.1 샌드박스에 대 한 액세스를 제공 되는 공급 업체 전용 사용자 그룹입니다.

-   공급 업체 디바이스 그룹 C XLDP.3 샌드박스에 대 한 액세스를 제공 되는 공급 업체 전용 사용자 그룹입니다.

## <a name="summary"></a>요약

Xbox Live 개발 서비스 품질 프로덕션 및 프로덕션 MSA 개발자 계정 프로덕션 환경에서 테스트 하는 게시자에 엄청난 기회를 제공 합니다. 기능 및 유연성 증가 제목 데이터를 만들고 관리 하는 동안 타이틀 개발 및 일반 가용성에 대 한 액세스를 XDP에서 새 구성 단계에 필요 합니다.

샌드박스는 프로덕션 환경에서 데이터를 분할 하는 방법입니다. 모든 콘텐츠는 단일 프로덕션 환경 이기 때문에 샌드박스를 하나의 환경에서 생성 된 데이터 거치지 않기를 통해 다른 "가상 환경"으로 작동 합니다.
