# <a name="multiplayer-scenarios"></a>멀티 플레이 시나리오
다양 한 유형의 멀티 플레이어 시나리오 및 올바른 시나리오 선택 플레이어 engagement 늘릴 수 있으며 플레이어에 도움이 되는 게임에 대 한 기본 확장 최대한 오랫동안 게임의 활성화 수명.

순위표, 통계 또는 소셜 요소와 같은 경쟁 Xbox Live 기능에서 주로 단일 플레이어 환경을 게임도 도움이 됩니다.

다음은 Xbox Live를 사용한 구현할 수 있는 보다 일반적인 멀티 플레이 시나리오 중 일부를 설명 합니다. 목록을 구현 하 고 테스트 하는 데 필요한 작업의 양과 복잡성 측면에서 정렬 및 게임을 지원 하려는 형식 시나리오를 결정할 때 이러한 요소를 고려해 야 합니다.

## <a name="comparative-indirect-play"></a>비교 (간접) 재생
이 시나리오에서는 플레이어 경쟁 하는 하지 서로 직접 게임의 동일한 인스턴스에서 직접 게임 플레이 하지 않고 있습니다. 단일 플레이어 환경을 지향 하지만 다른 플레이어 이전과 어떻게 비교 플레이어 수 있는 일부 경쟁 측면을 포함 하는 게임에도 적합 합니다.

이 시나리오에 대 한 예제 기능은 다음과 같습니다.

* 순위표, 여기서 게이머를 달성 하려고 모범 "score" 범주의 다른 플레이어를 기준으로 합니다. 만 친구 들과 경쟁에 대 한 소셜 순위표 시스템을 포함할 수 있습니다.
* 성과 및 여기서 퇴근 하고자 그의 친구의에 대 한 자신의 진행률/성능을 비교할 수 및 가능한 경우 brag 박 특히 까다로운 달성 하는 방법에 대 한 통계입니다.
* "영상" 또는 "가상 다중 접속", 여기서 퇴근 경쟁할 수 있습니다 다른 게이머의 기록에 대 한 (또는 자신의) 레이싱 게임의 목표와 같은 이전 성능.

이 유형의 다중 접속 다음 Xbox Live 서비스를 사용 하 여 수행할 수 있습니다.

* 상태
* 통계
* 주민 등록 관리자
* 도전 과제
* 순위표
* 연결 된 저장소

이 유형의 다중 접속 특정 Xbox Live 멀티 플레이 서비스 필요 하지 않습니다. 이 시나리오를 테스트 해야 여러 Xbox Live 계정에만 해당 합니다.

## <a name="local-play-living-room-play"></a>로컬 Play (재생 거실)
이 유형의 다중 접속 게임을 함께 실행 (서로 비교 또는 협조적으로) 두 개 이상의 플레이어를 기반으로 단일 장치에서. 제목을 각 플레이어에 대 한 모든 플레이어 또는 분할 화면 환경에 대 한 단일 화면을 사용할 수 있습니다. 또는 턴 기반 게임을 사용할 수 있습니다 "핫 사용자" 방법은 각 플레이어는 해당 설정 하는 동안 게임의 제어를 사용 합니다.

Xbox One 콘솔에서 단일 콘솔에서 여러 게이머 서명할 수 있습니다. 각 플레이어는 컨트롤러에 연결 됩니다. 현재 Windows 10 장치는 단일 Xbox Live 계정을의 로그인만 지원 하지만 Microsoft는 향후 업데이트에서 변경 하는이 조사 중입니다.

Xbox Live 멀티 플레이 서비스를 사용 하 여 멀티 플레이어의 로컬 play만 스타일을 디자인할 수 있지만, 더 나은 추가 작업으로 온라인 환경을 통합 하는 더 포괄적인 멀티 플레이어 시나리오의 하위 집합으로이 시나리오를 고려해 야 할 수 있습니다. 필요한 투자 멀티 플레이어 시나리오를 확장의 잠재적인 반환 값에 비해 최소화 됩니다.

이 유형의 다중 접속 이전 시나리오와 유사한 서비스를 따르면 됩니다.

* 상태
* 통계
* 주민 등록 관리자
* 도전 과제
* 순위표
* 연결 된 저장소

이 시나리오를 테스트에 여러 Xbox Live 계정 및 단일 장치에서 컨트롤러가 여러 개 필요 합니다.

##  <a name="online-play-with-friends"></a>친구 들과 온라인 게임
이 시나리오는 대부분의 기존 온라인 멀티 플레이 환경입니다. 이 시나리오에서는 Xbox Live 멤버를 친구 들만을 사용 하 여 게임을 실행 하려고 하지만 싫은 재생 하지 않습니다. 친구 초대 하거나 또는 진행 중 이므로 지속적으로 게임을 조인 합니다.

자녀에 대 한 광범위 한 멀티 플레이 제목 (같이 나중에)만 친구에 게 온라인 다중 접속을 제한 하 고이 시나리오를 대체할 수가 있어야 합니다. 자녀 모르는 사람과 상호 작용을 제한 하는 Xbox Live 서비스를 통해도 적용 됩니다.

이 유형의 다중 접속 다음 Xbox Live 서비스를 사용 하 여 수행할 수 있습니다.

* 멀티 플레이어 관리자
* 상태
* 통계
* 주민 등록 관리자

이 시나리오를 테스트에 여러 장치와 Xbox Live 계정을 여러 개 필요 합니다.

이 시나리오에는 플레이어를 게임 세션을 직접 선택할 수 있습니다. 플레이어를 제어할 이지만 세션을 적절 한 환경을 제공 하는 보장 되지 않습니다. 흥미로운 기술 수준에서 설정 하는 올바른 플레이어를 사용 하 여 세션을 채울 수 있습니다. 세션 목록 게임 작은 active 멀티 플레이 커뮤니티 경우 가장 효과적입니다.

## <a name="online-play-though-looking-for-group"></a>하지만 그룹을 찾고 온라인 게임

## <a name="simple-matchmaking"></a>간단한 매치 메이 킹
이 시나리오에서는 플레이어 (또는 그룹의 플레이어) 찾고 (플레이어에 게 알 수 없는) 다른 플레이어 온라인 게임에 대 한 합니다. 친구를 선택 하는 대신 게임이 다른 플레이어를 사용 하 여 게임 세션에 참가할 수 있도록 간단한 매 치메이 킹 흐름을 제공 합니다. 이 시나리오의 결혼 정보 회사 연결 흐름 간단: 플레이어 검색 하 고 중요 한 매 치메이 킹 구성 없이 다른 플레이어를 찾습니다. 이 시나리오는 큰 온라인 대상 그룹을 사용 하 여 가장 적합합니다.

를 일치 하는 플레이어 함께 적절 하 게 모든 결혼 정보 회사 평판을 인식 하 고 Xbox Live 목록 차단 하려고 해야 합니다. Xbox Live SmartMatch 서비스는 이러한 제한 사항은 자동으로 처리합니다. 이러한 도움을 안전 하 게 확인 하 고 더 나은 플레이어가 환경을 같은 제한에 따라 합니다.

결혼 정보 회사의 중요 한 부분 서비스 품질 (QoS) 네트워킹 검사 됩니다. 이러한 검사 두 플레이어 간에 네트워크 연결을 적절 한 게임 플레이 환경을 위해 충분 한지 확인 합니다. 이 매 치메이 킹을 반복 하 고 잘못 된 네트워크 연결 시 다른 플레이어를 찾을 수 있기 때문에 친구 시나리오를 사용 하 여 온라인 재생에서 다릅니다.
