# 개발자 센터 대시보드에서 실험 정의

A/B 테스트로 UWP(유니버설 Windows 플랫폼) 앱에서 실험을 실행하려면 개발자 센터 대시보드에서 실험을 정의하는 작업부터 시작합니다.

다음 섹션에서는 대시보드에서 실험을 정의하는 일반적인 프로세스를 설명합니다. 실험 만들기 및 실행의 종단 간 프로세스를 보여 주는 연습에 대한 자세한 내용은 [A/B 테스트로 첫 번째 실험 만들기 및 실행](create-and-run-your-first-experiment-with-a-b-testing.md)을 참조하세요.

## API 키 가져오기

시작하려면 개발자 센터 대시보드의 **실험** 페이지로 이동하여 실험의 *API 키*를 가져옵니다.

API 키는 앱을 개발자 센터 계정의 실험과 연결하는 고유 ID입니다. 각 실험은 정확히 하나의 API 키와 연결됩니다. 그러나 앱은 여러 API 키를 가질 수 있으며 각 API 키는 여러 실험과 연결될 수 있습니다. API 키를 사용하여 다른 실험 집합 간을 구분할 수 있습니다. 예를 들어 조직의 테스터에게 릴리스한 하나의 실험 집합이 있고 앱의 외부 사용자에게만 릴리스한 다른 실험 집합이 있을 수 있습니다.

이 API 키를 사용하여 앱 코드의 A/B 테스트 서비스와 연결해야 합니다. 자세한 내용은 [실험용 앱 코딩](code-your-experiment-in-your-app.md)을 참조하세요.

1. [개발자 센터 대시보드](https://dev.windows.com/overview)에 로그인합니다.
2. **앱**에서 실험을 만들 앱을 선택합니다.
3. 탐색 창에서 **서비스**를 선택한 다음 **실험**을 선택합니다.
4. **API 키** 섹션은 **API 키 #1**이라는 앱에 대한 기본 API 키를 제공합니다. 이 키를 사용하려는 경우 선택적으로 이 키에 식별 이름을 입력하고 복사해서 앱 코드에서 사용할 수 있습니다. 새 API 키를 생성하려면 **새 API 키** 이름을 선택하고 API 키의 식별 이름을 입력합니다.

## 실험 만들기

다음으로 새 실험을 만들고 실험의 목표를 정의합니다.

1. **실험** 섹션에서 **새 실험** 단추를 클릭합니다.
2. **API 키 이름** 섹션에서 이 실험과 연결할 API 키를 선택합니다. 하나의 API 키만 있는 경우 기본적으로 해당 API 키가 선택됩니다.
3. **실험 이름** 필드에서 실험을 쉽게 식별하는 데 사용할 수 있는 이름을 입력합니다. 실험을 만든 후 이 이름은 **실험 페이지**에서 초안, 활성 및 완료된 실험 목록에 표시됩니다.
4. 테스트 실험을 만들려는 경우 **테스트 실험** 확인란을 클릭합니다. 테스트 실험과 정규 실험 간 차이점은 테스트 실험이 활성화된 후 변경할 수 있다는 것입니다.

  테스트 실험은 고객에게 실험을 릴리스하기 전 클라이언트 디바이스에서 모든 변형을 테스트할 수 있도록 하기 위한 것입니다. 변형이 예상한 대로 클라이언트에게 제공되는지 확인하기 위해 한 변형에는 100% 배포를 할당하고 다른 변형에는 0%를 할당하여 테스트 실험을 활성화할 수 있습니다. 이 변형을 확인한 후 다른 변형에 대한 프로세스를 반복할 수 있습니다.
  > **참고** 내부 테스트를 통해 매개 변수의 유효성을 검사할 테스트 실험을 만드는 경우에만 이 확인란을 선택합니다. 고객에게 릴리스할 실험을 만들 경우 이 확인란을 선택하지 마세요.

5. **보기 이벤트 이름** 필드에서 실험에 대한 *보기 이벤트*의 이름을 입력합니다. 보기 이벤트는 사용자가 실험의 일부인 변형을 보기 시작할 때 작업을 나타내는 임의의 문자열입니다. 앱 코드는 사용자가 변형을 보기 시작할 때 개발자 센터에 이 보기 이벤트 문자열을 전송합니다. 자세한 내용은 [실험용 앱 코딩](code-your-experiment-in-your-app.md)을 참조하세요.
6. **목표와 전환 이벤트** 섹션에서 하나 이상의 실험 목표를 정의합니다.
  * **목표 이름** 필드에서 목표의 설명이 포함된 이름을 입력합니다. 실험을 실행한 후 이 이름은 실험에 대한 결과 요약에 표시됩니다.
  * **전환 이벤트 이름** 필드에서 이 목표에 대한 *전환 이벤트*의 이름을 입력합니다. 전환 이벤트는 이 목표에 대한 목표를 나타내는 임의의 문자열입니다. 앱 코드는 사용자가 목표에 도달할 때 개발자 센터에 이 전환 이벤트 문자열을 전송합니다. 자세한 내용은 [실험용 앱 코딩](code-your-experiment-in-your-app.md)을 참조하세요.
  * **목표** 필드에서 전환 이벤트의 발생 횟수를 최대화 또는 최소화할지 여부에 따라 **최대화** 또는 **최소화**를 선택합니다. 이 정보는 실험 결과 요약에 사용됩니다.

  >**참고** 개발자 센터에서는 24시간 내의 각 사용자 보기에 대한 첫 번째 전환 이벤트만 보고합니다. 사용자가 24시간 내에 앱에서 여러 전환 이벤트를 트리거하는 경우 첫 번째 전환 이벤트만 보고됩니다. 이렇게 하면 목표가 전환을 수행하는 사용자의 수를 최대화하는 것일 때 단일 사용자가 샘플 그룹 사용자의 실험 결과를 왜곡시키지 않을 수 있습니다.

## 변형 및 실험 설정을 정의합니다.

다음으로 변형 및 실험 설정을 정의합니다.

* *변형*은 실험에서 테스트할 하나 이상의 설정 컬렉션입니다. 모든 실험에는 최소 한 개의 설정과 두 개의 변형(컨트롤 포함)이 있어야 합니다. 하나의 실험에는 최대 다섯 개의 변형이 있을 수 있습니다.
* *설정*은 프로그램 변수를 초기화할 때 앱에서 사용하는 값입니다. 실험 중 설정 값은 여러 변형으로 변경됩니다. 실험을 종료하면 앱의 모든 사용자에게 릴리스하도록 선택한 변형의 값이 설정에 할당됩니다. 설정에는 문자열, 부울, 이중 및 정수 유형이 있습니다.

실험에 사용할 변형 및 설정을 정의하려면
1. **변형 및 설정** 섹션에는 두 가지 기본 변형인 **변형 A(컨트롤)** 및 **변형 B**가 표시됩니다. 더 많은 변형을 사용하려면 **변형 추가**를 클릭합니다. 선택적으로 각 변형의 이름을 바꿀 수 있습니다.
2. 그다음 변형에 대한 설정을 만듭니다. **설정 추가**를 클릭하여 각 새 설정을 만들고 각 변형의 설정 이름 및 설정 값을 입력합니다.
3. 기본적으로 변형은 앱 사용자에게 고르게 배포됩니다. 특정 배포 비율을 선택하려는 경우 **고르게 분배** 확인란을 선택 취소하고 **분포(%)**에 백분율을 입력합니다.

## 실험 저장

실험에 대한 필수 필드 입력을 마치면 **저장**을 클릭하여 실험을 저장합니다.

> **중요** 실험을 저장하면 아직 실험을 활성화하지 않은 경우에도 실험에 대한 API 키를 더 이상 변경할 수 없습니다.

원하는 실험 매개 변수를 설정하고 활성화할 준비가 완료되어 앱에서 실험 데이터 수집을 시작할 수 있으면 **활성화**를 클릭합니다. 실험이 활성화된 경우 앱은 변형 설정을 검색하고 개발자 센터에 보기 및 전환 이벤트를 보고할 수 있습니다.

> **중요** 실험을 활성화한 후 테스트 실험이 아닌 경우(실험을 만들 때 **테스트 실험** 확인란을 클릭하지 않은 경우) 더 이상 실험 매개 변수를 수정할 수 없습니다. 실험을 활성화하기 전 앱에서 실험을 코딩하는 것이 좋습니다.

## 다음 단계

개발자 센터 대시보드에서 실험을 정의하면 다음 단계를 수행할 준비가 완료된 것입니다.
1. [실험용 앱을 코딩합니다](code-your-experiment-in-your-app.md). Microsoft 스토어 참여 및 수익 창출 SDK의 API를 사용하여 실험에 대한 변형 설정을 가져오고, 이 데이터를 사용하여 테스트할 기능의 동작을 수정하고, 개발자 센터에 보기 이벤트 및 전환 이벤트를 전송합니다.
2. [개발자 센터 대시보드에서 실험을 실행하고 관리합니다](manage-your-experiment.md). 대시보드를 사용하여 실험의 결과를 검토하고 실험을 완료합니다.
