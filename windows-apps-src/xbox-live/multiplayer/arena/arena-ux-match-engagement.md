# <a name="match-engagement"></a>매치 연결

게이머는 등록 하 고는 토너먼트 확인, 재생 하도록 설정 하려는 합니다. 4 단계 중 하나로 참가자 될 수 있습니다. 각 단계는 고유한 게임 토너먼트 환경을 사용자 조건 집합을 포함 되어 있습니다. 4 단계 됩니다.:

1.  준비 (아레나 UI 또는 게임에서)
2.  (게임에서)
3.  결과 (아레나 UI 또는 게임에서)
4.  최종 (아레나 UI 또는 게임에서)

> [!NOTE]  
> 재생은 아레나 UI에서 지원 되지 않는 네 가지 유일한 단계입니다. 최소한 제목 리디렉션하여 참가자 각 일치 (끝나면 재생 단계)의 끝에 아레나 UI에 결과 볼 수 있습니다.

### <a name="user-impact"></a>사용자 영향

이 단계에서 일치 하는 항목에서 재생 하는 게이머 예상 되지만 상대 아직 알 수 없습니다. 이 단계는 토너먼트를 시작 하 고 참가자가 일치 세션을 시작에 대 한 대기 중인 경우 트리거됩니다. 이 대기 시간 때문일 수 있습니다.

* 체크 인 기간은 열려 있습니다.
* 다음 라운드 판매가 시작 합니다.
* 참가자 반올림 현재에서 bye에 있습니다.

### <a name="when-a-bye-is-needed"></a>bye 필요한 경우

bye 것을 방지 하려면 시드 분산 하는 경우에는 토너먼트의 모든 라운드에서 발생할 수 있습니다.

언제 든 지는 단일 제거 또는 이중 소거 토너먼트 단계 (즉, 4, 8, 16, 32, 64, 128 또는 256) 2의 제곱 되지 않은 플레이어 또는 팀의 많은, 예를 획득 해야 합니다. bye 단순히 있음을 나타내고 팀은 토너먼트의 첫 번째 라운드에 참여 하지 않아도 대신 두 번째는 무료 패스를 가져옵니다.

![팀 검색 스크린샷](../../images/arena/arena-ux-team-discovery.png)

* 아레나 UI: 토너먼트 세부 정보 페이지
* 게임에서: 토너먼트 목록, 홍보, 일치 로비

> [!TIP]  
> **UX 권장 사항**

>
> 이 단계 지원 되는 경우 토너먼트 세부 정보 페이지를 반환 하는 메서드를 제공 합니다.  
>
> 이벤트의 컨텍스트를 제공: 토너먼트 이름, 상태, 스타일, 설명, 시작 시간/종료 시간 등.

### <a name="playing-a-match"></a>일치 하는 재생

시작 시간에 일치 하는 세션 템플릿에서 정의 됩니다. 표준 UTC 형식으로 날짜 및 시간으로 설정 되어-예를 들어 `2009-06-15T13:45:30.0900000Z`. 시작 시간은 동시에 포함 하 여, 원하는 대로 토너먼트 이끌이 시작 시간 보다 오래 세션을 만들 수 있습니다. 일치 알림 제목 가져오기 정품 인증 되지 시작 시간 전에 하므로 시작 시 일치 참가자에 게 전송 됩니다. 일반적으로 아레나 세션 타이틀의 멀티 플레이 세션 수 취급 하는 동일한 방식으로 처리 합니다. 그러나는 몇 가지 차이점이 있습니다.

* 타이틀 게임 세션을 변경할 수 없습니다.
* 문제가 있는 연결 속도 사용 하 여 플레이어에 대 한 일반적인 vetting 프로세스가 없습니다.
* 타이틀 중재에 참여 해야 합니다.

### <a name="under-the-hood"></a>내부에서

#### <a name="protocol-activation"></a>프로토콜 활성화

아레나 UI 교육은 작업 사용자 일치 하는 항목을 재생할 준비가 되 면 참가자는 제목에 직접 전송 하 여-예를 들어 수락 "일치 준비" 알림을 보내면 아레나 하 여 합니다. 프로토콜 활성화에 두 번 발생할 수 있습니다: 제목이 시작 되는 이미 실행 중인 경우 또는 합니다. 두 경우 모두 활성화는 게임에 초대를 수락 하는 사용자에 대 한 응답에서 제목을 활성화 되 면 어떻게 될까요 비슷합니다.

* **제목 시작 되는 경우** -제목을 시작할 때 처음으로 **활성화** 이벤트가 발생 합니다. 해당 초기 정품 인증은 아레나 프로토콜 활성화를는 토너먼트에서 재생 하는 사용자가 제목이 시작 되었음을 의미 합니다. 제목 주 메뉴 및 로그인 화면을 우회 하 여 일치 항목을 최대한 빨리 건너뛰어야 합니다. 참여 사용자의 Xbox 사용자 ID (XUID) URI 정품 인증에서 제공 되 고에 이미 있어야 합니다.

* **제목이 이미 실행 중인 경우** - **활성화** 이벤트 제목을 이미 실행 중인 동안에 아레나 프로토콜 활성화를 사용 하 여 발생할 수 있습니다. 제목이 너무 프로토콜 활성화를 받을 때 유휴, 토너먼트 일치 항목으로 바로 이동 합니다. 명시적인 사용자 작업 (역할에 일치 하는 알림 또는 아레나 UI에서 일치 항목으로 이동)에 의해서만 활성화 이벤트가 트리거될 때문에 이렇게 하려면 안전 합니다. 제목 게임 플레이 하는 동안 프로토콜 활성화를 받은 경우 (저장, 필요한 경우) 게임을 두고 토너먼트 일치 하는 가능한 가장 편리한 방법으로 입력 하는 옵션 사용자 제시 해야 합니다.

* **참가자 Xbox 알림 메시지를 무시 하는 경우** -아레나 지원 제목은 현재 일치 하는 사용자가 있는지 여부를 확인 하려면 시작할 때 아레나를 항상 쿼리는 것이 좋습니다. 따라서 "확인 일치 항목" 제목을 표시 해야 하는 경우 UI를 일치 하는 있고 입력 것인지 여부를 요청 사용자에 게 알려줍니다.  

  이 경우 일치 알림을 받지 있으며 (것으로 예상 하는 일치 항목을 갖도록) 게임을 실행 하면 참가자 토너먼트 일치에 전환을 올바르게 리는 보장 합니다. 또는 경우에서 게임 크래시를 참가자가 일치 돌아가는 데 제목을 방금 relaunches.

> [!NOTE]  
> 제목은 프로토콜 활성화 URI에 XUID 확인 해야 합니다. 현재 플레이어는 타이틀의 일치 하지 않는 것을 경우 제목 사용자 컨텍스트도 전환 해야 합니다.

#### <a name="forfeit-time-out"></a>Forfeit 시간 제한

하나 이상의 플레이어 시작 시간 및 forfeit 시간 제한 (재생 다이어그램을 참조) 인 forfeit 시간 이전 세션에서 활성 상태 여야 합니다. 아무도 forfeit 시간 이전 활성 세션에 연결 하 고, 일치 하는 취소 되 고 두 팀 손실을 아레나 중재를 통해 제공 됩니다.
 
## <a name="3-results"></a>3입니다. 결과

![일치 다이어그램의 결과](../../images/arena/arena-ux-flow-results.png)

단계는 플레이어가 일치를 완료 되 면 결과 보고 된 중재 됩니다. 이 시점에서 것은 아직 밝혀지면 팀은 토너먼트에서 여전히 인지 합니다. 한편, "게임 종료" 화면 환경 (처럼 모든 멀티 플레이 일치) 세션의 결과 보고 계속 해야 합니다.

토너먼트 결과-콘테스트 (게임이), Win, 손실, 그리기, 순위 없거나 아니요 표시-수행 됩니다.

결과 즉시 사용 가능한 경우 생략이 단계에 대 한 것 같습니다.

### <a name="reporting-results"></a>결과 보고합니다.

일치 항목의 결과 *중재*라는 기능을 사용 하 여 아레나 및 세션을 통해 토너먼트 구성에 다시 보고 됩니다. 중재 일치 세션을 안전 하 게 일치 및 보고서 결과 재생 하는 데 사용 되는 프레임 워크입니다.

일치 세션 시작 되 면 세션 중재는 것으로 간주 됩니다. 중재 프레임 워크에서 적용 하는 고정된 일정을가지고 있습니다.
 
### <a name="arbitration-time-out"></a>중재 시간 제한

중재 제한 시간이 최대 시간, 일치 하는 참가자가 플레이 하 고 결과 보고 하는 시간을 시작 합니다. 이 값은 설정 토너먼트 이끌이 실행 이벤트에 대 한 세션 템플릿 및 아레나 멀티 플레이어 게임 모드 UGT 및 독점 판매 권 실행 이벤트에 대 한 요구 사항에 맞춰 타이틀 많은 시간을 사용할 수 있도록 합니다.

제목 언제 든 지 중재 시간과 시작 시간 사이 결과 보고할 수 있습니다. 중재 언제 든 지 forfeit 시간 사이의 중재 시간, 세션의 모든 활성 구성원에 결과 제출한 후 발생 합니다. 예를 들어 하나의 멤버 forfeit 시 세션에서 활성화 되 면 수 있습니다 (및 해야) 게시할 결과 및 중재가 발생 합니다. 얼마나 많은 결과 사용할 수 있는 중재 시간에 관계 없이 이미 있지 않으면 중재가 발생 합니다.

사용자 세션 (중재 제한 시간이 만료 되었거나 때문에, 게임 서버 arbitrates 세션 사용자 늦게 조인) 중재 이미가에 있을 경우 타이틀 일치를 종료 하 고 사용자에 게 arbitrated 결과 표시 해야 합니다.

중재 결과 항상 모든 팀에 대 한 결과 포함합니다. 개별 플레이어의 점수 보고 되 면 팀의 결과 뿐만 아니라 하지만 모든 팀에 대 한 결과의 전체 집합을 포함 합니다.
Xbox Live를 결과 보고 하는 첫 번째 클라이언트가 곧바로 중재가 자동으로 시작 합니다. 결과 보고 하는 모든 클라이언트는 즉시 끝납니다. 여기서 참가자 재생 세션 템플릿에 지정 된 중재 시간 보다 오래 인스턴스가 있을 수 있습니다. 이 경우 위험 일치는 참가자가 준비를 하기 전에 종료 해야 합니다.

> [!TIP]  
> **UX 권장 사항:** 고정된 일치 세션 시간 프레임을 제공 합니다.
>
> 일치 세션에 미리 정의 된 시간 제한을 중재 제한 시간에서 발생 하는 문제 될 위험이 줄어듭니다.

두 가지 방법으로 일치 하는 참가자가 완료 되기 전에 끝나는 수 없도록 방지 재생 중인:

* 중재 시간 제한에 대 한 일반적인 시간대를 설정 합니다.
* 타이틀의 게임 플레이 스타일을 사용 하 여 충돌 하는 고정 된 시간 범위를 경우 다음 솔루션을 시도해 보세요.
   1.   결과 보고 되지 않았습니다, 그리고 및 중재 제한 시간이 만료 되기 결정 (예를 들어 5 분이 유지 됩니다).
   2.   자신의 게임 X 끝나는 되려고 참가자에 게 경고 하는 UI를 표시 시간 (분). 이 옵션 모든 토너먼트에 대 한 구현할 수 있으며 중재 제한 시간을 일반적인 경우 대부분의 참가자는는 표시 되지 않습니다.
