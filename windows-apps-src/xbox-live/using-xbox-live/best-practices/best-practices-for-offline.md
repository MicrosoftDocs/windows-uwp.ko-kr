# <a name="best-practices-for-offline"></a>오프라인을 위한 모범사례

Xbox One은 연결 된 장치로 및 다중 접속 게임 및 비디오 스트리밍와 같은 최고의 연결이 필요 합니다. 그러나 Xbox One에서는 다양 한 형태의 오프 라인 재생. 이 항목에서는 오프 라인 재생을 처리 하는 방법을 설명 합니다.

## <a name="overview"></a>개요

전 세계 소비자는 계속 증가 수에는 인터넷에 액세스할을 수 합니다. 그러나 가지 전 세계의 연결을 예측할 수 없는 경우가 있습니다 라우터 실패 하는 경우 않았고 파이버 여전히 위치는 잘라내기, 서버 충돌 또는 무선 서비스를 삭제 합니다.

를 지원 하기 위해 소비자 및 환경을 넓은 Xbox One 허용 일반적인 사례는 인터넷 연결 손실 때때로 되었거나 완전히 사용할 수 없는 합니다. 연결 실패의 게임 알게 되 고 응답 하는 방법을 선택할 수 있습니다-전체 게임 플레이 계속 하는 오프 라인 모드로 다운 그레이드 여부 게임 플레이 완전히 종료 합니다.

## <a name="normal-online-operation"></a>일반 온라인 작업

일반적으로 Xbox One 콘솔 안정적인 인터넷 연결 및 Xbox live 및 타사 서비스에 대 한 모든 권한을 사용자에 완전히 연결 된 상태에서 작동 합니다. 이 연결 된 상태에는 Xbox Live 서비스 정기적으로 업데이트를 제공 하 고 게임과 사용자를 활용 하는 다른 백그라운드 서비스를 수행 하는 콘솔 상태 유효성 검사를 허용 합니다.

사용자가 대부분의 온라인 연결 한다고 가정 하는 것이 좋습니다.

## <a name="offline-play-principles"></a>오프 라인 재생 원칙

Online 연결을 사용할 수 없는 경우가 있습니다. 오프 라인 재생에 대 한 염두에서 다음 원칙을 사용 하 여 Xbox One이 설계 됩니다.

-   연결 문제에도 불구 하 고 재생 하는 사용자를 가장 중요 하 게 유지 합니다.

-   전혀 연결이 있는 경우에 재생 하는 사용자를 유지 합니다.

-   확인 오프 라인 재생 간단 하 고 예측 가능한 사용자의 경우 항상 연결 된 환경에서의 진정한을 유지 하면서 합니다.

## <a name="offline-modes"></a>오프 라인 모드

두 가지 대략적인 *연결 손실로* 시나리오:

-   인터넷 서비스의 전체 손실

-   하나 이상의 온라인 서비스의 손실

각이 모드는 다양 한 상황에 발생할 수 있는 있습니다. 이러한 게임 플레이 영향을 주는 일반적인 오프 라인 시나리오의 예제를 사용 하 여 아래 설명 되어 있습니다.

## <a name="offline-scenario-no-internet-service-upon-game-start"></a>오프 라인 시나리오: 게임 시작 시 인터넷 서비스 없음

게임에는 세 가지 유형 중 하나로 직접 선언할 수 있습니다.

-   *Xbox Live 필수*: 모든 모드 게임 플레이에 인터넷 연결이 필요 합니다.

-   *Xbox Live gold 정기 필요한*: 인터넷 연결 및 Xbox Live gold 정기 회원 자격을 모든 모드 게임 플레이에 필요 합니다.

-   *Xbox Live 없어도*: 게임에 하나 이상의 인터넷 연결이 필요 하지는 플레이 모드입니다. 기술적으로 응용 프로그램 매니페스트에서이 형식은 명시적으로 선언 되지 않았습니다. 선언 하지 않으면 자체 것으로 간주 "Xbox Live 필수" 처음 두 형식 중 하나는 앱 또는 오프 라인 지원 합니다.

게임 사용자가 시작한 콘솔은 오프 라인을 시스템 응용 프로그램 매니페스트에서 게임 연결 선언을 확인 합니다. 게임 (위의 처음 두 경우 중 하나) 연결에 필요한 경우 시스템 자동으로 사용자에 게 메시지를 표시 및이 제목은 실행 되지 않습니다.

콘솔을 오프 라인 상태인 경우 시스템 타이틀 연결이 필요 하지는 플레이 하나 이상의 모드에 시작 됩니다. 즉, "Xbox Live 필수" 또는 "Xbox Live Gold required" 제목 시스템 시작 되지 됩니다.

## <a name="offline-scenario-connectivity-lost-during-gameplay"></a>오프 라인 시나리오: 게임 플레이 중 연결이 끊어짐

게임을 이미 실행 중인 경우 연결이 손실 된 경우 제목 시스템에서 알림을 받게 됩니다. 게임 온라인 서비스에 사용 하지 않는 경우 중단 없이 세션을 계속 합니다. 게임 온라인 서비스를 사용 중인 경우 이러한 서비스가 더 이상 필요 없는 모드로 전환 하거나 오프 라인 상태로 인해 게임 세션이 종료 되는 플레이어에 알려야 합니다.

제목 "Xbox Live required" 또는 "Xbox Live Gold required" 자동으로 일시 중단 됩니다 콘솔 시간 기간에 대 한 모든 네트워크 연결이 끊어지는 경우 시스템 및 시스템에서 선언 된 사용자에 게 오류 메시지를 자동으로 제공 됩니다.

모든 다른 시나리오와 게임 플레이 일시 중지와 관련 된 상태를 저장 해야 하므로 사용자 데이터를 손실 하지으로 연결을 다시 얻은 후 해당 상태를 신속 하 게 반환할 수 있습니다.

## <a name="offline-scenario-when-a-single-xbox-live-service-is-down"></a>오프 라인 시나리오: 단일 Xbox Live 서비스가 중지 되는 경우

특정 온라인 서비스 사용할 수 있지만 인터넷 연결이 제대로 되는지 다른 경우가 있습니다.

예를 들어, 단일 Xbox Live 서비스는 짧은 기간에 대 한 오프 라인 수 있습니다. 이 경우 특정 호출 서비스는 제한 시간 또는 게임으로 오류를 보고 합니다. Xbox 360 또는 Windows의 경우 처리 하는 것 처럼 오프 라인 상태의 서비스 상태를 올바르게 처리 해야 합니다.

최소한 게임 충돌 하지 해야 하거나 응답 하지 않습니다. 게임 플레이 서비스 하지 않고 계속할 수 없습니다, 하는 경우 사용자에 게 상황을 보고 하 고 사용자는 서비스가 필요 하지 않습니다 게임의 다른 영역에서 계속 하도록 허용 합니다.

최상의 경우에서 (게임 서비스에 작성 된) 경우 나중에 전송 하거나 (서비스에서 게임을 읽고) 하는 경우 데이터에 대 한 사용자를 가정을 게임 플레이 캐시 데이터를 계속 합니다.

## <a name="offline-scenario-when-a-third-party-service-is-down"></a>오프 라인 시나리오: 타사 서비스가 중지 되는 경우

게임-타사 온라인 서비스에 의존 한 다음 게임 해당 서비스가 실패 하면 복원 력 있는 수도 있어야 합니다. 서비스가 실패 하면 다음 게임 해야 하지 충돌 하거나 응답 하지 않습니다. 게임 플레이 계속할 수 없습니다 하는 경우 사용자에 게 서비스 오류를 보고할 수 있습니다. 이상적으로 게임 플레이 계속할지 또는 영역 온라인 서비스를 필요 하지 않은 게임을 계속 하려면 사용자를 허용 해야 합니다.

## <a name="offline-scenario-when-xbox-live-cloud-compute-is-down"></a>오프 라인 시나리오: 아래로 클라우드 Xbox Live 계산 표시 되는 경우

Xbox One에서는 중 하나에 클라우드 기능입니다. 일부 게임 추가 계산 기능 또는 항상 사용 가능한 게임 서버에 액세스할 수 있는 Xbox Live 클라우드 계산 등 항상 연결 된 서비스를 완전히 사용할 수 있습니다. 항상 연결 된 모드가 허용 되 고 여기서 플레이어의 환경을 개선 하는 것이 좋습니다.

이 모드를 사용 하는 게임 경우 게임 서비스 중단에도 잘 복원 되도록 하는 것이 좋습니다 (여러 시간 (초) 및 최대 1 분)으로 인해 인터넷에 연결 하거나 총 오류나 특정 클라우드 서비스의 손실 된 합니다. 그러나 게임은 오프 라인 모드에 있이 필요가 없습니다. 게임에 대 한 연결이 실제로 필요 하 고 연결을 사용할 수 없는 경우 사용자에 게 알릴 및 게임 플레이 세션을 종료 하는 것입니다.

## <a name="xbox-requirements"></a>Xbox 요구 사항

가장 중요 한 오프 라인 시나리오를 처리할 때 중요 한 점은 게임 안정성입니다. 전체 연결 손실 또는 특정 온라인 서비스의 손실 방금 든이 게임 해야 하지 중단, 충돌 하거나 상태를 손실 하면 발생. 게임에는 네트워크 제한 시나리오 처리에 대 한 강력한 시스템을 있어야 하 고 온라인 서비스에 액세스 하는 모든 API에서 반환 된 오류를 처리 하기 위한 키를 누릅니다.

게임은 오프 라인 재생을 지원할 필요가 없습니다. 단순히 게임 서비스 연결 손실로 인해 계속할 수 없습니다, 하는 경우 다음 사용자에 게 게임 세션을 종료 하며 주 메뉴 또는 초기 대화형 상태를 반환 합니다.

## <a name="best-practices"></a>모범 사례

오프 라인 상황을 처리 하기 위한 모범 사례는 다음과 같습니다.

-   사용자가 대부분의 온라인 연결 한다고 가정 하는 게임을 디자인 합니다.

-   적합 한 게임 디자인, 게임 콘솔 오프 라인 상태인 경우에는 즐거운 경험이를 사용할 수 있는 모드를 디자인 고려 합니다.

-   서비스를 사용할 수 없게 될 수 있습니다. 연결이 실패할 수 있습니다. 견고한 오류 처리 제한 시간을 수 있는, 다운 된 경우 온라인 서비스 오류 조건을 보고 또는 인터넷 연결이 끊어진 경우 api를 빌드하십시오. 가능 하면 이러한 문제에도 불구 하 고 재생 하는 사용자를 유지 합니다.

-   Xbox 요구 사항 (XRs)를 준수 합니다. 충돌 또는 중단 하지 마세요.

-   PLM 제목 일시 중단 알림을 수신 하면 사용자 데이터가 손실 되지 않습니다 및 게임을 다시 시작할 때 해당 상태를 신속 하 게 반환할 수 있도록 상태를 저장 합니다.

-   응용 프로그램 매니페스트에 적절 하 게 제목을 플래그입니다. 표시할 제목을 "Xbox Live 필수"로 모든 모드 게임 플레이를 연결 해야 하는 경우.

-   Xbox One 게임 사용 하 고 모든 게임 모드에서 온라인 서비스 사용 수 있습니다. 단순히 게임 서비스 연결 손실로 인해 계속할 수 없습니다, 하는 경우 다음 사용자에 게 게임 세션을 종료 하며 주 메뉴 또는 초기 대화형 상태를 반환 합니다.

-   오류 메시지에 대 한 도움말 Xbox 서비스에 의존 하지 마십시오. 오프 라인 상태에 관련 된 도움말을 Xbox 도움말 서비스에는 Xbox Live에 대 한 연결에 필요합니다.

## <a name="conclusion"></a>결론

Xbox One 항상 연결 된 세계를 위해 설계 되었습니다. 그러나 콘솔 해당 비전으로 오프 라인 시나리오를 활용 하는 플레이어를 허용 합니다. 게임 연결 오류에 대처할 이어야 합니다. 오프 라인 재생 환경 사용 하 여 게임을 플레이어를 최대한 이러한 환경을 즐길 수 있도록 합니다. 네트워크 연결이 손실 될 때 항상 온라인 있는 게임에 정상적으로 오프 라인 상태로 사용자를 반환 합니다.
