# <a name="game-session-and-game-party-visibility-and-joinability"></a>게임 세션 및 게임 파티 유형과 joinability

Xbox One에 *가시성* 및 *joinability* 설정을 게임 파티와 게임 세션에 대 한 각각 멀티 플레이 환경에 대 한 액세스를 제어 합니다. 제공 하는 뛰어난 사용자 환경을 세션 참가 하기 위한 초대 플레이어 게임 세션 및 파티에, 제목 개발자는 이러한 설정을 이해 해야 합니다. 이 백서에서는 유형과 joinability, 간의 차이점을 검토 하 고 제목 소비자 가장 멀티 플레이어 사용자 흐름을 제공 하는 데 사용할 수 있는 것이 좋습니다 특정 설정에 설명 합니다.

## <a name="game-session-visibility"></a>게임 세션 표시


다중 접속 세션 디렉터리 (MPSD) 세션 액세스 두 설정에 의해 제어 됩니다: 세션 *가시성* 및 세션 *joinability*합니다. MPSD 수준의 세션 액세스를 제한 하려면이 설정은 사용할 수 있습니다.

액세스 제어의 첫 번째 기능인 세션 표시 됩니다. 세션의 표시 여부는 세션 생성 시 설정 된 상수입니다. 일반적으로 세션 템플릿에서 정의 하 고 어떤 유형의 사용자 읽기 및 세션에 대 한 쓰기 액세스를 결정 합니다. 표시 유형 설정에 대 한 값은

-   *오픈-* 세션의 연결이 읽고 모든 사용자 세션에 쓸 수 있습니다.

-   *표시-* 모든 사용자가 읽을 수 없지만 플레이어 세션에 작성할 수 있습니다만 가입 하 고 예약 합니다.

-   *Private-* 가입 하 고 예약 된 플레이어 읽거나 쓸 수는 세션에 있습니다.

> **참고:** 비공개로 표시 유형 설정에 재시도가 필요할 수 있습니다 합니다 **join ()** 초대 된 플레이어를 호출 합니다. 초대 UI 플랫폼을 통해 전달 되 면 플레이어 해당 세션에서 예약 되기 전에 다른 플레이어에서 받을 수 있습니다. 이 경합 상태가 발생할 수 있습니다는 **join ()** 비공개 하거나 세션 조인 플레이어에 대 한 예약이 필요 하기 때문에 실패 하는 초대 된 플레이어를 호출 합니다.
>
> 예약이 없습니다 (HTTP/403) 세션에서 액세스 오류가 발생 합니다. 초대 된 플레이어를 다시 시도해 야 합니다는 **join ()** 작업 하지만 재시도 5 초 마다 보다 더 자주 시도 하지 마십시오. 제목 플레이어는 조인 흐름 중 경합 상태를 방지 하 여 세션 표시 유형을 설정 하는 것이 좋습니다.

## <a name="game-session-joinability"></a>게임 세션 joinability


액세스 제어를 위한 두 번째 점은 joinability 합니다. 세션 수명 동안 동적으로 설정할 수 있습니다 하 고 어떤 유형의 사용자는 세션에 참가할 수를 결정 합니다. 세션 joinability 값은 다음과 같습니다.

-   *None* (기본값)-세션에 참가할 수 있는 사용자에 대 한 제한은 없습니다.

-   *로컬*-로컬 사용자 세션에 참가할 수만 있습니다.

-   *다음-* 로컬 사용자 및 다른 세션 멤버 뒤에 사용자를 예약 하지 않고 세션에 참가할 수만 있습니다.

세션 호스트나 중재자 joinability 설정을 통해 개인 세션을 만들 수 있습니다: 로컬 또는 다음 joinability 만드는 게임 세션에 대 한 액세스를 제한 되며 비공개로 설정 합니다.

또한 세션 호스트나 중재자 해야 기록해 joinability 세션의 이전 세션 초대 거부 될 수 있습니다 호스트 수준에서 필요한 경우 있도록 합니다. 예를 들어, 모든 초대 플레이어가 세션이 이미 가득 찰 때까지 연결할 세션 및/또는 파티 도착 하지, 하는 경우 호스트 또는 중재자 지시할 수 있습니다 모든 조인 플레이어 세션이 잠 궜 습니다을 세션 및/또는 자동으로 타사 할.

**참고:** 게임 세션 joinability 파티 앱 UI에 반영 되지 않습니다. 타사 앱 UI 초대 joinability를 제한 하는 경우에 하 고 **ShowSendInvitesAsync** 게임 세션 초대 허용 됩니다.

## <a name="game-party-joinability"></a>게임 파티 joinability


MPSD 세션을 제어 하는 것 외에도 또한 제목을 게임 파티 joinability 상태를 제어 합니다. 파티 수준에서 게임 타사 액세스를 제한 하려면이 설정을 사용할 수 있습니다.

파티 joinability 게임 파티를 만들면 동적으로 변경할 수 있습니다. 게임 파티의 joinability의 상태에 반영 된 파티 UI를 초대 합니다. 설정할 수 있습니다.

-   *초대 전용-* 이 설정에서는 게임 파티 참가 초대를 취소 합니다.

-   *친구 들 여 조인 가능* (기본값)*-* 이 설정에서는 게임 파티 (파티에 가입 하려면 파티 멤버에 조인 플레이어에 따라)를 조인 하는 플레이어에 대 한 friend 관계입니다.

Joinability 초대 전용 게임 파티를 만드는 데 사용할 수 있습니다. 파티에 대 한 액세스를 제한 하 고 플레이어가 참여 초대를 받은 필요에 joinability "초대"로 설정 해야 합니다.

**참고:** 파티 joinability 세션 joinability 영향을 주지 않습니다 하지만 파티 joinability 파티 앱 UI에 반영 됩니다. 플레이어는 수동으로 제목 외부 타사 앱에서 파티 joinability를 변경할 수 있습니다.

## <a name="recommendations"></a>권장 사항


가장 일반적인 제목 시나리오에 대 한 다음 권장 사항을 적용 합니다. 가능한 경우 제목에 이러한 설정을 따라야 합니다. 및-제목 논리 여부 새 플레이어 세션에 연결 하는 여부에 대 한 최종, 신뢰할 수 있는 대상을 사용 해야 합니다.

### <a name="recommended-game-session-visibility-open"></a>권장 되는 게임 세션 표시 유형: 열기

열려 있는 게임 세션 플레이어 예약 하지 않아도, 초대 하 여에 대 한 흐름을 간소화 합니다. 중재자를 세션 호스트 초대를 보낸 후에 세션 문서의 플레이어 예약 하지 않습니다. 대신, 세션 호스트나 중재자 초대 플레이어 로컬로 추적합니다. 이렇게 하면 즉시 호스트에 연결 하 고 있는지 여부를 확인할 해당 세션에 참가 해야, 거부 됩니다 (대기 플레이어 지원 됨) 하는 경우 대기 해야 하는 플레이어 수 있습니다. 중재자 또는 호스트에는 ultimate 기관 한 응답 하 고 유지 또는 세션에 새 멤버를 지정 하면 키를 누릅니다.

**참고:** 초대 된 플레이어가 제목 시작 되 고 최종 결정에 사용 되기 전에 중재자는 호스트에 연결 해야 합니다. 세션 가득 차거나 초대 거부 된 경우 사용자에 게 오류 메시지를 표시 하는 것이 좋습니다.

> **참고:** 중재자를 호스트에 대 한 연결을 설정 하는 보안 장치 주소가 필요 합니다. 합니다 **HostDeviceToken** 세션에는 현재 호스트 또는 세션 및는 중재자 세션 멤버 임을 보안 장치 주소에 초대 된 플레이어를 연결 해야 합니다.

### <a name="recommended-game-session-joinability-default-not-set"></a>게임 세션 joinability 권장: 기본값 (*설정 되지 않은*)

현재 게임 세션 joinability 파티 앱 UI에 영향을 주지 않습니다 및 사용자에 게 노출 하지 않습니다. 게임 joinability를 간소화 하기 위해 제목 흐름 기본값을 유지 해야 하 고 모든 조인 기관 중재자 또는 호스트를 통해 처리 해야 합니다.

### <a name="recommended-game-party-joinability"></a>게임 파티 joinability를 권장합니다.

게임 파티 joinability 제목을 사용자에 게 제공 하는 세션의 유형에 따라 달라 집니다. 두 시나리오는 다음과 같습니다.

-   게임을 열려면-열기를 게임에 대 한 게임 파티 joinability 두어야 기본값: 친구 들 여 조인 합니다. 이렇게 하면 파티와 게임 세션 나아가 게임에 참가 하려면 친구 초대 없이 합니다.

-   개인 게임-개인 또는 closed 게임, 게임 파티 joinability 초대 전용으로 설정 해야 합니다. 이 제한 다른 플레이어 파티에 참여할 (및 게임 세션 나아가) 초대 없이 합니다.

**참고**: 플레이어 joinability 타사 앱을 통해 게임 세션을 수동으로 변경할 수 있습니다.

게임 유형에 모두 중재자 또는 호스트 세션에 허용 되는 확인 하려면 ultimate 기관을 여전히 유지 되어야 합니다. 이 게임에서 흐름 열기 개인 전환에서 발생할 수 있는 경합을 해결 합니다. 중재자/호스트가 플레이어를 거부 하는 경우 거부 된 제목 인스턴스는 세션에서 플레이어를 제거 하 고 파티를 유지 하는 플레이어도 사용할 수 있도록 게임 내 UI를 표시 해야 합니다.

### <a name="maximum-session-size"></a>최대 세션 크기

게임 세션에 참가 하는 플레이어의 수를 제한 하는 세션에 대 한 최대 멤버 크기를 사용할 수 있습니다. 그러나이 한도 여전히 전체 중에 특정 시나리오를 분리 하는 대로 표시 하는 열기 스폿을 발생할 수 있습니다. 예를 들어, 네트워크 또는 전원 오류로 인해 플레이어가 끊어지면 지연을 반영 되지 않습니다 즉시 세션에서. 멤버는 현재 상태 서비스에서 연결 끊기 (최대 20 초)을 검색 하 고 플레이어가 비활성 제한 시간이 만료 된 후 삭제 됩니다 즉시 비활성으로 설정 됩니다.

반면에 하트 비트를 사용 하 여 연결 끊김을 감지 하는 피어 메시에 서 2-3 시간 (초) 내에서 연결을 끊도록 인식 경우가 즉시 플레이어 슬롯 엽니다 수 없습니다. 그러나 중재자 또는 호스트는 다른 멤버를 제거할 수 없습니다.

이 문제를 해결 하기 위해 세션의 최대 멤버 크기 항상 설정할 제목을 실제로 지 플레이어의 최대 개수 보다입니다. 예를 들어 플레이어 수 8 인 제목 최대 세션 크기 12에서 설정 해야 합니다. 이러한 방식으로 새 플레이어가 이전 플레이어 아직 초과 되지 된 경우에 참가할 수 있습니다. 중재자 또는 호스트 결정 세션 전체 인지 및 사용자 지정 세션 속성을 새 플레이어 여전히 연결할 수 있는지 여부를 결정 하는 집합 (**IsFull** : "true"). 세션에 참가할 수 있는 경우 플레이어를 조인에서 빠른 검사를 수 있습니다.

중재자 또는 호스트는 멤버를 인덱스로 초과 나타내는 사용자 지정 속성을 유지 관리도 해야 (예를 들어 **TimedOutPlayers** : “3, 5, 9”). 이 멤버를 현재 세션을 올바르게 식별 하는 새 플레이어 수 있습니다.

### <a name="notifying-waiting-players"></a>대기 중인 플레이어 알림

제목을 게임 파티 외에도 큐에 대기 중인된 플레이어 목록을 관리 하 여 대기 플레이어를 지원할 수 있습니다. 플레이어 호스트에 연결 하 고 게임 세션은 전체, 플레이어는 호스트의 내부 대기 목록에 추가 됩니다. 큐에 대기 중인된 플레이어 게임 세션을 조인 하지 않는 되지만 게임 파티에 유지 됩니다. 네트워크 트래픽을 최소화 하기 위해 대기 플레이어 연결을 끊습니다 호스트에서이 시점에서.

게임 세션에서 슬롯을 대기 플레이어 열리면 중재자 또는 호스트 예약을 플레이어에 대 한 호출 하 여 추가 합니다 **AddMemberReservation** 메서드. 이 시점에서 대기 중인 플레이어 되지 않으므로이 예약의 인식 하므로 중재자 또는 호스트를 호출 해야 합니다 **PullReservedPlayersAsync** 메서드. 이 인해 UI 알림 또는 **GameSessionReady** 알림 구성 및 제목 포커스 상태에 따라 모든 예약 된 플레이어에 대 한 알림입니다. 이 알림을 수신 하 고 세션을 조인 하는 경우 새 플레이어는 호스트에 다시 연결 합니다.

또는 플레이어는 호스트에 연결 된 상태로 유지 하 고 때 호스트 경고 플레이어 슬롯을 사용할 수는 세션에 참가할 수 있습니다. 그러나이 흐름에서 제목 외부 UI 알림을 표시할 수 없습니다.

**참고:** 제목 일시 중단 및/또는 종료와 더 이상 알림을 받을 경우 플레이어 게임 파티에서 자동으로 제거 됩니다.
