# <a name="xbox-live-multiplayer-concepts"></a>Xbox Live 멀티 플레이 개념

이 항목에서는 다양 한 중요 한 멀티 플레이 용어 및 Xbox Live 설명서에서 자주 사용 되는 개념을 설명 합니다. 이러한 개념 이해할 것 Xbox Live 멀티 플레이어의 작동 방식을 이해 하는 데 도움이 됩니다.

## <a name="multiplayer-session"></a>멀티 플레이 세션

멀티 플레이 세션 Xbox Live 사용자와 연결 된 속성의 그룹을 나타냅니다. 세션이 생성 되 고 제목, 유지 관리 하 고 Xbox Live 클라우드 상주 하는 보안 JSON 문서도 표현 됩니다. 게임 세션에 관련 된 기타 정보가 세션 문서 자체 얼마나 많은 지점 (세션에도 각 세션 구성원 가요), 사용 가능한 사용자 지정 메타 데이터는 세션에 연결 된 Xbox Live 사용자에 대 한 정보를 포함 합니다.

각 세션 게임 개발자가 정의 하 고 제목 인스턴스에 대 한 Xbox Live 서비스 구성에서 구성 되는 세션 템플릿을 기반으로 합니다.

타이틀을 만들어 업데이트 세션을 하는 동안 세션을 삭제 직접 수 없습니다.  대신, 모든 플레이어 세션에서 제거 되 면 Xbox Live 멀티 플레이 서비스는 지정 된 시간 제한 이후에 세션을 자동으로 삭제 됩니다. 세션에 대 한 자세한 내용은 [MPSD 세션 세부 정보](multiplayer-appendix/mpsd-session-details.md)를 참조 하세요.

타이틀 여러 세션을 사용 하도록 선택할 수 있지만 일반적인 멀티 플레이 구현은 두 개의 세션 사용:

* 세션-로비 여러 차례, 수준, 지도, 게임 등을 함께 유지 하고자 하는 친구의 그룹을 나타내는 세션입니다.
* 게임 세션-라운드, 일치, 수준 등 등 게임의 특정 세션 인스턴스에서 재생 하는 사람을 나타내는 세션 이것이입니다. 이 세션에 가입한 세션 인스턴스를 함께, 일반적으로 매치 메이 킹 서비스를 통해 여러 로비 세션의 멤버를 포함할 수 있습니다.

다음은 예제 시나리오: 철 John와 리사 자신의 친구와 멀티 플레이 게임 일부 재생 하려고 합니다. 철은 게임이 시작 되 고 John와 리사 자신의 게임에 초대 합니다. 중인 연결할 철, John 및 유가을 로비 세션에서 모든 됩니다. 이 세션에서 다른 사용자와 일치 하는 온라인에서 재생을 결정 합니다. 게임 게임 세션을 만들고 매치 메이 킹 Xbox Live 서비스를 사용 하 여 Xbox Live 다른 플레이어와 나머지 플레이어 슬롯에 맞게 합니다.

Bob과 김을 그들과 일치 되 고 라운드 함께 재생 중 5 개 가정해 보겠습니다. 라운드 끝난 후 철, John 및 유가을 게임 세션을 종료 하지만 여전히 함께 Bob과 김) (없이 로비 세션에서 되며 라운드, 다른 재생 하거나 다른 게임 모드로 전환 하도록 선택할 수 있습니다.

### <a name="session-member"></a>세션 멤버

세션 구성원은 Xbox Live 사용자 세션의 일부입니다.

### <a name="arbiter"></a>중재자

에 중재자 콘솔 또는 게임 세션의 상태를 관리 하는 디바이스입니다. 예를 들어 중재자 더 많은 플레이어를 찾기 위해 매치 메이 킹 게임 세션을 광고에 대 한 책임 것입니다.

중재자 제목으로 설정 됩니다. 게임의 호스트와 동일 하 하지만 같이 필요가 없습니다.

### <a name="session-host"></a>세션 호스트

세션 호스트는 콘솔 또는 게임을 실행 하는 장치 피어-투-피어 네트워크 호스트 기반 아키텍처를 기반으로 제목에 대 한 시뮬레이션을 재생 합니다. 이 콘솔 또는 장치는 일반적으로 중재자와 동일 하지만 같이 필요가 없습니다.

## <a name="multiplayer-service-session-directory"></a>서비스 멀티 플레이 세션 디렉터리

Xbox Live 멀티 플레이어 서비스 Xbox Live 클라우드에서 작동 하 고 여러 클라이언트에 걸쳐 게임의 멀티 플레이 시스템 메타 데이터를 제공 합니다. 이 메타 데이터를 추적 하는 시스템 멀티 플레이어 세션 디렉터리 또는 MPSD 라고 알려져 있습니다. MPSD 라이브러리의 현재 게임 세션으로 생각할 수 있습니다. 게임에 추가할 수 있는 검색 되, 수정 또는 타이틀에 관련 된 활성 세션을 제거 합니다. 또한는 MPSD 세션 상태를 관리 및 업데이트 세션 필요한 경우.

MPSD 제목을 사용자 그룹에 연결 하는 데 필요한 기본 정보를 공유할 수 있습니다. 세션 기능을 동기화 하 고 일관 된 있는지 확인 합니다. 셸 및 초대 보내기/수락 및 게이머 카드를 통해 가입 되 고 Xbox One 콘솔 운영 체제와 동등 합니다.

### <a name="session-handles"></a>세션 핸들

세션은 데이터의 조합으로 고유 하 게 MPSD에서 식별 됩니다.

* 제목 서비스 구성 ID (서비스 안내)
* 세션을 만드는 데 사용한 세션 템플릿의 이름
* 세션의 이름

세션 핸들에 MPSD에 있는 특정 세션에 대 한 참조를 포함 하는 JSON 개체입니다. 세션 핸들 기존 세션에 참가할 Xbox Live 멤버를 사용 합니다.

각 세션 핸들 타이틀은 단일 guid를 사용 하 여 세션을 참조할 수 있도록 해줍니다 핸들을 고유 하 게 식별 하는 guid를 포함 합니다.

여러 가지 방법으로 세션 핸들 수 있습니다.

* 핸들 초대
* 검색 핸들
* 활동 핸들
* 상관 관계 핸들
* 전송 핸들

#### <a name="invite-handle"></a>핸들 초대

초대 핸들 초대 하는 게임에 참가 하는 경우 멤버에 전달 됩니다. 초대 핸들에 초대 된 멤버의 게임 가입 올바른 세션 수 있도록 하는 정보가 들어 있습니다.

#### <a name="search-handle"></a>검색 핸들

검색 핸들 세션에 대 한 추가 메타 데이터를 포함 하며, 타이틀은 선택한 기준을 충족 하는 세션을 검색할 수 있습니다.

#### <a name="activity-handle"></a>활동 핸들

활동 핸들 수 멤버 참조 하 고 소셜 네트워크의 다른 멤버를 재생 하는 사용할 수는 친구의 게임에 참가 합니다.

#### <a name="correlation-handle"></a>상관 관계 핸들

상관 관계 핸들을 효과적으로 작동 별칭으로는 세션에 대 한 게임만 상관 관계 핸들의 id를 사용 하 여 세션을 참조할 수 있도록 합니다.

### <a name="transfer-handle"></a>핸들을 전송 합니다.

전송 핸들은 다른 세션에 플레이어 세션 간에 이동 하는 데 사용 됩니다.

### <a name="invites"></a>초대

Xbox Live 멀티 플레이 서비스에서 지원 되는 초대 시스템을 제공 합니다. 플레이어가 다른 플레이어의 게임 세션에 초대할 수 있습니다. 초대 받은 플레이어가 게임 초대를 받고 제목을이 정보를 사용 하 여 기존 세션 및 멀티 플레이어 경험에 참가 하 합니다. 타이틀 초대 흐름과 초대를 보낼 수 있습니다를 제어 합니다.

사용자가 또는 제목에서 직접 셸을 통해 초대를 보낼 수 있습니다. 초대한 플레이어에 게 더 많은 정보를 제공 하 여 제목을 초대에 대 한 알림 텍스트를 동적으로 설정할 수 있습니다. 초대는 플레이어에 게 표시 되지 않는 제목에 대 한 추가 데이터를 포함할 수도 있습니다 및 추가 정보를 운반 데 사용할 수 있습니다.

### <a name="join-in-progress"></a>진행 중인 가입

초대를 외에도 친구의 현재 게임 플레이 세션에 참가할 플레이어나 알려진된 다른 플레이어에 대 한 셸 옵션을 제공 또한 Xbox Live 합니다. 현재 게임 세션에 다른 경로 사용 하 고도 MPSD에 의해 좌우 됩니다. 타이틀 세션 있는 경우 및 진행 중인 가입에 대 한 노출 하는 세션을 제어 합니다.

### <a name="protocol-activation"></a>프로토콜 활성화

철 자신의 게임에 참가할 유가을 초대를 보내면 유가을 수락 또는 거절 하도록 선택할 수 사용자는 자신의 디바이스에서 알림을 받습니다.

OS 게임을 아직 실행 되지 하 고 게임이 활성화 된 이유에 대 한 정보를 포함 하는 활성화 이벤트를 트리거하는 경우 게임 및 모든 추가 세부 정보를 시작 하려고 시도할 초대를 수락 하 게 하는 경우 (예를 들어 초대의 경우는 세부 정보 포함 멤버에 초대 된 세션 뿐만 아니라 초대 플레이어의 ID입니다.)

이 이벤트를 처리 하는 프로세스는 프로토콜 활성화 라고 하 고 게임이 활성화 이벤트 인수에서 자세히 설명 하는 특정 상태에 자동으로 전환 해야 나타냅니다. 멤버는 멀티 플레이어 게임에 참가 하는 경우 세션 핸들 id 인수 중 하나로 지정 됩니다.

리사의 경우 초대를 수락 해야 자동으로 (필요한 경우) 게임을 시작 하 고 유가을 추가 작업을 수행할 필요 없이 철, 같은 게임 세션에 참가 뷰의 합니다.

프로토콜 활성화 수 있는 초대를 수락 하 여 트리거된 자신의 프로필 카드를 통해 게이머의 다른 멤버에 가입 하거나 연결 전체 클릭 도전 과제 합니다.

## <a name="smartmatch-matchmaking"></a>스마트 매치 메이 킹

스마트에는 익명 매치 메이 킹에 대 한 Xbox Live 서비스의 이름입니다. 이 서비스 일치 규칙 집합을 기반으로 구성 가능한 동일한 게임의 플레이어를 찾습니다.

매치 메이 킹 서비스는 MPSD 밀접 하 게 작동 하 고 입력 및 출력 매치 메이 킹에 대 한 세션을 사용 합니다. 매치 메이 킹 제목을 쉽게 수행 제목 내 싱글 플레이어 예를 들어 매치 메이 킹 흐름 하는 동안 다른 환경을 제공 하도록 허용 하는 서비스에서 수행 됩니다.

개인 또는 매치 메이 킹 입력 하려면 그룹 일치 티켓 세션을 만든 다음 다른 플레이어와 일치 하는 항목을 설정 하려면 찾으려고 매치 메이 킹 서비스 요청 합니다. 이 시간 동안 매치 메이 킹 서비스 (에서 일치 hopper) 내에 있는 "일치 티켓" 임시 생성 됩니다.

매치 메이 킹 서비스 선택 규칙 구성에 따라 함께 재생 세션 통계 각 플레이어와 일치 요청 시간에 주어진 추가 정보를 저장 합니다. 그런 다음 서비스, 일치 하 고 일치 항목의 사용자의 제목에 게 알리는 모든 플레이어를 포함 하는 일치 대상 세션을 만듭니다.

대상 세션 준비 되 면 제목 품질의 그룹 함께 재생 수 및 플레이어 게임 플레이 시작 하려면 세션에 참가 하 / 것인지 QoS (서비스) 검사를 수행할 수 있습니다. QoS 프로세스 및 matchmade 게임 플레이 하는 동안 제목 세션 상태 MPSD, 내에서 최신 상태로 유지 하 고 세션에 변경에 대 한 MPSD에서 알림을 받을 합니다. 그러한 변경 사항을 가입 하거나, 사용자를 포함 하 고 세션 중재자를 변경 합니다.

### <a name="match-ticket-session"></a>티켓 세션 일치

일치 티켓 세션 일치 여부를 확인 하려면 플레이어에 대 한 클라이언트를 나타냅니다. 일반적으로 함께 로비에 있는 플레이어의 그룹 또는 기타 제목 관련 그룹 플레이어의 기반으로 만들어집니다. 티켓 세션 더 많은 플레이어 추구 하는 진행 중인 게임 세션 수도 있습니다.

### <a name="match-ticket"></a>티켓 일치

매치 메이 킹 세션을 티켓을 제출 매치 메이 킹 시도 추적 하는 일치 티켓 생성 됩니다. 예를 들어 게임 지도 또는 플레이어 수준 티켓에 특성을 추가할 수 있습니다. 티켓 세션에서 플레이어의 특성과 함께 이러한는 일치 항목을 결정 하는 데 사용 됩니다.

### <a name="hoppers"></a>Hoppers

Hoppers는 논리적이 일치 티켓 수집 되 고 매치 메이 킹의 시작 부분에 지정 합니다. 동일한 hopper 내 티켓만 연결할 수 있습니다. 타이틀 여러 hoppers 수 있지만 매치 메이 킹에 한 번에 하나씩만 시작할 수 있습니다. 예를 들어 제목을 하나의 hopper 있는 플레이어에 대 한 기술은 일치 하는 가장 중요 한 항목을 만들 수 있습니다. 다른 hopper는 플레이어가 일치 하는 동일한 다운로드 가능한 콘텐츠를 구매한 경우 사용할 수 있습니다.

## <a name="quality-of-service-qos"></a>서비스 품질 (QoS)

온라인 멀티 플레이어 게임을 실행 하는 게이머, 게임의 품질 게임을 호스팅하는 하는 장치 간에 네트워크 통신의 품질에 영향을 받습니다. 저하 네트워크 대역폭이 부족 또는 대기 시간으로 인해 지연 또는 연결 삭제와 같은 게임 환경을 원하지 않는 시킬 수 있습니다.

QoS 참조 간의 모든 플레이어 충분 한 네트워크 연결 품질을 갖도록 플레이어 온라인 연결 (대역폭 및 대기 시간)의 강도 측정 합니다. 이것이 일치 하는 동안 net 인해 좋은 환경을 보장 하기 위해 매치 메이 킹 플레이어에 게 특히 중요 합니다. 덜 초대 친구 함께 재생 및 일반적으로 저하 연결의 영향을 감수할 위치에 적용 됩니다.

QoS 특정 조건에 따라 자동으로 처리 하는 세션을 구성할 수 또는 게임 누구 든 지 세션에 연결 될 때마다는 QoS 측정을 처리할 수 있습니다.
