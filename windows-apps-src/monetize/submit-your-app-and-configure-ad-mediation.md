# 앱 제출 및 광고 조정 구성


\[ Windows 10의 UWP 앱에 맞게 업데이트되었습니다. Windows 8.x 문서는 [보관](http://go.microsoft.com/fwlink/p/?linkid=619132)을 참조하세요. \]


사용하려는 모든 광고 네트워크를 포함하도록 앱을 빌드하여 모두 제대로 작동하는지 테스트한 후 앱을 제출할 수 있습니다. 제출 프로세스 중에 표시할 광고 조정 동작을 구성할 수 있습니다. 나중에 코드를 변경하거나 새 패키지를 제출하지 않고 이 동작을 조정할 수 있습니다.

## 앱 패키지의 기준 구성 만들기


패키지를 업로드하면 개발자 센터에서 사용자가 광고 조정을 사용 중인지를 자동으로 감지하여 사용 중인 광고 네트워크를 식별합니다. **패키지** 페이지에 **광고 조정 구성** 링크가 표시됩니다. 이 링크를 클릭하여 **Windows 광고 조정** 섹션에서 조정 논리를 구성할 수 있는 [광고를 통한 수익 창출](https://msdn.microsoft.com/library/windows/apps/mt170658) 페이지로 이동합니다. 이 페이지의 다른 섹션에 대한 자세한 내용은 광고를 통한 수익 창출을 참조하세요.

앱에 대해 처음으로 광고 조정 설정을 구성할 때 기준 구성을 만듭니다. 설정 후 다른 시장의 특정 광고 네트워크의 장점을 활용하도록 시장별 구성을 추가할 수 있습니다. 앱을 제출하기 전에 광고 조정 설정을 구성하지 않도록 선택할 경우 개발자 센터가 자동으로 [기본 설정](#default-ad-mediation-settings)을 적용하며, [나중에 해당 설정을 수정](#optimize-ad-mediation-after-the-app-has-been-published)할 수 있습니다.

다음 단계에서는 **광고를 통한 수익 창출** 페이지의 **Windows 광고 조정** 섹션에서 기준 구성을 만드는 방법에 관해 설명합니다.

1.  **조정 구성 대상**에서 구성하려는 앱 패키지를 선택했는지 확인합니다.
2.  **대상**에서 **기준**을 선택했는지 확인합니다.
3.  **새로 고침 빈도**에서 조정 주기의 길이(새 광고를 표시하는 빈도)를 선택합니다. 기간은 30초에서 120초 사이여야 합니다.
    **참고** 광고 네트워크 포털에서 새로 고침 빈도를 이미 구성한 경우 여기서 동일한 새로 고침 빈도를 설정해야 합니다.
4.  다음으로 **Windows 광고 조정** 섹션은 앱에 사용되는 모든 광고 네트워크를 나열하고, 앱이 각 광고 네트워크를 사용하는 빈도를 지정하는 두 가지 다른 방법을 제공합니다. **조정 유형** 드롭다운에서 다음 옵션 중 하나를 선택합니다.

    -   **가중치별 주문**. 앱에서 광고 네트워크를 사용하는 빈도를 지정하는 백분율 값을 각 광고 네트워크에 적용하려면 이 옵션을 선택합니다. 모든 광고 네트워크에 대해 설정하는 총 비율을 더하면 100%가 되어야 합니다. 자세한 내용은 [가중치별 광고 네트워크 주문](#order-ad-networks-by-weight)을 참조하세요.
    -   **순위별 주문**. 앱에서 각 광고 네트워크를 사용하는 빈도를 지정하는 순위를 1부터 *n*까지 각 광고 네트워크에 적용하려면 이 옵션을 선택합니다. 자세한 내용은 [순위별 광고 네트워크 주문](#order-ad-networkds-by-rank)을 참조하세요.

    개발자 센터는 앱에서 각 광고 네트워크를 사용하는 빈도를 지정하는 [기본 설정](#default-ad-mediation-settings)을 자동으로 적용합니다.

5.  앱에서 사용한 광고 네트워크 목록에서, 각 네트워크에 필요한 매개 변수를 보고 필수 매개 변수를 입력하려면 각 광고 네트워크 공급자의 드롭다운 화살표를 클릭합니다. 매개 변수 목록은 [광고 네트워크 선택 및 관리](select-and-manage-your-ad-networks.md)를 참조하세요.

    각 네트워크에 대해 확장된 매개 변수 목록에서, 선택적으로 **시간 제한** 필드를 사용하여 광고 네트워크에서 광고를 요청한 후 해당 요청을 중단하고 다른 네트워크에 요청할 때까지 광고 조정이 대기해야 하는 시간(2-120초)을 지정할 수 있습니다. 이미 [코드에서 이 값을 지정](add-and-use-the-ad-mediator-control.md#set-timeouts)한 경우 코드에서 지정한 값이 여기에서 지정한 값을 재정의합니다.

    Microsoft Advertising을 사용하는 경우 다음을 기록해 두세요.

    -   **Microsoft Advertising**의 매개 변수는 앱 패키지의 내용에 따라 채워집니다. **Microsoft Advertising**에 대해 사용자 고유의 **응용 프로그램 ID** 및 **광고 단위 ID**를 지정할 수도 있습니다.
    -   **Microsoft Advertising** 외에도 **Microsoft Advertising 하우스 광고**에 대한 행도 있습니다. 하우스 광고는 다른 앱에서 앱을 홍보하는 무료 광고를 만드는 방법을 제공합니다. 하우스 광고가 무료일지라도 하우스 광고의 할당은 다른 광고 네트워크와 동일한 풀에서 제공됩니다. 따라서 **Microsoft Advertising 하우스 광고**에 광고 요청을 할당한 경우 하우스 광고에 전체 광고 공급의 해당 비율을 사용하도록 선택합니다. 하우스 광고에 광고 요청을 할당하는 경우 개발자 계정에 등록한 하나 이상의 다른 앱에 대해서도 하우스 광고 캠페인을 만들어야 합니다. 자세한 내용은 [하우스 광고 정보](https://msdn.microsoft.com/library/windows/apps/mt148566)를 참조하세요.

6.  기본 구성을 저장하려면 **저장**을 클릭합니다.

### 가중치별 광고 네트워크 주문

앱에서 각 광고 네트워크를 사용하는 빈도를 지정하는 백분율 값을 지정하려면 **조정 유형** 드롭다운에서 이 옵션을 선택합니다. 모든 광고 네트워크에 대해 설정하는 총 비율을 더하면 100%가 되어야 합니다.

모든 네트워크에 요청을 똑같이 자동으로 분산하려면 **모든 활성 광고 네트워크에 똑같이 광고 요청 배포**를 클릭합니다. 요청을 동일하지 않게 분산하려면 앱에서 각각의 광고 네트워크를 사용하는 빈도를 나타낼 수 있는 슬라이더 컨트롤을 사용합니다. 모든 광고 네트워크에 대해 설정하는 총 비율을 더하면 100%가 되어야 합니다. 슬라이더 컨트롤을 끌 때 다음과 같은 여러 옵션을 사용할 수 있습니다.

-   숫자로 슬라이더를 끄는 경우 해당 숫자는 이 광고 네트워크가 조정 주기에서 앱의 첫 번째 옵션으로 호출될 시간의 백분율을 나타냅니다.
-   **백업**으로 슬라이더를 끄는 경우 이는 지정된 비율의 광고 네트워크에서 광고를 제공할 수 없는 경우에만 이 광고 네트워크를 호출함을 나타냅니다. 이는 백분율을 0%로 설정하는 것과 같습니다.
-   **비활성**으로 슬라이더를 끄는 경우 이는 이 광고 네트워크가 호출되지 않음을 나타냅니다. 광고 네트워크에 대한 어셈블리는 패키지에 남아 있지만 조정자는 해당 어셈블리를 호출하지 않습니다. 특정 시장에서 제대로 수행되지 않거나 지원하지 않는 것으로 알려진 광고 네트워크를 제외하려면 시장별 구성에서 이 옵션을 설정할 수 있습니다.
-   광고 네트워크에 대한 백분율을 조정할 때 **백업** 이외의 값을 선택한 다른 슬라이더 컨트롤은 전체 배포 합계가 100%가 되도록 자동으로 조정됩니다. 광고 네트워크에 대해 **잠금** 확인란을 선택하는 경우 해당 광고 네트워크가 현재 선택된 값에서 계속 고정됩니다. 그러면 잠긴 광고 네트워크에 대한 값에 영향을 주지 않으면서 다른 광고 네트워크에 대한 값을 조정할 수 있습니다.

### 순위별 광고 네트워크 주문

앱에서 각 광고 네트워크를 사용하는 빈도를 지정하는 순위를 1부터 *n*까지 각 광고 네트워크에 적용하려면 **조정 유형** 드롭다운에서 이 옵션을 선택합니다.

광고 네트워크에 대한 **활성** 확인란의 선택을 취소한 경우 이는 해당 광고 네트워크를 호출하지 않음을 나타냅니다. 광고 네트워크에 대한 어셈블리는 패키지에 남아 있지만 조정자는 해당 어셈블리를 호출하지 않습니다. 특정 시장에서 제대로 수행되지 않거나 지원하지 않는 것으로 알려진 광고 네트워크를 제외하려면 시장별 구성에서 이 옵션을 설정할 수 있습니다.

### 기본 광고 조정 설정

기본적으로 개발자 센터는 **광고를 통한 수익 창출** 페이지에서 앱에 다음 광고 조정 설정을 적용합니다. 이러한 동일한 기본값은 앱을 제출하기 전에 광고 조정을 구성하지 않도록 선택한 경우에도 적용됩니다.

-   앱에서 Microsoft Advertising을 사용하는 경우 **Microsoft Advertising**은 100(**가중치별 주문**을 선택한 경우) 또는 1(**순위별 주문**을 선택한 경우)로 설정되고, 다른 모든 광고 네트워크는 비활성으로 설정됩니다.
-   앱에서 Microsoft Advertising을 사용하지 않는 경우 모든 광고 네트워크는 비활성으로 설정됩니다.

## 새 대상 구성 만들기


앱의 기본 구성을 만든 후 특정 시장에서 사용할 추가 구성을 만들 수 있습니다. 이러한 새 구성은 기본 구성에서 초기 설정을 상속하지만, 앱에 대해 지원하는 특정 시장에 대한 세부 정보를 조정할 수 있습니다.

새 대상 구성을 만들려면:

1.  **대상**에서 새 구성을 만들려는 시장을 선택합니다.
2.  새로 고침 빈도 및 광고 요청 배포 정보를 원하는 대로 업데이트합니다.
3.  새 구성을 저장하려면 **저장**을 클릭합니다.

## 앱이 게시된 이후에 광고 조정 최적화


특정 앱에 대한 광고 조정을 조정하려면 앱을 다시 제출하지 않고 언제든지 조정할 수 있습니다. 이 기능은 계정을 아직 설정하지 않은 앱에 광고 네트워크를 이미 추가했거나, 광고 네트워크가 특정 시장에서 광고를 안정적으로 채울 수 없는 경우에 유용합니다. 시장별 구성뿐만 아니라 기준 구성도 변경할 수 있습니다. 원하는 경우 시장별 구성을 추가하거나 제거할 수도 있습니다.

앱의 앱 조정 설정으로 돌아가려면 다음 중 하나를 수행합니다.

-   대시보드의 계정 개요 페이지에서 **광고 조정** 섹션 아래의 앱을 클릭합니다.
-   대시보드의 앱 개요 페이지에서 **수익 창출**을 확장하여 **광고를 통한 수익 창출**을 클릭합니다.

## 광고 조정 설정 및 앱 업데이트


앱 업데이트를 제출할 때 앱의 기존 광고 조정 구성 정보는 다음 조건이 충족되는 경우 업데이트된 패키지에 자동으로 적용됩니다.

-   **AdMediatorControl** 컨트롤의 수는 이전 앱 패키지의 컨트롤 수와 같으며, 모든 컨트롤은 이전 앱 패키지와 ID가 같습니다.
-   광고 네트워크 공급자 목록이 이전 앱 패키지와 같습니다.

이러한 조건 중 하나가 충족되지 않으면 앱에 대한 기본 구성 및 해당 시장별 대상 구성을 다시 만들어야 합니다.

**참고** **AdMediatorControl**의 ID는 앱에서 컨트롤을 디자인 화면으로 끌 때 생성됩니다. 이 ID는 컨트롤을 삭제한 후 동일한 디자인 화면으로 새 컨트롤을 끌어서 교체하지 않는 한 변경되지 않습니다.

 

## 통합 개발자 센터 대시보드에서 광고 조정 보고서 검토


광고 조정 보고서를 검토하려면 개발자 센터에서 **분석** 섹션으로 이동하여 **광고 조정**을 클릭합니다. 이 보고서에 대한 자세한 내용은 [광고 조정 보고서](https://msdn.microsoft.com/library/windows/apps/mt148521)를 참조하세요.

## 예제 시나리오


다양한 목표와 시나리오를 지원하도록 다양한 방법으로 광고 네트워크를 구성할 수 있습니다. 아래 예제에서는 **가중치별 주문**을 사용하여 세 개의 광고 네트워크가 포함된 광고 조정을 구성하는 방법을 보여 줍니다. 여기서는 Microsoft Advertising, Inneractive 및 AdDuplex를 예제 네트워크로 사용합니다.

### 예제 1

최적화를 시작하기 전에 모든 네트워크에 대한 유효 노출률과 경비 효율 보기(eCPM)를 확인할 수 있습니다.

시작하려면 중재 트래픽을 균등하게 분배하도록 각 네트워크를 설정합니다. 나중에 각 네트워크의 보고서를 검토하여 각 시장에서 성능이 가장 우수한 광고 네트워크를 파악할 수 있습니다.

며칠 또는 몇 주 후에 각 광고 네트워크 포털에서 채우기 속도 및 eCPM을 확인할 수 있습니다. 그러면 시장별로 가장 적합한 광고 네트워크를 파악할 수 있습니다. 그런 다음 새 패키지를 제출할 필요 없이 특정 시장 또는 전체 시장에 맞게 조정할 수 있습니다.

### 예제 2

가능한 경우 Microsoft Advertising을 먼저 사용하려고 합니다. Microsoft Advertising이 광고를 제공할 수 없는 경우에 대비하여 다른 광고 네트워크(우선 순위 없음)를 백업으로 사용하는 것이 좋습니다.

| 광고 네트워크            | 구성 |
|-----------------------|---------------|
| Microsoft             | 100%          |
| Inneractive           | 백업        |
| AdDuplex              | 백업        |

### 예제 3

일반적으로 Microsoft Advertising을 먼저 사용합니다. Microsoft Advertising이 광고를 제공할 수 없는 경우 Inneractive를 AdDuplex보다 먼저 호출하는 것이 좋습니다.

| 광고 네트워크            | 구성 |
|-----------------------|---------------|
| Microsoft             | 90%           |
| Inneractive           | 10%           |
| AdDuplex              | 백업        |

### 예제 4

Microsoft Advertising과 Inneractive를 균등하게 사용하려고 하므로, 하나의 네트워크를 항상 먼저 호출하는 경우에 비해 앱에 더 많은 광고가 표시됩니다. 두 광고 네트워크 모두를 사용할 수 없는 경우 AdDuplex를 백업으로 사용합니다.

| 광고 네트워크            | 구성 |
|-----------------------|---------------|
| Microsoft             | 50%           |
| Inneractive           | 50%           |
| AdDuplex              | 백업        |
