## <a name="session-overview"></a>세션 개요

### <a name="what-is-mpsd"></a>MPSD 란 무엇 인가요?

멀티 플레이 세션 디렉터리 (MPSD)에 여러 클라이언트에 걸쳐 게임의 멀티 플레이 시스템 메타 데이터를 제공 하는 Xbox Live 클라우드에서 운영 하는 서비스입니다. **MultiplayerService 클래스**에 의해 래핑됩니다.

MPSD 제목을 사용자 그룹에 연결 하는 데 필요한 기본 정보를 공유할 수 있습니다. 동기화 된이 고 일관 된 세션 기능 인지 확인 합니다. 초대 보내기/수락 및 게이머 카드를 통해 가입 되 고 셸 및 콘솔 운영 체제와 동등 합니다.


### <a name="mpsd-sessions"></a>MPSD 세션

MPSD 세션은 하나 이상의 사용자가 게임을 사용 하는 시나리오와 **MultiplayerSession 클래스** 에 의해 표시 됩니다. 세션에 Xbox Live 클라우드 보안 JSON 문서로 MPSD로 저장 됩니다. 특히 MPSD 세션에는 다음과 같은 특징: 만들어지고 타이틀에 의해 관리 됩니다.

-   고유한 URI에 있습니다. 자세한 내용은 **세션 디렉터리 Uri**를 참조 하세요.
-   세션 멤버를 호출 하는 사용자가 연결 수 있습니다.
-   게임을 사용 하도록 설정 하는 데 유용 데이터를 저장 재생 예를 들어 멤버 당 특성, 게임 설정, 정보와 게임 서버 정보 부트스트랩 합니다.

MPSD 멀티 플레이어 게임 설정에 사용 하기 위해 여러 세션 변형을 지원 합니다. 모든 세션 플레이어의 Xbox 사용자 id (XUIDs)를 포함 하 고 장치 연결 주소 데이터를 보호 합니다.

-   게임 세션, 게임 플레이 패턴으로 사용 합니다. 피어 투 피어, 피어와 호스트, 피어-서버 또는 이러한 형식의 하이브리드 게임 세션 수 있습니다.
-   티켓 세션을 도우미 세션 동안 매치 메이 킹 일치 하는 상태를 추적 하는 데 사용 합니다. 로비 세션을 종종 이기도 하 고 게임 세션 경우가 있습니다. [스마트 매치](smartmatch-matchmaking.md)를 참조 하세요.
-   일치 하는 게임 플레이 나타내는 매치 메이 킹 만들어진 도우미 세션의 대상 세션 합니다. 이기도 거의 항상 게임 세션 합니다. [스마트 매치](smartmatch-matchmaking.md)를 참조 하세요.
-   로비 세션에 초대 된 게임 세션에 참가 대기 중인 플레이어를 수용 하기 위해 사용 하는 도우미 세션 여러 제목을 로비 세션과 게임 세션을 만듭니다. 자세한 내용은 **타이틀에서 관리 플레이어**를 참조 하세요.

## <a name="mpsd-change-notification-handling-and-disconnect-detection"></a>MPSD 알림 처리를 변경 하 고 감지를 분리 합니다.

MPSD 클라이언트가 실시간 활동 서비스 웹 소켓을 사용 하 여 연결할 수 있습니다. 연결이 하는 데 사용 합니다.

-   간략 한 알림 (어깨 임) 전송 세션 변경 되 면 시작 제목 이벤트 구독에 따라 합니다.
-   사용자 연결 끊기 감지 합니다.
-   비활성으로 사용자를 설정 하 고 이후에 기반 세션에서 제거 감지 연결 해제 합니다.


### <a name="making-user-connections"></a>사용자 연결 만들기

XSAPI 라이브러리 클라이언트와 MPSD 간의 연결을 관리합니다. 먼저 제목 **MultiplayerService.EnableMultiplayerSubscriptions 메서드**를 호출합니다. 이 메서드는 클라이언트 멀티 플레이 목적을 위해 실시간 활동 연결을 사용 하려는 XSAPI 지시 합니다. 그런 다음 제목 활성 상태로 설정 현재 사용자와의 첫 번째 **MultiplayerService.WriteSessionAsync 메서드** 또는 **MultiplayerService.WriteSessionByHandleAsync 메서드**를 호출 하면 연결이 생성 되 고 듣지 MPSD 하려면.

| 참고                                                                                                                         |
|-------------------------------------------------------------------------------------------------------------------------------------------|
| 세션 알림을 사용 하려면 세션 템플릿에 connectionRequiredForActiveMembers를 true로 설정 하는 연결이 검색. |

| 참고                                                                                                                                                                                                                                                                                                                            |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 이전 버전의 XSAPI, 제목 실시간 활동 서비스 사용자 연결을 만들고자 **RealTimeActivityService.ConnectAsync 메서드** 를 호출 합니다. 2015 멀티 플레이어에 대 한이 메서드를 없으며 주문형 연결이 만들어집니다. |

### <a name="subscribing-for-session-changes"></a>세션 변경에 대 한 구독

MPSD 관심 영역이 변경 되었음을 경량 알림으로 "shoulder 탭"를 사용 합니다. 제목을 변경의 정확한 특성을 확인 하 여 수정 된 리소스를 검색 해야 합니다. 구독 활성화를 사용 하 여 제목 어깨 **MultiplayerSession.SetSessionChangeSubscription 메서드**를 호출 하 여 세션 변경 내용 탭에 대 한 구독할 수 있습니다. 참조 [하는 방법: MPSD 세션 변경 알림을 등록](multiplayer-how-tos.md).


### <a name="handling-shoulder-taps"></a>탭 어깨 처리

해당 세션에 대 한 제목의 구독 일치 하는 세션에 대 한 변경 MPSD **RealTimeActivityService.MultiplayerSessionChanged 이벤트**를 사용 하 여 변경의 제목을 알립니다. 제목 해야 세션 검색 및 비교 이전에 캐시 된 보기를 사용 하 여 세션의 검색 된 버전을 적절 하 게 작업을 수행 합니다.


### <a name="handling-notifications-of-connection-state-changes"></a>연결 상태 변경 알림 처리

제목 변경 MPSD에 대 한 연결의 상태에 대 한 알림을 받을 수도 있습니다. 두 이벤트 신호 이러한 변경: * * RealTimeActivityService.MultiplayerSubscriptionsLost 이벤트 * * – 타이틀의 연결을 실시간 활동 서비스를 사용 하 여 MPSD 손실 될 때 발생 합니다. 이 이벤트가 발생 하면 제목은 멀티 플레이어 종료 해야 합니다.
-   * * RealTimeActivityService.ConnectionStateChanged 이벤트 * * 임시 타이틀의 연결을 실시간 활동 서비스의 상태 변경 시 발사-합니다. 제목이이 이벤트를 받으면 아무 작업도 수행할 필요가 없습니다 하지만 이벤트 진단용으로 사용 하는 데 유용할 수 있습니다.


### <a name="disconnecting-clients"></a>클라이언트 연결 끊기

타이틀에 대 한 클라이언트 **RealTimeActivityService.DisableMultiplayerSubscriptions 메서드**를 호출 하 여 알림을 비활성화 하면 제목은 MPSD에서 분리 합니다. 이 호출 직후 **MultiplayerSubscriptionsLost** 이벤트 발생을 나타내는 클라이언트 MPSD에서 분리가 합니다.

| 참고                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 멀티 플레이 이전 버전에서는 제목 호출의 * * RealTimeActivityService.Disconnect 메서드 * * 실시간 활동 서비스에서 분리 하 합니다. 이 메서드는 2015 멀티 플레이어 아무 않습니다. 연결을 끊은 **DisableMultiplayerSubscriptions** 이 호출 된 후, 사용자가 웹 소켓 연결을 예를 들어 있는 경우 현재 상태에 대 한 실시간 활동 서비스 구독이 자동으로 발생 합니다. |


### <a name="disconnect-detection"></a>감지를 분리 합니다.

MPSD 사용 하 여 해당 빨리 확인 하려면 사용자가 비정상적 연결이 때 검색 기능을 분리 합니다. 비정상적 연결 끊기 플레이어의 네트워크에 실패 하거나 제목을 충돌할 때 발생할 수 있습니다. MPSD는 비활성, 활성 연결이 끊긴된 플레이어의 상태를 변경 하 고 세션에 구성원의 구독으로 기반으로 적절 하 게 변경의 다른 세션 구성원에 게 알립니다.

## <a name="mpsd-handles-to-sessions"></a>MPSD 세션 핸들

MPSD 세션 핸들은 또한 추가 형식의 데이터를 포함할 수 있는 세션에 사용 되는 변경 불가능 추상 참조입니다. 파일 핸들 비슷합니다. 모든 핸들 핸들 ID (GUID)로 구성 된 서비스 구성 ID (서비스 안내), 세션 템플릿 이름과 세션 전체 세션 참조를 갖고 있습니다. 핸들을 업데이트할 수 없으므로 하지만 생성, 읽고, 및 수 삭제 합니다.

| 참고                                                                                                                                   |
|-----------------------------------------------------------------------------------------------------------------------------------------------------|
| 존재 하지 않는 세션에 대 한 핸들을 가리킬 수 있습니다. 존재 하지 않는 세션 이름을 사용 하 여 핸들을 만들고 새 세션을 만들 수는 발생 하지 않습니다. |


### <a name="handle-types"></a>핸들 유형

멀티 플레이 2015에는이 섹션에 설명 된 핸들 형식과 지원 합니다.

#### <a name="invite-handle"></a>핸들 초대

초대 핸들 특정 사용자를 초대를 (초대)를 나타냅니다. 형식별 데이터 소스 사람과 대상 사람 초대, 예를 들어 특정 게임 모드를 설명 하는 상황에 맞는 문자열을 포함 합니다.

초대 핸들 열려 세션에 대 한 읽기 / 쓰기 액세스 권한을 부여합니다. 세션 닫혀 있는 경우 핸들 세션을 읽기 전용 액세스 권한을 부여 합니다.

| 참고                                                |
|------------------------------------------------------------------|
| MPSD 세션 전체 또는 닫힌 경우에 초대를 만들 수 있습니다. |


#### <a name="activity-handle"></a>활동 핸들

현재 사용자가 수행 활동 핸들을 나타냅니다. 사용자의 활동 **MultiplayerActivityDetails 클래스**에 의해 표시 됩니다.

| 참고                                                                                                                                                                                                                      |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 활동 핸들도 명시적으로 삭제할 수만 특정 사용자에 대 한 소유 제목으로 합니다. 이 삭제 이루어집니다 **MultiplayerService.ClearActivityAsync 메서드**를 사용 합니다. |


### <a name="creating-an-invite-handle"></a>초대 핸들 만들기

초대 핸들을 만들려면 제목 **MultiplayerService.SendInvitesAsync 메서드**를 호출 합니다. 메서드를 초대를 보냅니다 지정된 된 사용자는 UI의 숫자 형태로 받는 사람 초대를 수락 하려면에서 작동할 수 있는 알림.


### <a name="creating-an-activity-handle"></a>활동 핸들 만들기

활동 핸들을 만들려면 제목 **MultiplayerService.SetActivityAsync 메서드**를 호출 합니다. MPSD 세션 멤버의 바인딩된 활동으로 새 핸들 ID를 설정합니다. 이전에 바인딩된 활동 되었으면 MPSD 해당 핸들을 삭제 합니다. 활성 구성원 비활성화 들어가거나 세션에서 때 MPSD 바인딩된 활동 핸들을 삭제 합니다.


### <a name="using-handles"></a>핸들을 사용 하 여

사용자가 (초대 핸들) 초대를 수락 및 친구의 현재 활동 (활동 핸들) 가입할 경우 제목 핸들을 사용 합니다. 이러한 경우 모두 제목 해야합니다.

1.  정품 인증 매개 변수를 제목에서 핸들 ID를 가져옵니다.
2.  로컬 MPSD 세션 개체를 만들고 활성으로 가입 합니다.
3.  적절 한 핸들을 전달 하 여 세션을 작성 합니다.

## <a name="synchronization-of-session-updates"></a>세션 업데이트 동기화

세션은 해당 구성원 중 하나에 의해 생성 하거나 업데이트할 수 있는 공유 리소스 이기 때문에 충돌 하는 쓰기에 대 한 가능성이 됩니다. 예기치 않은 결과가 발생할 수, 예를 들어 하나의 제목 다른 제목을가 변경한 덮어쓸 자신도 모르게 수 있습니다. MPSD의 이러한 충돌 해결 방법은 최적 동시성 및 읽기-수정-쓰기 패턴을 지원 하도록 합니다.

MPSD 세션 업데이트 동기화 두 관련된 고급 구현 패턴을 사용 합니다.

-   중재자 세션의 공유 부분을 업데이트합니다. 단일 중재자 관련 구현 하는 경우 대부분의 쓰기 작업에 대 한 동기화 된 업데이트를 사용 하 여 방지할 수 있습니다. 제목은 중재자 중재자의 id를 통신 관련이 하지 않는 한 세션 부분을 공유 하는 (1) 모든 업데이트 및 세션 내에서 멤버 영역을 사용 하면 제목 (2) 모든 업데이트에 대 한 동기화를 피할 수 있습니다.

| 참고                                                                                                                                                                                                                                                                                                                                              |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 하지만 위와 업데이트 동기화가 필요 하지 않은 것이 여전히 중요 업데이트를 동기화 하는 * * MultiplayerSessionProperties.HostDeviceToken 속성 * *. 해당 속성은 중재자 마이그레이션의 일부로 예를 들어 중재자 편안 하 게 식별 하는 데 사용 됩니다. |

-   모든 클라이언트 세션의 공유 부분을 업데이트합니다. 이 경우 세션 부분을 공유 하는 모든 업데이트 동기화 되어야 합니다. 그러나 제목 동기화 하지 않고 자신의 멤버 영역에 계속 작성할 수 있습니다.


### <a name="session-update-synchronization-using-the-multiplayer-winrt-api"></a>멀티 플레이 WinRT API를 사용 하 여 세션 업데이트 동기화

다음 멀티 플레이 WinRT API 메서드는 최적 동시성을 구현합니다.

-   **MultiplayerService.WriteSessionAsync 메서드**
-   **MultiplayerService.WriteSessionByHandleAsync 메서드**
-   **MultiplayerService.TryWriteSessionAsync 메서드**
-   **MultiplayerService.TryWriteSessionByHandleAsync 메서드**

각 쓰기 메서드 **MultiplayerSessionWriteMode 열거형** 값을 수락합니다. SynchronizedUpdate 값을 전달 사용 최적 동시성 업데이트 합니다.

다른 열거형 값 세션의 초기 작성 시 잠재적인 충돌을 해결 하는 데 도움이 됩니다. 잠재적으로 다른 제목에 쓸 수 있는 MPSD 세션의 모든 쓰기 동기화 된 업데이트를 사용 해야 합니다. 그러나 모든 쓰기 보호 되어야 합니다.

타이틀 MPSD 세션 쓰기 방법 중 하나를 사용 하 여 로컬 세션 개체에 쓰려고 시도 하는 HTTP/412 상태 코드를 받은 경우 로컬 복사본 가져오려면 **MultiplayerService.GetCurrentSessionAsync 메서드** 호출을 실행 하 여 고쳐야 합니다 다시 쓰기를 시도 하기 전에 세션의 최신 서버 버전입니다. 그렇지 않으면 로컬 세션 문서 잘못 된 데이터를 포함 하 계속 하 고 세션 작성에 대 한 호출을 계속 실패 합니다.

| 참고                                                                                                                                                                                  |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 제목 **TryWrite\ *** 메서드 중 하나를 사용 하는, 업데이트 된 세션에서 HTTP/412 상태 코드의 경우 반환 됩니다. 이 동작은 방지 **GetCurrentSessionAsync**호출 해야 합니다. |

제목 쓰기 세션 방법 중 하나를 호출 하면 세션의 업데이트 된 버전을 반환 될 수 있습니다. 인 경우, 제목은 스레드로부터 안전한 방식으로 로컬 캐시 된 복사본을이 새 버전으로 전환 됩니다.


### <a name="session-update-synchronization-using-the-multiplayer-rest-api"></a>멀티 플레이 REST API를 사용 하 여 세션 업데이트 동기화

MPSD는 ETag 설정 사용 하 여 HTTP "일치 경우" 헤더와 읽기-수정-쓰기 패턴을 사용 하 여 나머지 기능을 통해 세션 업데이트에 최적 동시성을 지원 합니다. ETag 쓰기 요청에 전달 되는 MPSD 이전 읽기 요청으로 반환 하는 것 이어야 합니다.

## <a name="calling-mpsd"></a>MPSD 호출

멀티 플레이 시스템 및 매치 메이 킹 사용 하기 위해 MPSD 액세스 하려면 제목에 대 한 두 가지가 있습니다.
-   권장 합니다. 역할을 RESTful 기능에 대 한 래퍼 클래스를 포함 하는 멀티 플레이어 WinRT API를 사용 합니다. **Microsoft.Xbox.Services.Multiplayer Namespace**를 참조 하세요. 스마트 매치 매치 WinRT API **Microsoft.Xbox.Services.Matchmaking Namespace**개체로 사용 합니다.
-   표준 HTTP를 직접 사용 하 여 멀티 플레이어 및 **Xbox Live Services RESTful 참조**에 포함 된 매치 REST Api를 호출 합니다. 해당 Uri (멀티 플레이어)에 대 한 **세션 디렉터리 Uri** 및 **매치 메이 킹 Uri** (매치 메이 킹)에 대 한 설명 되어 있습니다. 관련 된 JSON 개체는 **JavaScript 개체 Notation (JSON) 개체 참조**에 설명 되어 있습니다.


### <a name="using-the-multiplayer-winrt-api-to-call-mpsd"></a>멀티 플레이 WinRT API를 사용 하 여 MPSD 호출

MPSD를 호출 하 여 멀티 플레이 WinRT API 및 매치 메이 킹 WinRT API를 사용 하는 것이 좋습니다.

| 참고                                                                                             |
|---------------------------------------------------------------------------------------------------------------|
| XDK 샘플 XSAPI의 다른 요소는 멀티 플레이어 및 매치 메이 킹 WinRT Api를 사용 하 여 기록 됩니다. |

기본 REST 기능에 대 한 래퍼 코드 사용 보다 일반적인 접근 방식을 알아보려면 클라이언트 쪽 API 메서드를 사용 하 여 각 호출에 대 한 HTTP 트래픽을 처리 하지 않고도 수 있습니다. 이진와 XSAPI에 대 한 소스는 XDK 함께 제공 됩니다. 바이너리를 직접 사용 하 고 또는 소스를 수정 하 고, 필요에 따라 타이틀에 통합할 수 있습니다.


### <a name="using-the-multiplayer-rest-api-to-interact-with-mpsd"></a>멀티 플레이어 REST API를 사용 하 여 MPSD 상호 작용할 수

제목, 또는 해당 서비스 멀티 플레이 REST API 및 매치 메이 킹 REST API에 대 한 표준 HTTP 호출에 사용할 수 있습니다. 경우 나머지 기능을 직접 사용 하 여, 호출자에 게 문제 삭제, 배치, 게시을 세션 디렉터리 Uri에 대해 호출 대부분의 작업에 대 한 합니다. PUT 요청에서 요청 본문은 기존 세션에 병합 됩니다. 기존 세션에 없는 경우 요청 본문 [Xbox 개발자 포털 (XDP)](https://xdp.xboxlive.com) 또는 [Windows 개발자 센터](https://developer.microsoft.com/dashboard/windows/overview)에 저장 된 세션 템플릿 함께 새 세션을 만드는 데 사용 됩니다. 모든 필드는 선택 사항, 및 유일한 델타를 지정 해야 합니다. 따라서 {} 은 0 델타를 사용 하 여 유효한 PUT 요청 합니다.

세션의 공식 복사본 서버의 영향을 주지 않고 병합의 결과 반환 하는 가상 PUT 요청을 수행 하려면 쿼리 문자열을 추가할 수 있습니다 "? nocommit = true" PUT 요청 합니다.

요청 및 응답 멀티 플레이어 및 매치 메이 킹 REST API 메서드는 JSON 문서입니다. 멀티 플레이 세션 요청 구조를 **MultiplayerSessionRequest (JSON)를**참조 하세요. 관련된 응답 구조 **MultiplayerSession (JSON)** 에 표시 됩니다. 응답 구조 연결된 목록으로 세션 멤버 프레임 고 세션 및 해당 멤버의 다른 읽기 전용 속성 채워집니다.


### <a name="querying-for-sessions-and-session-templates-rest"></a>세션 및 세션 템플릿 (REST)에 대 한 쿼리

서비스 구성 및 세션 템플릿 수준에서 세션 정보 제목 쿼리할 수 있습니다. 이 항목에서는 멀티 플레이 REST API를 사용 하는 쿼리를 설명 합니다.


#### <a name="query-for-basic-session-information"></a>기본 세션 정보에 대 한 쿼리

세션 디렉터리와 매치 메이 킹 Uri 사용 하 여 기본 세션 정보에 대 한 쿼리를 설정할 수 있습니다. 쿼리 결과는 일부 세션 데이터를 사용 하 여 참조 인라인을 포함 하는 세션의 JSON 배열입니다. 기본적으로 쿼리 최대 100 개인 세션을 검색합니다.

| 참고                                                          |
|----------------------------------------------------------------------------|
| 모든 쿼리 키워드 필터는 XUID 필터 또는 둘 다 포함 해야 합니다. |


#### <a name="query-for-session-templates"></a>세션 템플릿에 대 한 쿼리

세션 템플릿 목록 서비스 안내, 뿐만 아니라 특정 세션 템플릿의 세부 정보를 검색 하려면 다음 Uri 중 하나에 대해 GET 메서드를 사용 합니다.
-   **/serviceconfigs/{scid}/sessiontemplates**
-   **/serviceconfigs/{scid}/sessiontemplates/{sessionTemplateName}**


#### <a name="query-for-session-state"></a>세션 상태에 대 한 쿼리

세션 상태에 대 한 쿼리를 하려면 이러한 Uri 중 하나에 대해 GET 메서드를 사용 합니다.
-   **/serviceconfigs/{scid}/sessions**
-   **/serviceconfigs/{scid}/sessiontemplates/{sessionTemplateName}/sessions**

멀티 플레이어 세션 탐색기에 액세스, Xbox One에서 Internet Explorer를 열고, **보기** 단추 및 입력 <https://sessiondirectory.xboxlive.com/debug> **주소** 필드에서.

### <a name="using-multiplayer-session-explorer"></a>멀티 플레이 세션 탐색기를 사용 하 여


#### <a name="open-the-main-page"></a>열고 메인 페이지

1.  도구의 기본 페이지에 액세스 합니다. 샌드박스에서 보안 컨텍스트 (사용자 로그인 및 샌드박스) 및 서비스 구성 Id (SCIDs)의 목록을 표시합니다.
2.  때마다 URI를 입력 하지 않아도 되도록이 홈 페이지를 고정 **메뉴** 단추를 누릅니다.


#### <a name="display-available-sessions-and-templates"></a>사용 가능한 세션 및 템플릿 표시

1.  로그인 사용자 멤버인 해당 서비스 안내에서 세션의 목록을 표시 하는 서비스 도구에서 안내를 클릭 합니다.
2.  이 동일한 페이지에는 서비스 안내 클릭 하 고 세션 템플릿 및 지역화 문자열은 서비스 안내에 대 한 서비스 구성에 표시 수 있습니다. 이러한 항목은 [XDP](https://xdp.xboxlive.com) 또는 [Windows 개발자 센터](https://developer.microsoft.com/dashboard/windows/overview)를 통해 수집 됩니다.


#### <a name="display-the-full-contents-of-a-session"></a>세션의 전체 내용을 표시합니다

멀티 플레이어 세션 탐색기에서 해당 세션의 전체 내용을 표시 하는 세션 이름을 클릭 합니다.

MPSD 하 여 표시 된 대로 세션에 따라 달라 표준 GET 메서드 응답의 세션의 URI에 대 한 몇 가지 이유:

-   GET 호출 Xbl 계약 버전 X 헤더에서 이전 계약 버전을 사용 될 있습니다. 세션 탐색기 계약의 최신 버전을 사용 하 여 세션을 항상 표시 됩니다.
-   때 세션은 일반적으로 통해 요청 GET, 변환 및 부작용 트리거될 수, 예를 들어 만료 시간 제한 합니다. 세션 탐색기 저장, 실행 모든 논리, 변환, 또는 의도 하지 않고 세션의 스냅숏을 표시 됩니다.
-   부작용으로 동시에 nextTimer JSON 개체 필드가 계산 되므로 MPSD 세션에 없습니다.
