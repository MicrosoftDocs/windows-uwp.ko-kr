# <a name="account-types-locations-and-fees"></a>계정 유형, 위치 및 요금

[파트너 센터](https://partner.microsoft.com/dashboard) 의 Windows 프로그램은 여러 [국가 및 지역](#developer-account-and-app-submission-markets)에서 개인 계정과 회사 계정을 모두 제공 합니다. 두 가지 유형의 계정 모두 스토어에 앱을 게시 하 고 추가 Microsoft 프로그램에 참여할 수 있는 액세스 권한을 제공 합니다.

> [!NOTE]
> 개발자 계정을 열 때 개발자 계정에 사용 하려는 Microsoft 계정를 사용 하 여 [파트너 센터](https://partner.microsoft.com/dashboard) 에 로그인 해야 합니다. 나중에 [계정에 사용자를 더 추가할](manage-account-users.md) 수 있는 옵션이 제공 됩니다.

[등록 프로세스를 시작](https://developer.microsoft.com/store/register)할 때는 개인 계정을 만들지 또는 회사 계정을 만들지 여부를 선택해야 합니다. 두 가지 유형의 계정을 모두 사용 하 여 앱, 게임, 추가 기능 및 서비스를 제출할 수 있습니다.

회사 계정은 조직 및 비즈니스용입니다. 회사 계정의 경우 회사의 계정을 설정할 권한이 있는지 확인 하기 위해 더 높은 [확인](#account-verification) 이 필요 합니다. 개별 계정은 계정 사용자에 게 다른 계정 사용자를 추가 하는 옵션을 사용 하는 경우에도 일반적으로 고유한 단일 개발자에 게 적합 합니다.

> [!IMPORTANT]
> 사용자가 선택한 후에는 계정 유형을 변경할 수 없습니다. 따라서 올바른 유형을 선택 해야 합니다.

두 계정 유형 간의 중요 한 차이점은 다음과 같습니다.

| 개별 계정 | 회사 계정 |
|--------------------|-----------------|
| <ul><li>약 $19 USD (일회용 등록 요금, 정확한 금액은 사용자의 국가 또는 지역에 따라 다름)</li><li>짧은 계정 확인 프로세스</li></ul> | <ul><li>약 $99 USD (일회용 등록 요금, 정확한 금액은 사용자의 국가 또는 지역에 따라 다름)</li><li>더 높은 계정 확인 필요</li><li>회사가 있는 국가 또는 지역에서 회사가 인식 되어야 합니다.</li><li>[Microsoft Store 정책](store-policies.md#1014-account-type)에 설명 된 대로 제한 된 기능으로 앱을 제출할 수 있습니다.</li></ul> |

회사 계정은 회사를 표시 하 고 계정을 설정할 수 있는 권한이 있는지 확인 하기 위해 몇 가지 추가 단계를 수행 하기 때문에 대부분의 비용이 더 많이 듭니다. [매장 정책](store-policies.md#1014-account-type)에 따라 재무 계정 정보에 액세스 하거나 보안 전용 타사 인증 공급자를 사용 하지 않고 기본 기능에 액세스 하기 위해 인증이 필요한 특정 기능의 앱은 회사 계정 으로만 게시할 수 있습니다.

## <a name="account-verification"></a>계정 확인

등록 프로세스를 완료 한 후에는 계정이 확인 프로세스를 거칩니다. 개별 계정에 대해 다른 회사에서 아직 게시자 표시 이름을 사용 하 고 있지 않은지 확인 합니다. 회사 계정의 경우 확인 파트너 Dun & Bradstreet에서 표시 하는 회사에 대 한 계정을 만들 수 있는 권한이 있는지 확인 합니다. 이 프로세스는 며칠에서 몇 주까지 걸릴 수 있으며, 회사에 대 한 전화 통화를 포함 하는 경우가 많습니다. 따라서 등록 양식을 작성할 때 모든 연락처 정보를 최신 상태로 유지 해야 합니다. 확인 되기 전 까지는 회사 계정에서 앱을 제출할 수 없지만, 대기 하는 동안 [앱 이름을 예약](create-your-app-by-reserving-a-name.md) 하 고 앱을 계속 빌드하고 테스트할 수 있으며, 제출 준비가 완료 될 수 있습니다.

**계정 설정** 페이지에서 확인 상태를 확인할 수 있습니다.
