# <a name="set-add-on-pricing-and-availability"></a>추가 기능 가격 책정 및 가용성 설정

추가 기능을 제출 하는 경우 [파트너 센터](https://partner.microsoft.com/dashboard)에서 옵션을 **가격 책정 및 가용성** 페이지 결정 얼마나 추가 기능에 대 한 고객 및 고객에 게 제공 해야 하는 방법에 요금이 부과 됩니다.

## <a name="markets"></a>지역/국가

기본적으로 추가 기능은 나중에 추가될 수 있는 잠재 지역/국가를 포함하여 모든 가능한 지역/국가에 기본 가격으로 나열됩니다.

그러나 앱과 마찬가지로 추가 기능을 제공하려는 지역/국가를 선택할 수 있습니다. 대부분은 앱과 같은 지역/국가를 선택하려고 하지만 필요에 따라 변경할 수도 있습니다. 

자세한 내용과 출시 시장의 전체 목록을 보려면 [시장 선택 정의](define-pricing-and-market-selection.md)를 참조하세요.

## <a name="visibility"></a>표시 여부

고객에게 추가 기능을 구매하도록 제공할지 결정할 수 있습니다. 

기본 옵션은 **상위 제품의 Store 목록에 표시 가능**입니다. 모든 고객이 사용할 수 있도록 설정할 추가 기능에는 이 옵션을 선택해 둡니다. 

추가 기능을 광범위하게 제공하고 싶지 않은 경우에는 **스토어에서 숨김**과 다음 옵션 중 하나를 선택합니다.

-   **만 부모 제품 내에서 구입할 수**: 이 옵션을 선택 하면 앱 내에서 추가 기능을 구매 하는 모든 고객을 허용 하지만 추가 기능을 앱 스토어 목록에 표시 되거나 저장소에서 검색할 수 없습니다. 내부 테스트의 초기 기간과 같이 제품을 광범위하게 사용할 수 없는 경우에만 이 옵션을 사용합니다.
-   **취득을 중지 합니다. 직접 링크를 사용 하 여 모든 고객을 나열 하는 제품의 저장소를 볼 수 있지만 이러한 이전에 제품을 소유 또는 프로 모션 코드를 Windows 10 장치를 사용 하는 경우만 다운로드할 수 있습니다. 이 추가 기능 부모 제품의 목록에 표시 되지 않으면**: 이 옵션을 선택하면 추가 기능이 앱 목록에 표시되지 않고 새로운 고객이 추가 기능을 구매할 수 없습니다. 그러나 **고객은 Windows 8.1 또는 이전 버전에 대 한이 옵션이 지원 되지 않습니다**합니다. 이전에 게시 된 앱 Windows 8.1 사용할 수 있거나 이전 이면 추가 기능 구매 고객에 대 한 사용할 수 있습니다. Windows 8.1 또는 이전 고객에 게 추가 기능을 제공을 중지 하려면 앱에 대 한 새 제출물을 게시 한 다음 추가 기능을 제공 하는 코드를 제거 하려면 앱을 업데이트 해야 합니다. 이 앱에는 Windows 8.1 또는 이전 대상 하지 하는 경우에 좋습니다. 것이 고객에 게 더 나은 환경을 사용할 수 있도록 설정 되어 있는 추가 기능이 제공 되지 않습니다.
    
 > [!NOTE] 
 > **취득 중지** 옵션을 선택하거나 앱 코드에서 추가 기능을 제거하는 앱 업데이트를 제출해도 운영 체제와 관계없이 이미 추가 기능을 구매한 고객에게는 영향을 주지 않습니다.


## <a name="schedule"></a>일정

기본적으로 (**표시 여부** 섹션에서 **스토어에서 숨김** 옵션 중 하나를 선택한 경우를 제외하고) 앱 인증과 게시 프로세스가 완료되는 즉시 고객이 앱을 사용할 수 있습니다. 다른 날짜를 선택 하려면 **옵션 표시**를 선택해 이 섹션을 확장하세요. 

자세한 내용은 [정확한 릴리스 일정 구성](configure-precise-release-scheduling.md)을 참조하세요.


## <a name="pricing"></a>가격 책정

추가 기능에 대 한 기본 가격을 선택 해야 합니다 (선택 하지 않으면를 **취득 중지** 옵션을 **가시성** 섹션). 기본 선택 사항은 **무료**이므로 추가 기능에 대 한 비용이 청구 (.99 USD부터 시작)의 사용 가능한 가격 계층 중 하나를 선택 하도록 하려는 경우.

또한 가격 변경 예약을 통해 추가 기능의 가격이 변경되는 날짜와 시간을 표시할 수 있습니다. 뿐만 아니라 특정 시장에서 가격 변경을 사용자 지정할 수 있는 옵션도 있습니다. 

> [!TIP]
> 구독 추가 기능에 대 한 이후 전송에서 더 높은 기본 가격을 선택 하 여 또는 가격을 증가 하는 가격 변동을 예약 하 여 추가 기능을 게시 한 후 가격을 발생 시킬 수 없습니다. 이러한 방법 중 하나를 사용 하 여 더 낮은 가격으로 선택할 수 있지만 가격 인하 되 면 해당 새 가격 보다 높습니다 발생 하는 일을 할 수 없습니다. 이 때문에 구독 추가 기능에 대 한 적절 한 가격 계층을 선택 하도록 하려면 특히 유용 합니다. 

자세한 내용은 [앱 가격 설정 및 예약](set-and-schedule-app-pricing.md)을 참조하세요.


## <a name="sale-pricing"></a>할인 판매 가격 책정

한정된 기간 동안 추가 기능을 할인된 가격으로 제공하려는 경우 할인 판매를 만들고 예약할 수 있습니다. 자세한 내용은 [할인 앱 및 추가 기능 판매](put-apps-and-add-ons-on-sale.md)를 참조하세요.
