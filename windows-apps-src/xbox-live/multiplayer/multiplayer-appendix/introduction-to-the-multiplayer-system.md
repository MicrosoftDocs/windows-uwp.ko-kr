## <a name="about-the-multiplayer-system"></a>멀티 플레이 시스템에 대 한


멀티 플레이 2015 MPSD 및 게임 세션을 직접 사용을 최적화합니다. 단일 제목 또는 사용자에 대 한 여러 동시 세션에 대 한 더 나은 지원을 제공 하 고 제목을 사용 하도록 설정 하려면 Xbox 서비스 API (XSAPI)를 업데이트:

-   사용자의 현재 동작과 조인 가용성을 알립니다.
-   사용자가 볼 수 (제목을 지정) 컨텍스트 문자열과 함께 세션에 초대를 보냅니다.
-   검색 하 고 제목 코드를 통해 세션에 참가할 합니다.
-   간략 한 알림 (어깨 임) 세션 변경 내용에, 예를 들어 변경 이벤트와 연결 상태 변경에 대 한 구독을 반영 하는 업데이트를 받을 수 있도록 MPSD에 대 한 웹 소켓 연결을 유지 합니다. 또한 MPSD 빠르게 검색 하 고 클라이언트 연결 끊기에 따라 동작을 웹 소켓 연결을 사용 합니다.
-   스마트 매치를 사용 합니다.

## <a name="multiplayer-components-interfaces-and-architectures"></a>멀티 플레이 구성 요소, 인터페이스 및 아키텍처

### <a name="components-of-2015-multiplayer"></a>2015 멀티 플레이어의 구성 요소

멀티 플레이어 시스템이 여러 구성 요소로 이루어져 있습니다. 것 외부 매치 메이 킹 시스템과 전용된 서버와 같은 다른 구성 요소가 수 있을 만큼 유연 합니다.
#### <a name="multiplayer-session-directory-mpsd"></a>멀티 플레이 세션 디렉터리(MPSD)


멀티 플레이 세션 디렉터리 (MPSD) 세션의 컬렉션을 보유 하는 서비스를입니다. 세션은 클라우드에 거주 하 고 게임 플레이 사용자의 그룹을 나타내는 보안 문서로 정의 됩니다. MPSD 세부 정보 [멀티 플레이어 세션 디렉터리 (MPSD)를](multiplayer-session-directory.md)참조 하세요.


#### <a name="multiplayer-apis"></a>멀티 플레이 Api

멀티 플레이 2015 **Microsoft.Xbox.Services.Multiplayer Namespace**통해 구현 하는 멀티 플레이어 WinRT API를 제공 합니다. 해당 구성 요소는 정의 하는 **MultiplayerService 클래스**, MPSD 래핑하는 웹 서비스를 포함 합니다.

멀티 플레이어 REST API를 포함 합니다. WinRT API 메서드에서 호출 되는 Uri 및 JSON 개체를 정의 합니다. 나머지 기능 제목에서 직접 호출에 사용할 수 있지만 MPSD WinRT API를 통해 간접적으로 액세스 하는 것이 좋습니다. 자세한 내용은 **MPSD 호출**을 참조 하세요.

#### <a name="xbox-party-system"></a>Xbox 타사 시스템

2015 멀티 플레이어 Xbox 타사 시스템 외부 엔터티로 채팅 당사자만을 지원합니다. 타이틀 채팅 당사자에 게 구성원을 검색 하 타사 시스템과 상호 작용할 수 있습니다. 자세한 내용은 **2015 멀티 플레이어에서 지 원하는 당사자를**참조 하세요.

이제 타사 시스템 게임 타사를 사용 하는 대신 MPSD 세션을 통해 직접 게임을 지원 합니다. 같은 작업이 구성원 상호 작용, 공간을 사용할 수 있게 되 면 게임에 플레이어의 끌어오는 및 대기 기간 동안 사용자의 참여를 사용 하도록 설정 하려면 세션을 사용 하 여 제목입니다.


### <a name="2015-multiplayer-interfaces"></a>멀티 플레이 2015 인터페이스

2015 멀티 플레이어는 다른 Xbox 구성 요소를 여러 인터페이스를 사용합니다.
#### <a name="xbox-secure-sockets"></a>Xbox 보안 소켓

멀티 플레이 2015 Xbox 보안 소켓 및 Winsock을 사용 하 여 장치 간에 하위 수준 네트워크 통신을 지원 합니다. 네트워킹 기능 타이틀은 보안 장치 연결을 제공할 수 있도록 인터넷 프로토콜 보안 (IPSec)을 사용 합니다. Xbox 보안 소켓 기능의 세부 정보에 대 한 **네트워킹 개요** 를 참조 하세요.


#### <a name="xbox-live-real-time-activity-service"></a>Xbox Live Activity 서비스

멀티 플레이 2015 **activity 서비스** 를 사용 하 여 타이틀 MPSD 세션 변경에 가입할 수 있도록 하 고 클라이언트의 자동 검색 사용 분리 합니다. 자세한 내용은 [실시간 활동 (RTA) 서비스](../../real-time-activity-service/real-time-activity-service.md)에서 제공 됩니다.


#### <a name="xbox-live-matchmaking-service"></a>Xbox Live 매치 메이 킹 서비스

**매치 메이 킹 서비스** 의 기본 설정 및 데이터 및 스마트 매치 메이 킹 하는 동안 제공 된 정보에 따라 플레이어에서 그룹을 형성 합니다. 이 서비스의 멀티 플레이어의 세부 정보에 대 한 [스마트 매치](smartmatch-matchmaking.md)를 참조 하십시오.


#### <a name="xbox-live-reputation-service"></a>Xbox Live 신뢰도 서비스

*신뢰도 서비스* 에 대 한 신뢰도 및 매치 메이 킹 평판으로 필터링 하는 동안 사용자 통계를 관리합니다. 스마트 매치를 통해 2015 멀티 플레이어에서 사용 됩니다. 자세한 내용은 [평판](../../social-platform/people-system/reputation.md)을 참조 하세요.


#### <a name="xbox-live-compute"></a>Xbox Live 계산

Xbox Live 계산 처리 능력 제목 2015 멀티 플레이어를 사용 하 여 클라우드를 제공 합니다. Xbox Live 계산을 사용 하 여 자세한 정보를 [사용 하 여 Xbox 멀티 플레이어에서 계산 Live](https://developer.microsoft.com/en-us/games/xbox/docs/xboxlive/xbox-live-partners/xbox-live-compute/using-xbox-live-compute-in-multiplayer)를 참조 하세요.

## <a name="parties-supported-by-2015-multiplayer"></a>2015 멀티 플레이어에서 지 원하는 당사자
Xbox One 용 2015 멀티 플레이어 시스템 수준 구문으로 "게임 자"를 노출 하지 않습니다. 그러나 "파티 채팅" 2014 멀티 플레이어와 마찬가지로 시스템 수준 지원지 않습니다.

| 참고                                                                                                                    |
|--------------------------------------------------------------------------------------------------------------------------------------|
| 제목에 계속 것과 유사한 구현 게임 타사 사용 하지만 대신 MPSD 세션을 사용 하 여 사용자 환경을 구현할 수 있습니다. |


### <a name="chat-party"></a>파티 채팅

채팅 파티 채팅 서로, Xbox One 타사 앱을 통해 사용자가 관리 하는 사람들의 그룹입니다. 사용자가 게임 세션에서 재생 하는 동안 또는 다른 콘솔 작업을 수행 하는 동안 파티 채팅 될 수 있습니다. 하지만 채팅 타사에서 사용자와 해당 사용자에 대 한 기타 활동 간의 없는 동등한 됩니다.

채팅 자는 *PartyChat 클래스*를 통해 채팅 타사의 상태를 검사 하려면 제목을 사용 하 여 노출 됩니다. 예를 들어 제목을 *PartyChat.GetPartyChatViewAsync 메서드*를 사용 하 여 채팅 타사의 멤버를 열거할 수 있습니다.


### <a name="implementing-features-similar-to-those-related-to-the-game-party"></a>게임 당사자와 관련 된 것과 유사한 기능을 구현 합니다.

2014 멀티 플레이어 게임 타사 몇 가지 목적을 제공 합니다. 것에 제목을 사용할 수 있습니다.

-   사용자의 현재 동작과 조인 가용성 보급
-   (초대) 세션에 초대 보내기
-   검색 및 세션에 참가
-   파티에 등록 된 세션에 특정 변경 시 알림 수신
-   스마트 매치 메이 킹 사용
-   여러 게임 세션에서의 사용자 그룹을 함께 유지

멀티 플레이 2015 MPSD 세션을 통해 직접 위의 모든 기능을 지원합니다.

## <a name="multiplayer-terminology"></a>멀티 플레이 용어


| 용어                                 | 설명|
| --- | --- |
| 활성 플레이어                        | 플레이어가 세션 내에서 활성 상태로 설정 되었습니다. 제목은 플레이어가 게임에 참여 하는 경우이 상태로 플레이어를 설정 합니다. 자세한 내용은 [세션 사용자 상태](mpsd-session-details.md)를 참조 하세요.                                                                                                                                                                                                                                                                                          |
| 중재자                              | 예를 들어 광고에 더 많은 플레이어를 찾으려고 매치 메이 킹 게임 세션에는 게임의 멀티 플레이 세션 디렉터리 (MPSD) 세션의 상태를 관리 하는 게임 세션에서 단일 콘솔. 중재자 제목으로 설정 됩니다. 중재자는 항상 게임의 호스트 아닙니다. [중재자 세션을](mpsd-session-details.md)참조 하세요.                                                                                                                                                                            |
| 정렬된 게임                        | 게임 초대 하는 다른 플레이어를 매치 메이 킹에서 모든의 작업 없이, 한 명의 플레이어를 통해서만 만들어지는의 유형입니다.                                                                                                                                                                                                                                                                                                                                                                                                    |
| 파티 채팅                           | 그룹 채팅 함께 사람입니다. 동일한 활동에서 사용자를 맺을 수 있습니다 하거나 게임, 음악 또는 앱 등의 다양 한 활동 맺을 수 있습니다. **2015 멀티 플레이어에서 지 원하는 당사자를**참조 하세요.                                                                                                                                                                                                                                                                                 |
| 게임 초대                          | 게임 세션에 참가 하 라는 초대 합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| 게임 세션                         | 사용자가 실제로 재생 하는 함께 세션 합니다. 매치 메이 킹 또는 가입 로비, 예를 들어 모든 멀티 플레이어 시나리오는 결국 게임 세션에서 종료 합니다. 세션은 종종 조인을 사용 하도록 설정 하려면 사용자의 현재 활동으로 알려집니다. 최근 빌드하는 데 사용 되는 플레이어가 목록입니다. [MPSD 세션 세부 정보](mpsd-session-details.md)를 참조 하세요.                                                                                                                                                           |
| 게임 세션 호스트                    | 게임을 실행 하는 콘솔 제목 피어 투 피어 네트워크 호스트 기반 아키텍처를 기반으로 시뮬레이션을 재생 합니다. 이 콘솔은 일반적으로 중재자와 동일 하지만 같이 필요가 없습니다.                                                                                                                                                                                                                                                                                                                            |
| 핸들 (또는 세션 핸들)           | 추가 상태와 관련 된 동작 MPSD 세션에 대 한 참조입니다. [MPSD 세션에 처리](multiplayer-session-directory.md)를 참조 하세요.                                                                                                                                                                                                                                                                                                                                                                    |

### <a name="new-concepts-in-2015-multiplayer"></a>멀티 플레이 2015의 새로운 개념


#### <a name="web-socket-connections-to-mpsd"></a>MPSD에 대 한 웹 소켓 연결

MPSD는 이제 타이틀을 웹 소켓 연결을 유지 관리할 수 있습니다. 이러한 연결 하면 클라이언트가 세션을 변경할 때 알림을 받을 수 있습니다. 자세한 내용은 [MPSD 변경 알림을 처리 하 고 연결 끊기 감지를](multiplayer-session-directory.md)참조 하세요.


#### <a name="mpsd-session-handles"></a>MPSD 세션 핸들

2015 멀티 플레이어 입력 된 데이터를 포함할 수 있는 세션에 대 한 참조가 MPSD 세션 핸들에 대 한 지원을 추가 합니다. 자세한 내용은 [MPSD 세션에 처리](multiplayer-session-directory.md)를 참조 하세요.


### <a name="summary-of-new-2015-multiplayer-winrt-api-functionality"></a>새로운 2015 멀티 플레이 WinRT API 기능 요약

새 멀티 플레이 WinRT API 기능 전환에 대 한 제목 이미 2014 멀티 플레이어 게임 타사 API와 함께에서 사용 편의성을 위해 기존 XSAPI를 기반으로 합니다.

2015 멀티 플레이어 *MultiplayerActivityDetails 클래스*에 사용자가 있는 세션의 예를 들어, 사용자의 현재 활동 세부 정보를 나타내는 추가 합니다.

*MultiplayerService 클래스*에 새 기능이 추가 되었습니다. 예로 메서드 및 사용자와 소셜 그룹, 다양 한 필터와 핸들을 사용 하 여 세션의 검색에 대 한 작업을 처리 하는 속성을 읽고 게임 초대 및 세션 전송을 핸들을 사용 하 여 기록 합니다.

*MultiplayerSession 클래스* 는 세션 변경 형식에 작용, 알림, 세션 비교 및 닫힐 때 세션을 설정에 대 한 구독에 기능을 추가 합니다.

*MultiplayerSessionReference 클래스* **IMultiplayerSessionReference**, 지원 간 네임 스페이스를 상속할 변경 됩니다. 클래스는 또한 새 URI 경로를 구문 분석 메서드 있습니다.

## <a name="differences-between-xbox-360-and-xbox-one-mpsd-session-functions"></a>Xbox 360 및 Xbox One MPSD 세션 함수 간의 차이점

| 함수 | Xbox 360 | Xbox One |
|---|---|---|
| **게임 세션 정보 가져오기** | XSessionGetDetails, XSessionSearchByID, 또는 제목 추적을 수행합니다. | 제목에서 MPSD 세션 정보를 요청합니다. |
|**호스트 마이그레이션** | 필요한 경우 제목 XSessionMigrateHost를 호출 합니다. | 마이그레이션의 원인에 따라 제목 세션에 대 한 새 호스트를 할당할 수 또는 새 MPSD 세션을 만들 수 있습니다. |
| **여러 플레이어 세션** | XNetUnregisterKey와 XNetReplaceKey 예를 들어, 한 번에 둘 이상의 세션을 처리 하기 어렵습니다. | 서비스 기반 세션 하나의 세션의 관리 간결 하 고 여러 세션의 처리를 간소화 합니다. |
| **Signouts 하 고 분리** | 제목 처리에 대 한 연결을 끊습니다 및 XCloseHandle 또는 XSessionDelete와 다르게 아웃 합니다. | MPSD는 signouts 간소화 및 처리와 게임 구성에서 설정 된 시간 초과 분리 합니다. |
| **매치 메이 킹** | 매치 메이 킹 클라이언트 기반 쿼리 | 품질 및 제목 내에서 쉽게 백그라운드 매치 메이 킹 더 나은 일치를 허용 하는 서비스 기반 매치 합니다. |


### <a name="sessions"></a>세션

세션에서 Xbox 360 게임 플레이의 인스턴스를 나타냅니다. 사용자가 매치 메이 킹 서비스에서 세션에 대 한 검색 하 고 세션의 끝에 통계를 보고 합니다.

Xbox One에서 세션은 더 포괄적이 고 플레이어의 그룹을 나타냅니다. 세션은 콘솔과 모든 네트워크 연결이 필요 하 고 세션의 모든 사용자 간에 공유 해야 하는 정보를 보유 합니다. 이 정보의 일부 예로 세션 및 사용자 지정 게임 데이터의 각 콘솔의 보안 주소 세션에서 플레이어 수 있습니다.


### <a name="xbox-matchmaking"></a>Xbox 매치 메이 킹

Xbox 360 타이틀 매치 메이 킹 특성의 스키마와 해당 특성을 통해 검색 하는 쿼리 집합을 구성 하 여 수행 합니다. 런타임 시 제목 호스트 중 하나를 선택 세션 또는 하나에 대 한 검색 합니다.

Xbox One에서 매치 메이 킹은 서버 기반 및 플레이어 및 타이틀 더 이상 것인지 결정 호스트 하거나 검색 합니다. 대신, 플레이어의 각 미리 구성 된 그룹 "티켓" 세션 만들고 매치 메이 킹 서비스에 해당 세션을 전송 합니다. 서비스 다음 다른 세션 찾아 새 "대상" 세션을 형성 하는 그룹을 결합 합니다. 클라이언트는 일치 알리고 서비스 품질 (QoS) 게임 플레이 시작 하기 전에 다른 세션 멤버를 사용 하 여 연결 유효성 검사를 수행 합니다.


### <a name="xbox-live-compute-service"></a>Xbox Live 계산 서비스

Xbox Live 계산 하는 서비스에는 Xbox One에서 제공 하는 새로운 제품입니다. 피어 투 피어 네트워크에서 가능한 것 보다 더 큰 멀티 플레이어 시나리오를 통해 더욱 클라우드에 탄력적 계산 강점을 개발자 수 있습니다. Xbox Live 계산 하는 서비스에 대 한 자세한 내용은 XDK 설명서에서 Xbox Live 계산 설명서를 참조 하세요.
