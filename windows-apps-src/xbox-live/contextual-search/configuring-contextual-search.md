# <a name="configuring-contextual-search"></a>상황별 검색 구성

## <a name="configuration-info"></a>구성 정보

### <a name="designing-your-data"></a>데이터 디자인
제목 상황별 검색을 위해을 구성할 때에을 제목을 고유 하 게 설정 하는 것을 고려해 야 합니다.  사용자는 동영상 콘텐츠를 찾고 검색 하려면의 맨 위에 것 무엇입니까?  세분성은 데이터 해야?

상황별 검색 메타 데이터에는 유용한 검사 되도록 이러한 검색 구문을 반영 해야 하므로 콘텐츠를 필터링 및 정렬 하는 기능을 지원 합니다.  적절 한 규칙이 게임 데이터에 두 개의 피벗 생각 하는 것:
1. 예: 상태-게임입니다.  문자, 무기, 맵, 역할, 검사점/스토리 포인트, DLC 수준
2. 사용자 기술-예입니다. K/D 비율, Prestige, 순위, 속도 실행 시간

첫 번째 피벗 콘텐츠 필터링에 대 한 훌륭한 이면 두 번째 정렬에 상당히 유용 합니다.  또한 표시 되는 콘텐츠의 형식에 따라 다양 한 수준의 stat 세분성은 중요 합니다.  브로드캐스트, 제한 된 수의 특정된 시점에 활성 브로드캐스트 없다는 가정 하에 대 한 낮은 세분성 데이터 발생할 가능성이 높음을 성공적으로 검색 해야 합니다.  게임 클립에 대 한 수 계속 증가 하면 더 상세한 세분성 데이터는 더 정확한 결과 확인 하도록 합니다.

이미 Xbox Live 기능 등 다양 한에 대 한 만든이 데이터 또는 Hero 통계를 이미 활용 하 여 좋은 기회 이므로 작업을 완료 매우 높습니다.

통계를 사용 하 여 콘텐츠를 필터링 하는, 하는 경우 모든 상황에 맞는 검색 통계를 언제 라도 쿼리를 사용할 수 있는지 알아야 합니다.  이 지원 하도록 적절 하 게 사용자 이벤트를 디자인 해야 합니다.

예를 들어 SinglePlayerMap 등 MultiplayerMap 통계 필터 콘텐츠를 사용 하는 경우 플레이어만 것 중 하나에서 한 번에 수입니다.  그러나 값이 모두를 언제 든 지 쿼리할 서비스에서 제공 됩니다.  하나를 설정 하면를 지우는 기타 중요 한 것입니다.  문자열 기반 통계에 대 한 빈 문자열은 훌륭한 (하지 않아야 하는 옵션으로 UI 구성에 포함).

### <a name="configuring-a-stat-for-contextual-search"></a>상황별 검색을 위해는 Stat 구성
설정한 이벤트 및 통계를 구동 하는 분류 되 면을 제목 상황별 검색을 위해 구성 하는 것은 쉽습니다.  아직 수행 하지 않은 친숙 한 경우 상황에 맞는 검색 설정에 다른 기존 XDP 또는 파트너 센터 설명서를 참조 하세요.

![](../images/contextual_search/config02.png)

위의 이미지는 서비스 구성 주로 XDP에서 상황에 맞는 검색 타일을 클릭 한 후 기본 페이지의 예시입니다.

이 페이지를 방문한 처음 인 경우에 있기 상황별 검색을 위해 이미 구성 된 통계를 볼 수 있습니다.  상황별 검색을 위해 모든 Hero 통계는 자동으로 설정 때문입니다. 구성에서 이러한 Hero 통계를 유지 해야 하는 있지만 일반적으로 정렬 하는 것에 대 한 뛰어난 (및를 이미 완료)입니다.

현재, 상황별 검색을 위해 구성한 상태 인스턴스의 최대 수는 10입니다.

이 페이지에서는 각 통계를 다음과 같은 방식으로 표현 됩니다. 우선 순위-표시 이름 (Stat 인스턴스 이름)

이러한 각 다음 섹션에서 자세히 설명 되어 있습니다.

![](../images/contextual_search/config01.png)

위의 이미지가 기존 및 새로운 상황별 검색을 stat 만들기 또는 편집 하려는 경우 표시 되는 화면입니다.

stat 상황별 검색을 위해에서 성공적으로 구성 하려면 다음 단계를 수행 합니다.
1. Stat 인스턴스를 선택 합니다.

  ![](../images/contextual_search/config03.png)

  인스턴스는 지원 되는 stat 참고-stat 서식 파일은 허용 되지 않습니다.  또한 stat 인스턴스 (통계 부분 XDP 또는 파트너 센터에서에서 구성 됨)에 대해 설정한 표시 유형을 알고 있어야 합니다.  로 표시 된 통계에만 **열고** 타사 환경에 표시 됩니다.

2. Stat 우선 순위를 선택 합니다. 이것이 검색 환경을/알고리즘에 대 한 다른 사용자를 기준으로이 통계의 중요성을 설명 하는 방법입니다.  허용 되는 값은 1 ~ 10 (1은 가장 높은).  0 또는 무시할이 빈 상태로 둡니다.
3. 표시 이름을 추가 합니다.  최종 사용자에 게 노출 되는 지역화할 수 있는 문자열입니다.
4. 필터링 하거나 결과 정렬 하려면이 stat을 사용할 것인지 확인 합니다.  (의 stat 값 숫자인 경우-정의 된 범위에서 가급적) 특정 인스턴스에서 모두를 확인할 수 있습니다.
5. 상태 값은 표시 하는 방법을 선택 합니다.  경우에는 3 개의 옵션이 있습니다.
   * 원시 값의 상태 이며 범위 요구 사항은 없습니다 표시 됩니다.  가장 잘 정렬에 사용 됩니다.
   * 설정 값을 지역화할 수 있는 개별 문자열에 매핑됩니다.  가장 필터링에 사용 됩니다.
   * 범위 값을 최소 및 최대 범위 내에서 해당합니다.  이 형식으로 필터링 하려는 경우 유용 하 고이 데이터를 정렬 합니다.

#### <a name="explaining-sets"></a>설정 설명
집합에 익숙하지 않은 경우에 최종 사용자에 게 노출 될 수 있는 지역화 가능한 문자열에 가능한 상태 값을 매핑할 수 됩니다.  이들은 사용 하 여 필터링 하려는 상황에 맞는 검색 통계를 매우 중요 합니다.

{% 원시 %} 예를 들어 Stat SinglePlayerMap를 호출 하는 정수 있습니다 가정해 보겠습니다.  최종 사용자에 게 다양 한 숫자를 보여 주는 아무런 의미가 있습니다.  같은 매핑 집합을 만드는 대신 ```{{0, “Blood Gulch”}, {1, “Lockout”}, {2, “Zanzibar”}}```합니다.  사용자에 게 문자열을 표시 합니다. 하지만 수 상황별 검색 검색 쿼리의 일부로 사용 합니다.  정수 또는 문자열 집합에 지역화 된 문자열에 매핑할 수 있습니다.

### <a name="updating-your-contextual-search-document"></a>상황별 검색 문서를 업데이트 하는 중
현재 쉽게 변경할 수 있습니다 통계를 상황에 맞는 검색 문서에만 (즉, 일시적인 데이터) 지금은 브로드캐스트를 지원 했습니다.  (2015 년 9 월) 태그를 추가 하거나 통계를 제거 하기로 게임 클립을 지원 하기 위해 먼저 되 면 이미 인덱싱된 있는 클립 영향 됩니다.  새 상태를 추가 하는 경우만 게임 클립 및 브로드캐스트를 만든 후 변경 내용이 적용 됩니다 태그가 지정 되며 새 상태를 사용 하 여 쿼리에서 표시 합니다. 이전 게임 클립 쿼리가 가능이 통계에서 되지 않습니다. 상황별 검색 문서에서는 상태를 제거 하면 게임 클립에 연결 된 이전 stat 쓸모 없게 됩니다 및 반환 되지 않습니다.
