# <a name="rich-presence-policies-and-limitations"></a>다양 한 상태 정책 및 제한 사항

타이틀에 대 한 다양 한 상태를 구현 하는 경우 다음 정책 및 제한을 준수 해야 합니다.

-   각 제목 이상 1 문자열 집합이 있어야 합니다. 하지만 수 있는 얼마나 많은 문자열 집합에는 위 제한이 없습니다.
-   각 다양 한 상태 문자열 및 각 열거형에 대 한 중간 문자열 문화권 뿐만 아니라 기본 문자열을 정의 해야 합니다.
-   또는 사용 하 여 숫자 통계 문자열에 매개 변수를 설정 하는 문자열입니다. DateTime 통계를 사용할 수 없습니다.
-   통계를 다양 한 상태 문자열에 사용 하는 경우 (통계에 대 한 모든 열거형 포함) 이러한 통계 동일한 서비스 안내 및 샌드박스 사용할 수 있어야 합니다.
-   (매개 변수의 값 포함) 총 44 문자 1 회선 있습니다. Xbox 360 다양 한 상태 제한 하는 것과 비슷합니다. 문자열의 길이 수 증가 하는 경우 확인 하려면 고객과 작업할 했습니다. 문자열 더 수 경우 공지가 됩니다.
    -   유니코드 문자는 필수 하 고 디스플레이 대 한 인코딩을 u t F-8 작업할 수 있어야 합니다.
-   사용자 이름이 이러한 규칙을 수행 해야 합니다.
    -   허용 된 문자는 'A' 'Z', 'a' ~ 'z'를 통해 밑줄 ('\_ '), 및 ' 0'부터 ' 9'.

        문자 제한은 없습니다.

-   문자열;에서 작업을 문자열 확인 안 함 맞춤법 검사 및 문자열 제대로 지역화 되어 있는지 확인 등의 모든 문자열 유효성 검사를 수행 해야 합니다.
    -   우리는 문자열 집합 (예: 남용 또는 불쾌 언어) 적합 하지 않습니다. 그래야 듣는 사람도, Microsoft는 만족도 하도록 업데이트 되었으며, 문자열 될 때까지 다양 한 상태를 사용 하 여 제목을 수 없습니다.
-   타이틀 데이터 플랫폼에서는 통합 없는 경우 문자열에 매개 변수로 통계를 사용 하기 위한 옵션이 있습니다.
    -   모든 문자열 (토큰이 없습니다 172.16.10.255) 해당 경우에서 완전히 미리 정의 된 이어야 합니다.
-   열거형 모든 열거형에서 고유 해야 이름과 통계 이름은 고유 해야 합니다.
-   줄을 표시할 수 있는 문자 수를 초과 이며 줄 바꿈, 줄 자동으로 잘립니다.
