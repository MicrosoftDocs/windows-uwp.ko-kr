# <a name="device-information-properties"></a>디바이스 정보 속성



**중요 API**

- [**Windows.Devices.Enumeration**](/uwp/api/Windows.Devices.Enumeration)

각 장치에는 특정 정보가 필요 하거나 장치 선택기를 빌드할 때 사용할 수 있는 연결 된 [**Deviceinformation**](/uwp/api/Windows.Devices.Enumeration.DeviceInformation) 속성이 있습니다. 지정 된 특성을 가진 장치를 찾기 위해 열거 하는 장치를 제한 하는 AQS 필터를 이러한 속성으로 지정할 수 있습니다. 이러한 속성을 사용 하 여 각 장치에 대해 반환 하려는 정보를 나타낼 수도 있습니다. 그러면 응용 프로그램에 반환 되는 장치 정보를 지정할 수 있습니다.

장치 선택기에서 [**Deviceinformation**](/uwp/api/Windows.Devices.Enumeration.DeviceInformation) 속성을 사용 하는 방법에 대 한 자세한 내용은 [장치 선택기 빌드](build-a-device-selector.md)를 참조 하세요. 이 항목에서는 정보 속성을 요청 하는 방법에 대해 설명 하 고 몇 가지 공통 속성과 용도를 나열 합니다.

[**Deviceinformation**](/uwp/api/Windows.Devices.Enumeration.DeviceInformation) 개체는 id ([**DeviceInformation.Id**](/uwp/api/windows.devices.enumeration.deviceinformation.id)), 종류 ([**deviceinformation. kind**](/uwp/api/windows.devices.enumeration.deviceinformation.kind)) 및 속성 모음 ([**deviceinformation. 속성**](/uwp/api/windows.devices.enumeration.deviceinformation.properties))으로 구성 됩니다. **Deviceinformation** 개체의 다른 모든 속성은 **속성** 속성 모음에서 파생 됩니다. 예를 들어 [**Name**](/uwp/api/windows.devices.enumeration.deviceinformation.name) 은 **system.object 표시**에서 파생 됩니다. 즉, 속성 모음에는 항상 다른 속성을 결정 하는 데 필요한 정보가 포함 됩니다.

## <a name="requesting-properties"></a>요청 속성

[**Deviceinformation**](/uwp/api/Windows.Devices.Enumeration.DeviceInformation) 개체에는 [**Id**](/uwp/api/windows.devices.enumeration.deviceinformation.id) 및 [**Kind**](/uwp/api/windows.devices.enumeration.deviceinformation.kind)와 같은 몇 가지 기본 속성이 있지만 대부분의 속성 [**은 속성의**](/uwp/api/windows.devices.enumeration.deviceinformation.properties)속성 모음에 저장 됩니다. 이로 인해 속성 모음에는 속성 모음에서 속성을 원본으로 사용 하는 데 사용 되는 속성이 포함 되어 있습니다. 예를 들어 [System. ItemNameDisplay](/windows/desktop/properties/props-system-itemnamedisplay) 를 사용 하 여 [**Name**](/uwp/api/windows.devices.enumeration.deviceinformation.name) 속성을 원본으로 사용 합니다. 이는 사용자에 게 친숙 한 이름을 가진 일반적인 속성 및 잘 알려진 속성의 경우입니다. Windows에서는 속성 쿼리를 보다 쉽게 수행할 수 있는 여러 가지 친숙 한 이름을 제공 합니다.

속성을 요청 하는 경우 사용자에 게 친숙 한 이름을 가진 공용 속성으로 제한 되지 않습니다. 개별 장치 또는 드라이버에서 제공 하는 사용자 지정 속성을 포함 하 여 사용할 수 있는 속성을 요청 하는 기본 GUID 및 속성 ID (PID)를 지정할 수 있습니다. 사용자 지정 속성을 지정 하는 형식은 " `{GUID} PID` "입니다. 예: " `{744e3bed-3684-4e16-9f8a-07953a8bf2ab} 7` ". 

> [!Note]
> 장치 드라이버의 장치 속성 키 헤더 파일에서 속성 Guid의 목록을 찾을 수 있습니다.

일부 속성은 모든 [**Deviceinformationkind**](/uwp/api/windows.devices.enumeration.deviceinformationkind) 개체에 공통적으로 적용 되지만 대부분은 특정 종류에 대해 고유 합니다. 다음 섹션에서는 개별 **Deviceinformationkind**를 기준으로 정렬 된 몇 가지 공통 속성을 나열 합니다. 서로 다른 종류와 서로 관련 된 방법에 대 한 자세한 내용은 **Deviceinformationkind**를 참조 하세요.
