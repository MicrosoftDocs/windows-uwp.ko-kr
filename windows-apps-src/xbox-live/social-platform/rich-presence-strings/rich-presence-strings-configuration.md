# <a name="rich-presence-configuration"></a>대화 상대의 구성

대화 상대의 문자열을 구성 하는 데 전용 Xbox 개발자 포털 (XDP)의 일부가 됩니다. XDP에서 사용할 수 있는, 될 때까지 수동 구성을 사용 해야 합니다. 수동 구성 개발자 계정 관리자에 의해 제공 하 고 환경에 수집에 대 한 개발자 계정 관리자에 게 보내기 스프레드시트 템플릿에서 문자열을 정의 하는 것입니다.

## <a name="strings"></a>문자열

서식 있는 현재 상태 문자열에 문자열 집합을 정의 해야 합니다. 각 문자열 집합에는 식별자로 사용할 이름을 지정 되어야 합니다. 만들어진 각 문자열 집합 그룹 제목 출시 될 로캘에 대 한 적절 하 게 지역화 된 문자열을 포함 해야 합니다. 지역화 된 문자열을 외에 culture 중립 문자열도 제공 해야 합니다. 이 문자열은 지정 되지 않습니다는 로컬에 대 한 요청 하는 경우 사용 합니다.

각 문자열 집합 작업도 가능을 더욱 향상 시킵니다 게임 내 데이터 문자열을 매개 변수화 하 여 사용 합니다. 문자열 집합을 결과 문자열 문자 제한 규정을 통해 이동 하지 않는 된다면 많은 매개 변수를 원하는 대로 있을 수 있습니다. 참조 [대화 상대의 문자열: 정책 및 제한 사항](rich-presence-strings-policies-and-limitations.md) 대 한 자세한 내용은 합니다.


## <a name="enumerations"></a>열거형

아래에서 설명 하는 열거형 데이터 플랫폼에서 이벤트를 통해 정의 됩니다.


## <a name="statistics"></a>통계

대화 상대의 문자열에서 숫자 통계 값을 사용할 수 있습니다. 예를 들어, 한 슈팅에서 수도 있습니다 플레이어 지금 해달라는 얼마나 많은 파괴 표시할 대화 상대의 문자열입니다. 구성 매개 변수를 "Kill Count" 통계 바뀌어야 함을 표시 해야 합니다. "Kill Count" 통계 데이터 플랫폼을 통해 일반적인 스키마 이벤트 시스템을 통해 만들어지고 통계 구성을 통해 정의 해야 합니다. 문자열을 다른 사용자가 읽을 때 서비스를 이동 하 고 문자열을 항상 최신 상태로 유지할 수 있도록 "Kill Count" 통계에 대 한 최신 값을 검색 합니다.


## <a name="state"></a>상태

또한 대화 상대의 문자열의 실제 상태 값을 나타내는 통계를 사용할 수 있습니다. 예를 들어 레이싱 게임에서 플레이어를 원하면 현재는 자동차 표시할 대화 상대의 문자열 또는 지도 또는 플레이어에서 racing는 추적을 사용할 수 있습니다. 상태 정보를 구성 하는 방법은 다음 두 단계가 있습니다.

1.  상태 정보에 대 한 모든 게임에서 문자열에 대 한 열거형을 정의 합니다. 예를 들어 게임에서 맵 5 개가 있고 맵 대화 상대의 문자열의 일부가 되도록 하려면, 경우 맵 5 개가 해야 합니다도 지역화 된 문자열을 열거할 수 있습니다.
2.  통계 열거를 연결 합니다. 여기에서 다시 검색 됩니다. 최신 값 데이터 플랫폼에서 통계의 값을 조회 하 여 합니다. 다음 통계를 변환 하는 열거형 값을 결정 합니다.


## <a name="ingestion"></a>수집

스프레드시트를 마친 후 수집에 대 한 개발자 계정 관리자에 전송 합니다. 대화 상대의 팀 구성을 검토 하 고 올바르게 구성 되어 있고가 해당 서비스에 수집 된는 있는지 확인 합니다.


## <a name="example-game"></a>예제 게임

예제의 나머지 부분을 게임을 시작 하는 최신 버킷에 작업 하 고 가정해 보겠습니다. 버킷 나무 버킷, 실버 버킷 및 golden 버킷와 같은 게임에서 다양 한 유형의 여러 가지가 있습니다. 시작 버킷의 각 형식에 대 한 통계를 결정 합니다. 산, Desert, the Beach 등 버킷의 시작을 게임에서 다양 한 위치 이기도 합니다. 버킷의 시작 되 면 해당 파손, 항상 버킷을 더 많이 검색 수 있다면 합니다. 일부 버킷은 해야 (부팅을 시작 하는 "오버 커밋됨" 충분히 강력한 필요) 하는 특별 한 기어 버킷 성공적으로 시작 합니다. 게임에는 충분 한 플레이어 버킷 동시에 시작을 시도 하는 경우 적절 한 부팅 필요 하지 않습니다 멀티 플레이 모드도 포함 됩니다.

이 게임에 대 한 어떤 대화 상대의 문자열 구성이 다음과 같다는 살펴보겠습니다.


## <a name="configuring-statistics-for-rich-presence"></a>서식 있는 현재 상태에 대 한 통계를 구성합니다.

## <a name="string-configuration-example"></a>문자열 구성 예제

열거형 및 통계에 정의 된 했으므로 문자열을 만들 수 있습니다.

이 예에서는 테이블은 아래에서 첫 번째 열에는 다른 스레드에 대 한 문자열 집합을 참조 하는 데 이름입니다. 이 예제에서는 두 번째 문자열 집합을 세 번째 "멀티 플레이 게임"에 대 한 그 다음으로 첫 번째 문자열 집합에 대 한 "playingMap" 문자열 "totalKicked"를 사용 하기로 했습니다.

다음 두 열 함께 이동합니다. 이들은 다양 한 현재 상태 문자열에 대 한 로캘 문자열 쌍입니다. 첫 번째 예에서는 영어 문자열을 사용 하려는 "에서 재생 {0}"입니다. {0} 토큰 값으로 대체 됩니다. 에서는 다음 다른 로캘에 대 한 문자열을 지역화합니다. 열거형에서와 마찬가지로 사용자 로캘 문자열 쌍을 지정 하지 않은 로캘을 사용 하는 경우에서 사용할 문화권 중립 문자열 뿐만 아니라 기본적으로 있어야 합니다. 열거형에 사용 되는 로캘 문자열에 대 한 로캘 일치 해야 합니다.

마지막 열은 다양 한 현재 상태 문자열에 공백을 입력 하는 데 매개 변수입니다. 통계 서비스에서 매개 변수의 값을 찾는 현재 상태를 원한다 면 여기에 매개 변수를 참조 하려면 상태 이름을 사용 해야 합니다. 사용자 통계를 사용 하 여 문자열에서 하면 않을 문자열을 설정 하 고 다음에 대 한 걱정 하지 수 있습니다. 문자열에 맵 이름, 빈 칸에 맞게 CurrentMap Stat를 사용 하는 경우 다음 서비스에서 업데이트 문자열을 적절 하 게 플레이어 게임에서 맵을 이동할 때. 대화 상대의 문자열에서 통계를 사용 하지 않는 경우 매개 변수를 지정할 수 있습니다.
