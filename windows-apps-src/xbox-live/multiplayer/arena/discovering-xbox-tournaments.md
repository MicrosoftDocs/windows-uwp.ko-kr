# <a name="discovering-xbox-tournaments"></a>Xbox 토너먼트 검색

게이머 여러 옵션이 Xbox 대시보드의 또는 Xbox 앱에서 PC에 참여 또는 토너먼트를 감시 하는 방법에 대 한 자세한 내용을 보려면. 제목 내에서 토너먼트 검색을 늘릴 수 있습니다.  

퇴근 Arena 지원 토너먼트에서 찾을 수 있습니다.

* Xbox 집에서 최근에 재생된 게임의 토너먼트 허브 열기
* 플레이어 토너먼트 재생, 지금 재생에 대 한 등록 프로필을 방문 합니다.
* 에 속해 클럽에서 찾고 (LFG) 그룹에 대 한 게시 팀에 응답 합니다.
* Mixer 앱, 웹 사이트 또는 조사식 피벗 게임 허브/Arena에서 토너먼트 스트림을 확인합니다.
* Xbox 대시보드에서 게임 허브를 방문 합니다.

## <a name="promoting-tournaments-in-game"></a>승격 토너먼트 게임

사용자는 제목에 관심이 있으므로 토너먼트를 시청 하세요. Xbox 게임 고유한 기회 게이머는 게임 플레이 환경을 확장 하는 새로운 방법을 찾고 있는 경우 중요 한 특정 시점에서 찾으려는 경우

> [!TIP]
> **UX 권장 사항:** 의사 결정 순간 토너먼트를 승격 합니다.
>
> 재생을 유지 하거나 (예를 들어 게임 시작, 주 메뉴, 멀티 플레이 메뉴 또는에서 일치 항목의 끝) 종료 여부를 결정 하는 것이 게이머 경우 토너먼트에 대 한 인지도 시간에 발생 합니다.

### <a name="1--use-a-dynamic-announcement-feature-example-message-of-the-day"></a>1.  동적 알림 기능을 사용 하 여 (예: 하루 중 메시지)

게임 UI에 알림 기능을 일반적으로 작성 되어 타이틀 즉시 업데이트를 허용 하는 콘텐츠 관리 시스템을 사용 합니다. 게이머에 콘텐츠 업데이트 또는 다운로드 가능한 콘텐츠 출시에 의존 하지 않고 새 콘텐츠를 푸시할 때 유용 합니다. 예를 들어, Halo 노출 커뮤니티 공지 사항 및 팁을 "메시지의 the Day" 기능을 사용 합니다. 토너먼트 프로 모션은이 시나리오에 대 한 완벽 한 적합 합니다.

**사용자 영향**

* 게이머는 게임 세션을 시작 하기 전에 새 경쟁 게임 기회가 소개 됩니다.
* (다른 발표와 함께 시작) 시 일시적인를 노출 합니다.
* 자세한 게이머 게임을 Arena 허브로 전송 됩니다.
* 기능을 채우는 콘텐츠 업데이트가 필요 하지 않습니다.
* 관련성 및 승격의 타이밍 유연한 상태로 유지 됩니다.

## <a name="browsing-tournaments-in-game"></a>토너먼트 게임 검색

Arena Api는 게임 내에서 검색 환경을 만들을 제목에 대 한 충분 한 데이터를 제공 합니다. 검색 기능을 빌드 하려는 경우 추가 **토너먼트** 멀티 플레이어 섹션의 진입점입니다.

> [!TIP]
> **UX 권장 사항**  
> 멀티 플레이 게임 모드로 토너먼트를 제공 합니다.  
> Xbox Arena 새로운 기능 이므로 계층 1 또는 계층 2 수준에서 표시 되도록 해야 합니다.

###### <a name="ui-example-tournaments-listed-as-an-additional-mode-in-the-multiplayer-section"></a>UI 예제: 추가 모드로 멀티 플레이어 섹션에 나열 된 토너먼트

![토너먼트 모드](../../images/arena/arena-ux-tournament-mode.png)

### <a name="provide-a-comprehensive-list-of-tournaments"></a>토너먼트의 종합적인 목록을 제공 합니다.

게이머 신속 하 게 토너먼트에서는 현재 등록 상태를 확인 하 고 토너먼트 게임 플레이 세션 간에 이동할 수 있습니다.

#### <a name="user-impact"></a>사용자 영향

* 대시보드에서 토너먼트 상태를 확인 하려면 게임에서 나가려면 게이머의 필요를 최소화 합니다.
* 게이머 Xbox Arena 알림을 수행 하지 못한 경우 좋은 백업 솔루션으로 사용 됩니다.
* 관리 하는 게임 개발자를 위한 추가 비용이 포함 됩니다.
* 게이머 조인, 등록, 체크 인, 시드 및 팀 구성에 대 한 Arena UI에 연결 됩니다.

> [!NOTE]  
> Arena Api는 사용자가 생성 토너먼트 (클럽 생성)에 대 한 쿼리를 제공 하지 않습니다.


> [!TIP]
> **UX 권장 사항**  
> 크기를 조정 하 고 큰 토너먼트 목록을 관리 하는 수에 대 한 메서드를 제공할 수 있는 UI를 만듭니다.

### <a name="filters"></a>필터

필터링 하 여 토너먼트 **모든**, **Active** (대회가 열릴 때까지 모든 항목을 다루는 종료) **Canceled**, 및 **완료** 상태 및 토너먼트 퇴근 되었거나 (예: 내 토너먼트)에 참여 합니다.

###### <a name="ui-example"></a>UI 예제:

![토너먼트 필터 화면](../../images/arena/arena-ux-filters.png)

> [!NOTE]  
> API에서 지 원하는 작업 외부에서 사용자 지정 필터를 추가 하는 것이 가능 합니다 *있지만 권장 되지는*합니다.

요청에 있으 나 쿼리 매개 변수에 지정할 수 없습니다이 후 다른 속성을 제목 필터링 수 있습니다. 이렇게 하면 UI의이 형식에 대 한 신뢰할 수 없는 다른 속성별로 필터링 합니다. Arena Api를 한 번에 제목이 지정 된 수의 결과 검색-예를 들어 제목을 지정할 수는 **maxItems** 10 토너먼트의 값입니다. 성능 관리을 제목 고 위험 큰 목록이 있는 사용자를 부담을 최소화할 수 있습니다. 그러나 게임에서 제공 되는 모든 사용자 지정 필터는 해당 시점에 요청 된 10 개 항목에만 적용 됩니다. 이 API 호출한 후 마다 일관 되지 않은 필터링 된 결과 수가 될 수 있습니다. 예를 들어 10 토너먼트 반환 된 첫 번째 집합 '재생 합니다.'와 같은 사용자 지정 필터를 관련 된 2 포함 하지만 다음 10 9의 이러한 항목을 표시 하 고 등 수 있습니다. 신뢰할 수 있는 유일한 방법은 각 사용자 지정 필터를 완전히 채울을 제목에 대 한 모든 활성 토너먼트 요청 및 필터링 하는 것이 좋습니다 없는 경우

#### <a name="filter-recommendations"></a>필터 권장 사항:

##### <a name="all"></a>ALL

모두 보기 **Active**를 **Canceled**, 및 **완료** 토너먼트, 대부분 정렬할 최근.

결과:

* 토너먼트 상태
* 토너먼트 시작 날짜
* 토너먼트 상태 — 예: 등록 열기
* 대시보드에서 토너먼트 세부 정보 페이지에 딥 링크

##### <a name="my-tournaments"></a>내 토너먼트

참가자에 현재 등록 되어 있는 보기 토너먼트 합니다.

결과:

* 현재 로그인 한 사용자에 참여 중인 토너먼트
* 토너먼트 상태
* 토너먼트와 일치 하는 (match 준비 상태)을 입력 하는 메서드
* 대시보드에서 토너먼트 세부 정보 페이지에 딥 링크

##### <a name="active"></a>ACTIVE

생성 된 모든 토너먼트 보기: 예정, 등록, 체크 인 및 재생을 위해 열려 있습니다.

결과:

* 현재 사용자 아직 조인 되지 않은를 등록 하려면 열려 있는 토너먼트
* 등록을 닫을 때까지 남은 시간

##### <a name="completedcanceled"></a>완료/취소

최근 결과 확인 합니다. 방금 종료 토너먼트 완료 된 후에 간단히 사라지는 대신 시간이 지남에 따라 에이징 아웃 수 있습니다.

결과:

* 대시보드에서 토너먼트 세부 정보 페이지에 딥 링크
* 토너먼트 상태
* 종료 날짜/시간

##### <a name="canceled"></a>취소

취소 된 보기 토너먼트 합니다.

결과:

* 대시보드에서 토너먼트 세부 정보 페이지에 딥 링크
* 취소 날짜/시간
